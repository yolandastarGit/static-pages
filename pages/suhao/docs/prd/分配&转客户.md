# 操作一：分配线索（PRD 页面设计）

## 一、操作概述

**操作名称：** 分配线索

**功能定位：** 将单条线索的负责人从当前负责人变更为其他销售人员。仅变更本条线索的负责人，不影响该客户下其他历史线索，也不影响后续新线索的归属规则。

**适用场景：**
- 临时调整单条线索的跟进人（如负责人请假、离职）
- 线索误分配需要更正
- 主管根据业务需要重新分配线索

**触发入口：**

| 入口 | 位置 | 前置条件 |
| :--- | :--- | :--- |
| 入口一 | 线索列表 → 操作列【更多】下拉 → 【分配线索】 | 单条线索操作 |
| 入口二 | 线索列表 → 顶部【批量分配】按钮 | 选中至少一条线索 |
| 入口三 | 线索详情 → 顶部【更多】下拉 → 【分配线索】 | 当前线索详情页 |

**执行权限：** 当前线索负责人本人 或 销售主管（含以上）


## 二、弹窗字段定义

### 2.1 单个分配（入口一 / 入口三）

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 线索编号 | 只读展示 | — | 展示当前线索编号 |
| 客户名称 | 只读展示 | — | 展示当前线索关联的客户企业名称 |
| 分配状态 | 只读展示 | — | 展示“已分配 / 未分配”，当前线索必有负责人 |
| 站点 | 只读展示 | — | 展示线索所属站点 |
| 当前负责人 | 只读展示 | — | 展示当前线索负责人姓名 |
| 新负责人 | 下拉选择 | 是 | 可选当前站点下所有销售人员 |
| 分配备注 | 多行文本 | 是 | 记录分配原因，必填，便于追溯 |

### 2.2 批量分配（入口二）

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 已选线索数 | 只读展示 | — | 展示已选中的线索数量 |
| 新负责人 | 下拉选择 | 是 | 可选当前站点下所有销售人员 |
| 分配备注 | 多行文本 | 是 | 统一分配原因，所有选中线索共用，必填 |


## 三、弹窗交互说明

### 3.1 单个分配

**打开弹窗：**
- 标题：`分配线索`
- 所有只读字段自动加载当前线索数据
- 新负责人下拉列表加载当前站点下所有销售人员
- 分配备注为空，等待用户输入

**提交校验：**

| 校验项 | 校验规则 | 错误提示 |
| :--- | :--- | :--- |
| 新负责人 | 不可为空 | “请选择新负责人” |
| 新负责人 | 不可与当前负责人相同 | “新负责人与当前负责人相同，无需分配” |
| 分配备注 | 不可为空 | “请输入分配备注” |

**提交成功：**
1. 线索负责人更新为新负责人
2. 线索状态不变（若当前为“跟进中”则保持不变）
3. 自动生成跟进记录：
   - 跟进方式：备注
   - 跟进内容：“由 [原负责人] 分配至 [新负责人]”
   - 跟进人：系统自动记录为操作人
   - 跟进时间：当前时间
4. 关闭弹窗，页面刷新，成功提示：`线索已成功分配给 [新负责人]`

**提交失败：**
- 提示：`分配失败，请稍后重试`
- 保留用户已填内容

### 3.2 批量分配

**打开弹窗：**
- 标题：`批量分配线索（共 N 条）`
- 已选线索数自动展示
- 新负责人下拉列表加载当前站点下所有销售人员
- 分配备注为空，等待用户输入

**提交校验：**

| 校验项 | 校验规则 | 错误提示 |
| :--- | :--- | :--- |
| 新负责人 | 不可为空 | “请选择新负责人” |
| 分配备注 | 不可为空 | “请输入分配备注” |

**提交成功：**
1. 所有选中线索负责人批量更新为新负责人
2. 各线索状态不变
3. 每条线索自动生成跟进记录：
   - 跟进方式：备注
   - 跟进内容：“由 [原负责人] 分配至 [新负责人]”（各自记录原负责人）
   - 跟进人：系统自动记录为操作人
   - 跟进时间：当前时间
4. 关闭弹窗，页面刷新，成功提示：`成功分配 N 条线索`

**批量分配部分失败：**
- 弹窗提示：`成功分配 N 条，失败 M 条`
- 失败条目列出具体原因（如线索不存在、无权限等）


## 四、业务规则

| 规则编号 | 规则说明 |
| :--- | :--- |
| BR-025 | 线索任意时刻仅允许一个负责人 |
| BR-028 | 销售主管可重新分配线索 |
| BR-030 | 每次关键沟通或状态变化必须新增跟进记录 |
| BR-031 | 跟进记录仅允许新增，不修改历史，不删除 |

**影响范围说明：**
- 分配线索仅变更**当前线索**的负责人
- 该客户（企业）下**其他历史线索**的负责人保持不变
- 该客户**后续新生成的线索**，仍按原规则（收件账号绑定）自动分配，不受本次分配影响


## 五、异常状态处理

| 场景 | 处理方式 |
| :--- | :--- |
| 当前负责人与新负责人相同 | 提交时校验拦截，提示“新负责人与当前负责人相同，无需分配” |
| 目标负责人账号已停用 | 下拉列表中不展示已停用用户，无法选择 |
| 被分配的线索已被其他人分配 | 提交时校验，若已被分配则提示“线索已被分配，请刷新后重试” |
| 批量分配时部分线索无权限 | 提交时校验，跳过无权限线索，提示“部分线索无权限分配，已跳过” |


## 六、操作对比

| 对比项 | 分配线索 | 转客户 |
| :--- | :--- | :--- |
| 操作对象 | 单条线索 | 客户（含当前线索） |
| 当前线索负责人 | 变更 | 变更 |
| 历史线索负责人 | 不变 | 不变 |
| 客户档案负责人 | 不变 | 变更 |
| 后续新线索归属 | 按原规则分配 | 自动分配给新负责人 |
| 批量操作 | 支持 | 不支持 |
| 触发入口 | 线索列表 / 线索详情 | 线索列表 / 线索详情 |
| 执行权限 | 本人或主管 | 本人或主管 |


# 操作二：转客户（客户级负责人转移）（PRD 页面设计）


## 一、操作概述

**操作名称：** 转客户

**功能定位：** 将线索及其所属客户的负责人从当前负责人变更为其他销售人员。触发该操作的**当前线索**负责人变更为新负责人，**客户档案负责人**变更为新负责人，且该客户**后续新生成的所有线索**自动分配给新负责人。历史线索负责人保持不变。

**适用场景：**
- 客户交接（销售离职、岗位调整）
- 客户归属转移（区域调整、客户分级管理）

**触发入口：**

| 入口 | 位置 | 前置条件 |
| :--- | :--- | :--- |
| 入口一 | 线索列表 → 操作列【更多】下拉 → 【转客户】 | 单条线索操作 |
| 入口二 | 线索详情 → 顶部【转客户】按钮 | 当前线索详情页 |

**执行权限：** 当前线索负责人本人 或 销售主管（含以上）

**前置条件：** 线索状态必须为“高意向”


## 二、弹窗字段定义

### 单条转客户（入口一 / 入口二）

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 线索编号 | 只读展示 | — | 展示当前线索编号 |
| 客户名称 | 只读展示 | — | 展示当前线索关联的客户企业名称 |
| 站点 | 只读展示 | — | 展示线索所属站点 |
| 当前负责人 | 只读展示 | — | 展示当前线索负责人姓名 |
| 客户档案负责人 | 只读展示 | — | 展示当前客户档案负责人姓名（若客户对象无负责人字段，则展示“未设置”） |
| 新负责人 | 下拉选择 | 是 | 可选当前站点下所有销售人员 |
| 转交备注 | 多行文本 | 是 | 记录转交原因，必填，便于追溯 |
| 影响范围说明 | 只读提示 | — | 提示文案：“本次转交后，该客户后续所有新线索将自动分配给 [新负责人]，历史线索负责人保持不变。” |


## 三、弹窗交互说明

### 3.1 打开弹窗

- 标题：`转客户`
- 所有只读字段自动加载当前线索及客户数据
- 新负责人下拉列表加载当前站点下所有销售人员
- 转交备注为空，等待用户输入
- 底部展示影响范围说明（灰色提示框）

### 3.2 提交校验

| 校验项 | 校验规则 | 错误提示 |
| :--- | :--- | :--- |
| 线索状态 | 必须为“高意向” | “当前线索状态非高意向，不可转客户” |
| 新负责人 | 不可为空 | “请选择新负责人” |
| 新负责人 | 不可与当前负责人相同 | “新负责人与当前负责人相同，无需转交” |
| 转交备注 | 不可为空 | “请输入转交备注” |

### 3.3 提交成功

1. **当前线索**负责人更新为新负责人
2. **线索状态**变更为“已转客户”
3. **客户档案负责人**更新为新负责人
4. 系统建立 **客户 → 新负责人** 的绑定关系，后续该客户新生成的线索自动分配给新负责人
5. 自动生成跟进记录：
   - 跟进方式：备注
   - 跟进内容：“客户 [客户名称] 由 [原负责人] 转交至 [新负责人]”
   - 跟进人：系统自动记录为操作人
   - 跟进时间：当前时间
6. 关闭弹窗，页面刷新，成功提示：`客户已成功转交至 [新负责人]`

### 3.4 提交失败

- 提示：`转客户失败，请稍后重试`
- 保留用户已填内容


## 四、业务规则

| 规则编号 | 规则说明 |
| :--- | :--- |
| BR-032 | 线索达到高意向方可转客户 |
| BR-034 | 转客户时校验当前站点客户是否存在，存在则关联，不存在则创建 |
| BR-036 | 转客户时同步 AI 企业信息/画像/风险/标签，不同步意向总结 |
| BR-030 | 每次关键沟通或状态变化必须新增跟进记录 |
| BR-031 | 跟进记录仅允许新增，不修改历史，不删除 |

**影响范围说明：**
- 当前线索负责人变更为新负责人
- 客户档案负责人变更为新负责人
- 该客户（企业）下**其他历史线索**的负责人**保持不变**
- 该客户**后续新生成的所有线索**，自动分配给新负责人


## 五、异常状态处理

| 场景 | 处理方式 |
| :--- | :--- |
| 当前负责人与新负责人相同 | 提交时校验拦截，提示“新负责人与当前负责人相同，无需转交” |
| 目标负责人账号已停用 | 下拉列表中不展示已停用用户，无法选择 |
| 线索状态非高意向 | 入口按钮置灰或隐藏，悬停提示“仅高意向线索可转客户” |
| 转客户过程中线索已被他人转客户 | 提交时校验，若已被转客户则提示“线索已被转客户，请刷新后重试” |


## 六、操作对比

| 对比项 | 分配线索 | 转客户 |
| :--- | :--- | :--- |
| 操作对象 | 单条线索 | 客户（含当前线索） |
| 当前线索负责人 | 变更 | 变更 |
| 历史线索负责人 | 不变 | 不变 |
| 客户档案负责人 | 不变 | 变更 |
| 后续新线索归属 | 按原规则分配 | 自动分配给新负责人 |
| 批量操作 | 支持（批量分配） | 不支持 |
| 前置条件 | 无条件 | 线索状态为“高意向” |
| 触发入口 | 线索列表 / 线索详情 | 线索列表 / 线索详情 |
| 执行权限 | 本人或主管 | 本人或主管 |
| 线索状态变化 | 不变 | 变更为“已转客户” |
