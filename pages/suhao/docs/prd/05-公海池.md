## 一、模块概述

**模块归属：** 线索中心 → 公海池

**模块定位：** 管理所有当前没有负责人的线索，是线索分配与流转的核心管理页面。

**面向角色：** 销售主管、系统管理员


## 二、页面整体布局

公海池页面采用 **标准企业后台列表布局**，从上至下分为三个区域：

| 区域 | 内容 |
| :--- | :--- |
| **筛选条件栏** | 多维度筛选条件，支持精细化查询 |
| **工具栏** | 批量分配、刷新等操作入口 |
| **表格区** | 公海线索数据表格，含分配、回收等操作按钮及分页 |


## 三、筛选条件栏

**功能定位：** 提供多维度筛选条件，支持精细化查询。

### 3.1 筛选字段

| 筛选项 | 类型 | 说明 | 默认值 |
| :--- | :--- | :--- | :--- |
| **站点** | 下拉选择 | 按站点过滤线索，可选范围为用户已授权的站点 | 全部可见站点 |
| **入池时间** | 日期区间 | 按线索进入公海池的时间范围筛选 | 本月 |
| **线索关键词** | 文本输入 | 模糊搜索线索编号、线索名称、客户企业名称 | 空 |
| **来源渠道** | 下拉选择（多选） | 按来源渠道筛选 | 全部 |
| **意向产品** | 下拉选择 | 按 AI 识别的客户感兴趣产品类目筛选 | 全部 |

> 说明：站点筛选是公海池的核心筛选维度。公海池按站点分别管理，不同站点的公海线索相互隔离。

### 3.2 操作按钮

| 按钮 | 说明 |
| :--- | :--- |
| **查询** | 按当前筛选条件刷新表格数据 |
| **重置** | 将所有筛选条件恢复为默认值 |
| **刷新** | 手动刷新公海池数据，获取最新入池/回收线索 |


## 四、表格区

### 4.1 顶部操作按钮

位于表格左上方，提供线索批量操作入口：

| 按钮 | 说明 | 前置条件 |
| :--- | :--- | :--- |
| **批量分配** | 将选中的多条线索批量分配给指定负责人 | 至少选中一条线索 |
| **回收** | 将选中的线索回收至公海池（将已分配线索重新放回公海） | 至少选中一条线索 |

### 4.2 表格字段

| 字段 | 说明 | 展示规则 |
| :--- | :--- | :--- |
| **复选框** | 用于选中单条或多条线索 | 位于表格最左侧 |
| **线索编号** | 系统自动生成的唯一编号 | 固定格式，如 LEAD-2026-0911 |
| **线索名称** | 默认企业名称 | 点击可跳转至线索详情页 |
| **来源渠道** | 线索来源渠道 | 取值：官网询盘、自然询盘、WhatsApp、展会、客户转介绍、其他 |
| **来源站点** | 线索所属站点 | 展示站点名称 |
| **入池时间** | 线索进入公海池的时间 | 展示完整日期时间 |
| **意向产品** | AI 识别的客户感兴趣产品类目 | 展示前 2 个产品类目，超出 2 个时显示“+N” |
| **操作** | 单条线索操作入口 | 包含：【分配】、【查看详情】 |

### 4.3 操作列详细说明

| 按钮 | 说明 | 前置条件 |
| :--- | :--- | :--- |
| **分配** | 弹出分配确认弹窗，将线索分配给指定负责人 | 无条件 |
| **查看详情** | 跳转至线索详情页 | 无条件 |


## 五、分配线索弹窗

### 5.1 单个分配

**触发入口：** 点击操作列的【分配】按钮

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 线索编号 | 只读展示 | — | 展示当前线索编号 |
| 线索名称 | 只读展示 | — | 展示当前线索名称 |
| 来源站点 | 只读展示 | — | 展示线索所属站点 |
| 目标负责人 | 下拉选择 | 是 | 可选当前站点下所有销售人员 |
| 分配备注 | 多行文本 | 否 | 记录分配原因，可选 |

### 5.2 批量分配

**触发入口：** 点击顶部【批量分配】按钮

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 已选线索数 | 只读展示 | — | 展示已选中的线索数量 |
| 目标负责人 | 下拉选择 | 是 | 可选当前站点下所有销售人员 |
| 分配备注 | 多行文本 | 否 | 统一分配原因，所有选中线索共用，可选 |

### 5.3 分配逻辑

1. 选择目标负责人后，点击确认执行分配
2. 分配成功后：
   - 线索的“当前负责人”更新为目标负责人
   - 线索状态从“公海待分配”变更为“已分配”
   - 线索自动移出公海池
   - 系统自动生成一条跟进记录（跟进方式：备注，内容：“由 [操作人] 从公海分配至 [负责人]”）
3. 分配失败时，弹窗提示错误原因

### 5.4 对其他模块的影响

| 影响模块 | 影响内容 |
| :--- | :--- |
| **线索列表** | 分配后的线索出现在线索列表中（状态变更为“已分配”），负责人更新为分配的目标人 |
| **线索详情** | 负责人字段更新 |


## 六、回收线索弹窗

### 6.1 触发入口

**触发入口：** 点击顶部【回收】按钮（仅当选中线索为“已分配”/“跟进中”状态时可用）

### 6.2 弹窗字段

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 已选线索数 | 只读展示 | — | 展示已选中的线索数量 |
| 当前负责人 | 只读展示 | — | 展示各线索当前负责人（列表展示） |
| 回收原因 | 多行文本 | 是 | 记录回收原因 |

> 说明：回收操作仅适用于已分配/跟进中的线索。公海池中线索本身处于未分配状态，无需回收。具体回收范围为“选中线索中已分配/跟进中的条目”。

### 6.3 回收逻辑

1. 回收操作由销售主管或系统管理员执行
2. 回收成功后：
   - 线索的“当前负责人”清空
   - 线索状态变更为“公海待分配”
   - 线索重新进入公海池
   - 入池时间更新为当前时间（重新计时）
   - 系统自动生成跟进记录（跟进方式：备注，内容：“由 [操作人] 回收至公海，原因：[回收原因]”）
3. 回收失败时，弹窗提示错误原因

### 6.4 对其他模块的影响

| 影响模块 | 影响内容 |
| :--- | :--- |
| **线索列表** | 回收后的线索从线索列表中移除（负责人清空，状态变更为“公海待分配”） |
| **线索详情** | 负责人字段清空 |


## 七、自动回收规则

**功能定位：** 系统根据站点配置，自动将超时未跟进的线索回收至公海池。

### 7.1 配置项

| 配置项 | 说明 | 默认值 |
| :--- | :--- | :--- |
| **启用自动回收** | 是否启用该站点的自动回收功能 | 关闭 |
| **回收超时时长** | 线索无跟进记录的天数阈值 | 7 天 |
| **超时计算基准** | 从“最近跟进时间”或“分配时间”开始计算 | 最近跟进时间 |
| **是否允许再次分配** | 回收后是否可再次分配给同一负责人 | 否 |
| **排除状态** | 哪些状态的线索不参与自动回收 | 已转客户、无效 |

> 说明：自动回收配置在站点中心→公海规则中由管理员设置。

### 7.2 自动回收执行逻辑

1. 系统定时任务每日执行一次（如每日凌晨 2:00）
2. 扫描所有站点中已启用自动回收的线索
3. 筛选条件：
   - 线索状态为“已分配”或“跟进中”
   - 最近跟进时间距离当前时间超过回收超时时长
4. 执行回收操作，逻辑同手动回收
5. 回收后生成跟进记录

### 7.3 对其他模块的影响

| 影响模块 | 影响内容 |
| :--- | :--- |
| **站点中心** | 公海规则配置入口位于站点中心，管理员在此配置自动回收参数 |
| **线索列表** | 自动回收后的线索从线索列表中移除（负责人清空，状态变更为“公海待分配”） |


## 八、异常状态与兜底逻辑

| 场景 | 处理方式 |
| :--- | :--- |
| 公海池无数据 | 显示“公海池暂无线索”占位图 |
| 分配时目标负责人无权限 | 下拉列表仅展示当前用户有权限管理的团队成员 |
| 批量分配部分失败 | 弹窗汇总展示成功和失败的数量，失败条目列出具体原因 |
| 数据加载失败 | 显示“数据加载失败，请刷新重试”，提供【重新加载】按钮 |
| 回收时选中线索包含已转客户 | 仅回收状态为“已分配”/“跟进中”的线索，“已转客户”线索跳过，弹窗提示“已跳过 N 条已转客户线索” |


## 九、交互说明汇总

| 交互行为 | 说明 |
| :--- | :--- |
| 点击查询 | 按当前筛选条件刷新表格 |
| 点击重置 | 所有筛选条件恢复默认值 |
| 点击刷新 | 手动刷新公海池数据 |
| 点击分配（单条） | 弹出分配线索弹窗 |
| 点击查看详情 | 跳转至线索详情页 |
| 勾选复选框 | 选中单条线索，顶部批量操作按钮状态同步更新 |
| 点击批量分配 | 弹出批量分配弹窗 |
| 点击回收 | 弹出回收确认弹窗 |
| 分配成功 | 线索移出公海池，表格刷新 |
| 回收成功 | 线索重新进入公海池，表格刷新 |


## 十、与其他模块的关系

| 关联模块 | 关系说明 |
| :--- | :--- |
| **线索列表** | 公海池是线索列表的子集（未分配线索）；分配后线索进入线索列表；回收后线索从线索列表移回公海池 |
| **线索详情** | 公海池和线索列表共用线索详情页，点击线索名称/查看详情跳转至同一详情页 |
| **线索中心** | 公海池与线索列表并列，均属于线索中心一级菜单 |
| **站点中心** | 公海规则（含自动回收配置）在站点中心配置 |
| **跟进记录** | 分配和回收操作自动生成跟进记录，记录操作轨迹 |
