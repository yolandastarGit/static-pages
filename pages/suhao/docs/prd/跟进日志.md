## 一、操作概述

**操作名称：** 跟进日志

**触发入口：** 线索列表 → 顶部【跟进日志】按钮

**页面形式：** 独立页面（数据量较大，采用分页列表）

**功能定位：** 展示当前用户有权限查看的全部线索的所有跟进记录，是销售管理者全局查看团队跟进情况的统一入口。

**面向角色：** 销售业务员（仅查看本人负责线索的跟进日志）、销售主管（查看团队全部跟进日志）


## 二、页面整体布局

跟进日志页面采用 **单页表格布局**，从上至下分为三个区域：

| 区域 | 内容 |
| :--- | :--- |
| **统计信息栏** | 展示日志总数、今日跟进数 |
| **筛选条件栏** | 多维度筛选条件，支持精细化查询 |
| **表格区** | 跟进日志数据表格，含分页 |


## 三、统计信息栏

| 统计项 | 说明 |
| :--- | :--- |
| **跟进日志总数** | 当前筛选条件下（或全部）的跟进记录总条数 |
| **今日跟进数** | 今日创建的跟进记录数量 |
| **待跟进数** | 下次跟进时间 = 今日 的跟进记录数量（仅展示统计数字，不显示具体明细） |


## 四、筛选条件栏

| 筛选项 | 类型 | 说明 | 默认值 |
| :--- | :--- | :--- | :--- |
| **时间范围** | 日期区间 | 按跟进时间筛选 | 本月 |
| **跟进人** | 下拉选择 | 可选当前用户有权限查看的团队成员；仅本人时默认选中自己 | 全部（主管）/ 本人（业务员） |
| **线索关键词** | 文本输入 | 模糊搜索线索编号或线索名称 | 空 |
| **跟进方式** | 下拉选择（多选） | 电话 / 邮件 / WhatsApp / 会议 / 拜访 / 备注 | 全部 |
| **阶段** | 下拉选择（多选） | 按跟进阶段筛选，选项取自后台配置的阶段列表 | 全部 |

**操作按钮：**

| 按钮 | 说明 |
| :--- | :--- |
| **查询** | 按当前筛选条件刷新表格数据 |
| **重置** | 将所有筛选条件恢复为默认值 |
| **导出** | 将当前筛选结果导出为 CSV 文件，导出范围为当前筛选条件下的全部数据（不限当前页），受操作权限控制 |


## 五、表格区

### 5.1 表格字段

| 字段 | 说明 | 展示规则 |
| :--- | :--- | :--- |
| **跟进时间** | 跟进记录的创建时间 | 展示完整日期时间，按倒序排列 |
| **线索编号** | 关联的线索编号 | 展示编号，点击跳转至线索详情页 |
| **线索名称** | 关联的线索名称 | 展示名称，点击跳转至线索详情页 |
| **跟进人** | 创建跟进记录的用户 | 展示用户姓名 |
| **跟进方式** | 跟进方式 | 电话 / 邮件 / WhatsApp / 会议 / 拜访 / 备注 |
| **阶段** | 跟进时记录的阶段 | 展示阶段名称 |
| **跟进内容** | 跟进记录详细内容 | 展示前 50 个字符，超出截断加“...”；鼠标悬停显示完整内容；支持点击弹出完整内容弹窗 |
| **下次跟进时间** | 计划下次跟进时间 | 展示日期，已过期的以红色高亮提示 |


### 5.2 分页

| 配置项 | 说明 |
| :--- | :--- |
| 默认每页条数 | 50 条 |
| 每页条数选项 | 20 / 50 / 100 |
| 跳转 | 支持输入页码跳转 |


## 六、交互说明

### 6.1 列表交互

| 交互行为 | 说明 |
| :--- | :--- |
| 点击线索编号 | 跳转至对应线索详情页（新窗口或当前窗口） |
| 点击线索名称 | 跳转至对应线索详情页（新窗口或当前窗口） |
| 悬停跟进内容 | 显示完整内容的气泡提示（Tooltip），内容超过 100 字符时，弹窗查看 |
| 点击跟进内容 | 弹出完整内容弹窗，展示全部跟进内容（富文本支持） |

### 6.2 跟进内容弹窗

点击跟进内容时弹出，包含：

| 元素 | 说明 |
| :--- | :--- |
| 线索名称 | 展示关联线索名称 |
| 跟进时间 | 展示完整跟进时间 |
| 跟进人 | 展示跟进人姓名 |
| 完整跟进内容 | 展示全部跟进内容（支持换行/格式） |
| 关闭按钮 | 关闭弹窗 |

### 6.3 排序规则

- 默认按 **跟进时间倒序** 排列（最新记录在前）
- 支持表格列排序：跟进时间、下次跟进时间

### 6.4 分页加载

- 切换页码时，表格数据重新加载
- 显示当前数据范围及总条数，如“第 1-50 条，共 1,280 条”


## 七、数据权限

| 角色 | 数据范围 |
| :--- | :--- |
| 销售业务员 | 仅查看本人负责线索的跟进记录（跟进人 = 本人） |
| 销售主管 | 查看本团队全部线索的跟进记录 |
| 区域负责人 | 查看本区域全部线索的跟进记录 |
| 系统管理员 | 查看系统全部线索的跟进记录 |

> 数据权限与系统权限配置保持一致（BR-049）。


## 八、业务规则

| 规则编号 | 规则说明 |
| :--- | :--- |
| BR-031 | 跟进记录仅允许新增，不修改历史，不删除 |
| BR-060 | 操作日志记录新增、编辑、删除（逻辑）、导出、登录、配置 |
| BR-061 | 登录日志记录时间、IP、浏览器、方式、结果 |


## 九、异常状态与兜底逻辑

| 场景 | 处理方式 |
| :--- | :--- |
| 无任何跟进记录 | 显示“暂无跟进记录”占位图 |
| 搜索无结果 | 显示“未找到匹配的跟进记录”，建议“请调整筛选条件后重试” |
| 数据加载失败 | 显示“数据加载失败，请刷新重试”，提供【重新加载】按钮 |
| 导出数据量过大 | 单次导出上限 10,000 条，超出时提示“数据量过大，请缩小筛选范围后重试” |


## 十、与线索详情跟进日志的区别

| 对比项 | 跟进日志（独立页面） | 线索详情-跟进日志 |
| :--- | :--- | :--- |
| 数据范围 | 全部线索的所有跟进记录 | 单条线索的跟进记录 |
| 入口 | 线索列表顶部【跟进日志】按钮 | 线索详情 → 左栏“跟进日志”标签页 |
| 用途 | 全局查看，多维度筛选 | 单条线索销售过程追溯 |
| 支持导出 | 是 | 否 |
