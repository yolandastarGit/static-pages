## 一、操作概述

**操作名称：** 跟进日志

**触发入口：** 线索详情页 → 左栏【跟进日志】标签页

**页面形式：** 线索详情页内嵌内容区

**功能定位：** 展示当前线索的全部跟进记录，用于追溯单条线索的销售跟进过程。系统不再提供线索中心下的独立跟进日志二级页面。

**面向角色：** 对当前线索有查看权限的用户


## 二、内容区布局

跟进日志作为线索详情页内部标签内容展示：

| 区域 | 内容 |
| :--- | :--- |
| **表格区** | 展示当前线索的跟进记录 |
| **空态区** | 当前线索暂无跟进记录时展示空态提示 |


## 三、表格字段

| 字段 | 说明 | 展示规则 |
| :--- | :--- | :--- |
| **跟进时间** | 跟进记录的创建时间 | 展示完整日期时间，按倒序排列 |
| **线索** | 当前线索名称 | 展示关联线索名称 |
| **跟进人** | 创建跟进记录的用户 | 展示用户姓名 |
| **方式** | 跟进方式 | 电话 / 邮件 / WhatsApp / 会议 / 拜访 / 备注 |
| **阶段** | 跟进时记录的阶段 | 展示阶段名称 |
| **内容** | 跟进记录详细内容 | 表格内展示摘要，点击详情查看完整内容 |
| **下次跟进** | 计划下次跟进时间 | 无计划时展示“-” |


## 四、交互说明

| 交互行为 | 说明 |
| :--- | :--- |
| 切换到跟进日志标签 | 在线索详情页左栏展示当前线索跟进记录 |
| 点击详情 | 以提示或弹窗方式展示完整跟进内容 |
| 录入跟进 | 通过线索详情页顶部【录入跟进】按钮新增跟进记录，新增后同步显示在本标签内容中 |


## 五、业务规则

| 规则编号 | 规则说明 |
| :--- | :--- |
| BR-031 | 跟进记录仅允许新增，不修改历史，不删除 |
| BR-060 | 操作日志记录新增跟进等关键操作 |


## 六、异常状态与兜底逻辑

| 场景 | 处理方式 |
| :--- | :--- |
| 无任何跟进记录 | 显示“暂无跟进记录”占位提示 |
| 数据加载失败 | 显示“数据加载失败，请刷新重试” |
