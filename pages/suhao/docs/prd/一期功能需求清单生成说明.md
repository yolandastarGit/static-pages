## 功能分类原则

1. **消息管理** — 合并邮件和 WhatsApp 双渠道，以甲方理解的"消息"概念统一命名
2. **AI 智能能力** — 将 PRD 中分散在各章节的 AI 能力集中呈现，突出"AI 自动"而非"人工操作"的主流程
3. **线索管理 / 公海池 / 客户管理 / 合同管理** — 按销售业务闭环的四个阶段组织
4. **工作台** — 作为日常使用入口独立模块
5. **数据分析** — 三个分析页面按业务主题命名（销售经营/获客分析/客户经营）
6. **系统管理 / 站点管理 / AI 能力管理 / 通知管理** — 后台管理类功能按管理对象分类
7. **个人中心 / 登录认证** — 用户侧基础功能

## 一期范围判断原则

- **一期核心闭环：** 客户获取（邮件/WhatsApp）→ AI 自动识别询盘 → 自动创建线索 → 线索跟进管理 → 客户沉淀 → 合同记录 → 基础数据分析
- **判断标准：** 支撑上述核心闭环运转的功能均纳入一期；增强性、配置优化类功能标注为 P1 但仍属一期范围
- **排除标准：** PRD 明确标注"延二期"的功能不纳入本清单

## 注意事项

1. CSV 中所有功能均来源于 PRD_v1.md，未自行新增功能
2. 本期合同为「合同记录管理」（方案 A），不含审批流程、OA 流程、合同签署流程
3. 角色和菜单当前版本为预置数据，不支持动态 CRUD
4. AI 为辅助能力，不可用时自动降级为规则引擎，不影响业务正常运行
5. 本清单面向甲方确认交付范围，不包含技术实现细节和内部业务规则
