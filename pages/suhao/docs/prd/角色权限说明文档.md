> **文档来源：** 基于《AI 智能 CRM 产品需求文档（PRD_v1.md）》提取整理，**不新增、不修改**权限规则；与 PRD 冲突时以 PRD 为准。  
> **适用版本：** 对齐 PRD 动态权限改版（§4.6–§4.8、§23.3/§23.4）。  
> **角色说明：** **外贸业务员即预置「业务员」角色**。另支持**自定义角色**（见下文）。

## 权限模型总原则

系统权限按「**站点权限 + 角色数据权限档位 + 角色菜单/按钮权限**」共同控制（PRD §4）。

| 权限维度 | 说明 |
| :--- | :--- |
| 站点权限 | 用户只能访问授权站点范围内的数据（数据档=全部时不受限） |
| 数据权限档位 | 角色上配置：仅本人 / 负责站点 / 授权站点（只读） / 全部（§4.2）；自定义角色创建时必选 |
| 菜单权限 | 角色勾选的菜单权限码；决定侧栏/路由可见 |
| 按钮权限 | 角色勾选的按钮权限码；决定按钮显隐与接口鉴权 |
| 消息边界 | 监督只读 + **仅账号本人可写**（§7.3、§4.8）；不得用角色名替代权限码 |

**实现铁律（PRD §4.6）：**
- 运行时一律按权限码判定（如 `hasPerm('lead:export')`）；**禁止** `role === 'sales'` / `collaborator` 控制按钮。
- 下文矩阵仅为**预置四角色默认权限包**；用户实际权限 =【角色管理】当前勾选（保存后即时生效，无需发版）。
- 完整权限码清单以 **PRD §4.7** 为唯一权威（本文不重复全表，避免双份漂移）。

## 动态权限与角色管理（MVP）

```
预置权限点（§4.7，随版本入库）
  → 角色管理：勾选菜单 + 按钮 + 选择数据档位
  → 用户管理：用户绑定唯一角色 + 授权站点
  → 登录下发：菜单码集合 / 按钮码集合 / 数据档位 / 授权站点
  → 前后端按权限码控制
```

| 项 | 口径 |
| :--- | :--- |
| 预置角色 | 系统管理员、运营专员、业务员、协同人；带 §4.3/§4.7 默认包 |
| 自定义角色 | 可新增；必选数据档位；可勾选 §4.7 普通权限点；**不可**获得 §4.8 专用能力 |
| 用户绑角色 | **单选**一个角色 |
| 系统管理员保护 | 不可删/停用/降权；**菜单/按钮勾选只读**；至少一名启用管理员用户 |
| 权限点新建 | 菜单管理 MVP **不可**新建权限点；新按钮须发版（前端 + 种子数据）后再勾选分配 |
| 「不改代码生效」 | 仅指给角色**勾选/取消已预置权限**；全新按钮能力仍须发版 |

**预置角色专用硬约束（§4.8，自定义角色不可通过勾选获得）：** WhatsApp 仅预置业务员可绑；邮箱绑定仅预置业务员/运营；站点主邮箱【转派给业务员】仅预置运营；消息代发仅账号本人；预置业务员强制单站点。

## 权限基础口径（全角色通用）

### 站点与数据权限

| 术语 | 含义 | 是否参与权限判断 |
| :--- | :--- | :--- |
| **站点**（`site_id`） | 线索/客户/合同归属站点；权限与统计统一依据（§2.4） | **是** |
| **授权站点**（`user_site_permissions`） | 用户可访问站点集合 | 限定可见上限（数据档=全部时不受限） |
| **邮箱关联站点**（`mail_site_id`） | 个人邮箱=业务员唯一授权站点；站点主邮箱=站点绑定运营邮箱 | 邮件线索站点取值；不直接作权限判断 |
| **数据权限档位** | 配置在角色上的四档之一（§4.2） | **是** |

**数据可见性 / 操作判定：**

```
数据可见 = （数据档=全部）OR（数据站点 ∈ 用户授权站点）
        + 角色数据档位过滤（本人 / 负责站点 / 授权站点只读 / 全部）
操作可执行 = 用户具备对应按钮权限码（另受 §4.8 硬约束）
```

### 操作权限类型

业务归类：新建、编辑、删除、分配、转客户、导出、导入、配置等。落地为 **PRD §4.7 按钮权限码**。

### 渠道账号绑定规则（§4.8 硬约束）

| 账号类型 | 预置业务员 | 预置运营专员 | 预置协同人 | 预置系统管理员 | 自定义角色 |
| :--- | :---: | :---: | :---: | :---: | :---: |
| 邮箱（个人，可多个） | ✅ | ✅ | ❌ | —（运维） | ❌（本期） |
| 站点主邮箱（1 站点 1 邮箱） | ❌ | ✅（站点配置指定） | ❌ | 配置 | ❌ |
| WhatsApp（1 个/人） | ✅ | ❌ | ❌ | ❌ | ❌ |

> **两类邮箱：** ① 业务员个人邮箱——线索归本人；② 站点主邮箱——线索归运营、【转派给业务员】T1–T3、不进公海。

## 二、数据权限（预置角色默认档位）

| 角色 | 默认数据档位 | 站点范围 | 可查看对象 | 限制说明 |
| :--- | :--- | :--- | :--- | :--- |
| **业务员** | 仅本人 | 授权站点（**强制单站点**） | 本人负责线索/客户/合同/跟进；本人邮箱/WA | 负责人=本人 |
| **运营专员** | 负责站点 | 授权站点=负责站点（可多） | 负责站点内业务数据；公海；业务员消息只读 | 团队管理未启用 |
| **协同人** | 授权站点（只读） | 授权站点（可多） | 授权站点内业务数据；公海只读；消息只读 | 默认不勾选写按钮；默认可导出 |
| **系统管理员** | 全部 | 全部站点 | 全公司数据与配置；消息只读 | 权限勾选只读保护 |
| **自定义角色** | 创建时四选一 | 随档位与授权站点 | 由勾选菜单/按钮决定 | 无 §4.8 专用能力 |

**适用范围：** 线索、客户、合同、跟进记录（§4.2）。  
**公海池**见第六节（`lead-pool:assign` 等）；**消息**见第五节。

## 四、按角色权限边界说明（预置角色默认语义）

### 系统管理员权限说明

## 七、站点主邮箱线索归属与转派（T1–T3）

> 站点主邮箱收到客户询盘、AI 识别为有效询盘后默认生成线索，初始负责人为该运营专员，由其手动转派给业务员（PRD §7.5.2、§27.1）。**未识别成线索的邮件也可转派**（仅投递）。业务员个人邮箱线索直接归业务员本人、无需转派。

### 线索归属

| 场景 | 系统规则 |
| :--- | :--- |
| 业务员个人邮箱收到有效询盘 | 生成线索，负责人 = 业务员本人，直接进入本人线索列表 |
| 站点主邮箱收到有效询盘 | 生成线索，初始负责人 = 站点主邮箱绑定的运营专员 |
| 运营转派（已关联线索） | 按 T1/T2 投递邮件并处理负责人 |
| 运营转派（未关联线索） | 按 T3 **仅投递邮件**；不建线索、不自动挂旧线索；业务员可后补【生成线索】 |
| 运营是否仍可查看 | 可以，运营仍可查看负责站点内该线索及其相关沟通记录 |

### 转派规则 T1–T3（运营专员转派时系统自动判断）

| 规则 | 判断条件 | 系统动作 |
| :---: | :--- | :--- |
| T1 | 当前邮件**已关联线索**，负责人是**当前运营** | 可选业务员 → 投递 + 负责人改为目标业务员；不新建线索 |
| T2 | 当前邮件**已关联线索**，负责人已是**业务员 A** | **目标锁定为 A，不允许换人** → 投递；负责人保持 A |
| T3 | 当前邮件**未关联线索** | **允许转派** → **仅投递**；不建线索、不自动关联旧线索。若同站点同发件邮箱已有业务员 A 的线索：目标**默认 A，可改选**（换人不改动旧线索）；提示不合并 |

**口径拆分：**

- **建线索：** 一封邮件一条线索（BR-021），入库不做按发件邮箱自动合并。
- **「站点 + 客户原始发件邮箱」：** 仅用于 T2 锁定同人 / T3 默认选人与提示，不是建线索去重键。
- 一封邮件最多关联一条线索；一条线索可关联多封邮件（转派投递件挂回、或人工关联等明确动作）。
- **转派仅存在于邮箱渠道，WhatsApp 无转派。**

| 运营专员无线索转派 | ✅（仅投递） | 不改动任何旧线索负责人（T3） |
| 业务员主动修改线索负责人 | ❌ | 业务员仅维护本人负责数据 |
| 协同人修改负责人 | ❌ | 协同人严格只读 |
| 系统管理员修改负责人 | ✅ | 全局权限 |

## 九、字典 / AI 与系统参数权限说明

### 字典管理配置

| 能力 | 系统管理员 | 协同人 | 运营专员 | 业务员 |
| :--- | :---: | :---: | :---: | :---: |
| 字典类型列表查看 / 字典项管理 | ✅ | ❌ | ❌ | ❌ |
| 新增字典类型 / 停用字典类型 | ❌ | ❌ | ❌ | ❌ | 类型系统预置固定（§23.5） |
| 业务可配字典项增删改启停、父子维护 | ✅ | ❌ | ❌ | ❌ |
| 受控/基础字典项改展示名与排序 | ✅ | ❌ | ❌ | ❌ | 不可增删启停项（线索/合同状态、登录方式、操作类型、录入方式、AI 业务场景） |
| 跟进阶段扩展属性（是否允许转客户、是否计入高意向） | ✅ | ❌ | ❌ | ❌ |

> **业务角色：** 仅消费字典下拉选项（筛选/表单），**不可**进入字典管理。AI 分析标签不进字典；人工线索/客户标签选项由字典维护。

### AI 能力配置

| 能力 | 系统管理员 | 协同人 | 运营专员 | 业务员 |
| :--- | :---: | :---: | :---: | :---: |
| AI 能力管理（新增/编辑/启停/删除，含业务关联） | ✅ | ❌ | ❌ | ❌ |
| 系统参数·业务规则「AI 功能启用」总开关（关闭后邮件/WhatsApp 分析、自动提取企业信息、批量 AI 提炼等全部停用）| ✅ | ❌ | ❌ | ❌ |
| 站点新增/编辑·AI 开关（原 AI 规则 Tab） | ✅ | ❌ | ❌ | ❌ | 已并入站点新增/编辑，仅开关 |

> **业务关联（§21.6，仅系统管理员可配）：** 四场景编码固定（字典 AI 业务场景，仅可改展示名/排序）。一条 AI 能力可多选绑定多个场景；每个场景同时仅一条**启用**能力占用；冲突时拦截保存/启用（不抢占）；停用立即释放占用。服务商/模型从字典选择（可先在字典增补）。调用条件：「AI 功能启用」开启 **且** 该场景存在绑定且启用的 AI 能力，且所引用服务商/模型字典项未停用，否则不调 AI（视同不可用）。

### AI 有效询盘识别（10 类判定）

AI 对邮件/WhatsApp 会话判定，命中以下任一类型即为有效询盘 → 自动生成线索（PRD §21.8）：产品咨询、价格咨询、样品需求、采购需求、交期咨询、定制需求、合作代理、项目询盘、复购/新需求、明确联系方式交换。自动回复/广告/无业务内容/垃圾信息为无效（不生成）。有效询盘识别分别路由至「邮件意向分析」「WhatsApp 意向分析」所绑定的启用能力（§21.6）。

### AI 分析结果查看

| 场景 | 权限规则 |
| :--- | :--- |
| 邮件 AI 智能分析面板 | 跟随邮件数据权限（本人可操作邮箱；运营/协同/管理员按只读查看范围） |
| WhatsApp AI 意向分析 | 跟随 WhatsApp 数据权限 |
| 线索详情·客户画像页签 | **只读**；跟随线索数据权限（§4.2）；内容引用询盘联系人侧画像（§7.9 / §11.5.3） |
| 线索列表【编辑】·客户画像 | 跟随线索【编辑】权限；写入询盘联系人侧；人工编辑后 AI 不覆盖（BR-017）；业务员不可借编辑改负责人 |
| 客户详情·AI 客户画像 | 跟随客户数据权限（§4.2）；详情只读展示 |

> **原则：** AI 分析结果查看权限**不单独扩展**，严格跟随对应业务数据的数据权限。客户画像**编辑不扩展独立权限位**，复用线索/客户编辑权限。业务角色**不可**配置字典、业务关联或 AI 服务商/模型；仅消费分析结果与下拉选项。

### AI 不可用时的业务兜底

| 角色 | 兜底行为 |
| :--- | :--- |
| 全部业务角色 | AI 失败**不阻断业务**；消息保留，可使用「生成线索（人工兜底）」手动创建线索（协同人只读除外） |
| 全部角色 | AI 面板显示灰占位/错误提示（引用 §29.7），不影响邮件查看与跟进 |
| 系统管理员 | 负责检查字典（服务商/模型/场景）、AI 能力配置（含业务关联）、系统参数「AI 功能启用」、API 端点异常处理 |

> **不可用判定（对齐 §21.5 / §21.6 / §23.5）：** 总开关关闭、当前场景无人绑定启用能力、绑定能力停用、所引服务商/模型字典项停用、Token 无效/调用失败等，均按 AI 不可用降级处理。

## 十、权限测试关注点

### 系统管理员

- [ ] 可查看全部站点线索/客户/合同数据，不受授权站点限制
- [ ] 可进入站点中心（含站点运营数据）、AI 能力管理、系统管理全部子菜单
- [ ] 可配置字典（类型列表→字典项；不可新增类型；跟进阶段扩展属性；AI 服务商→模型父子）
- [ ] 可配置 AI 能力（服务商/模型选自字典；业务关联四场景互斥、冲突拦截、停用释放）、系统参数「AI 功能启用」、通知场景、站点主邮箱（运营人员与主邮箱）
- [ ] 某场景无启用绑定能力或服务商/模型字典停用时，业务侧按 AI 不可用降级，管理员可进字典/AI 能力管理补配
- [ ] 可执行公海分配、批量转客户、变更负责人等「全部」权限操作
- [ ] **不可**代业务员回复/发送/删除邮件或 WhatsApp（查看业务员消息只读）
- [ ] 系统管理员账号不可被停用/删除

### 协同人

- [ ] 仅可查看授权站点内数据，范围外不可见
- [ ] **严格只读**：不可新增/编辑/删除/分配/转客户/变更负责人/标记异常/录入跟进
- [ ] **不可**代发/回复/转发/删除业务员邮件或 WhatsApp（仅只读查看）
- [ ] 可导出权限范围内数据
- [ ] **不可**进入系统管理、站点中心配置、AI 能力管理、通知管理配置
- [ ] 可查看销售经营/获客分析/客户经营（授权站点数据）
- [ ] 权限仅按数据站点（`site_id`）过滤，无独立「来源站点」字段

### 运营专员

- [ ] 仅可查看负责站点（=授权站点）内数据
- [ ] 可分配负责站点公海线索、新建线索/客户、批量转客户、回收至公海、变更负责人（负责站点内）
- [ ] 编辑线索时可改负责人（须变更说明），并可编辑客户画像
- [ ] 可将站点主邮箱邮件转派给业务员，验证 T1–T3（尤其 T2 锁定同人、T3 无线索仅投递且默认可改）
- [ ] **只读**查看负责站点内业务员邮箱/WhatsApp，操作按钮置灰
- [ ] **不可**查看非授权站点数据
- [ ] **不可**绑定 WhatsApp
- [ ] **不可**进入系统管理、站点中心配置、AI 能力管理

### 动态权限与自定义角色（对齐 §4.6 / §23.3）

- [ ] 角色管理可为非系统管理员角色勾选/取消菜单与按钮权限，保存后刷新权限或重新登录即生效（无需发版）
- [ ] 前端/后端按权限码鉴权，不存在仅按角色名控制的关键按钮
- [ ] 可新增自定义角色：必选数据档位 + 勾选 §4.7 权限点；无转派/绑 WA 等 §4.8 能力
- [ ] 用户仅能绑定一个角色
- [ ] 预置系统管理员：权限勾选只读；不可删除/停用；系统至少一名启用管理员
- [ ] 菜单管理不可新建权限点；新按钮须发版写入种子数据后再分配

### 业务员

- [ ] 仅可查看本人为负责人的线索/客户/合同
- [ ] 可绑定多个个人邮箱、1 个 WhatsApp；仅可操作本人绑定账号消息
- [ ] 个人邮箱有效询盘线索直接归本人（无需转派）
- [ ] 可新增/编辑本人线索（含 WhatsApp、关注点、客户画像）；**编辑中负责人只读**
- [ ] 公海池**只读**，不可分配、不可标记异常
- [ ] **不可**批量转客户、**不可变更负责人**（列表更多与编辑弹窗均不可改派）
- [ ] **不可**进入销售经营、获客分析、站点中心、AI 能力管理、系统管理
- [ ] 可查看客户经营简化版（仅本人客户）
- [ ] 授权站点仅允许单站点配置
- [ ] 数据站点（`site_id`）参与权限判断（与 PRD §2.4 一致，无「来源站点」拆分）

### 线索编辑与客户画像（跨角色，对齐 §13.3 / §11.5.3）

- [ ] 详情页客户画像只读；编辑入口仅在线索列表【编辑】
- [ ] 有编辑权限角色可改工商/参展/风险；保存后详情页签同步；人工优先 AI 不覆盖（BR-017）
- [ ] 业务员编辑不可改站点/状态/阶段/时间/负责人；运营/管理员编辑可改负责人且须变更说明
- [ ] 新增/导入/消息生成线索：邮箱/电话/WhatsApp 至少一个；含关注点字段
- [ ] 公海「待分配」不出现在线索列表
