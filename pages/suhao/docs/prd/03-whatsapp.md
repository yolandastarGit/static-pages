## 一、模块概述

**模块归属：** 沟通中心 → WhatsApp

**模块定位：** 统一管理用户绑定的 WhatsApp 账号会话，提供会话查看、消息浏览、AI 智能洞察、线索流转等核心能力。

**面向角色：** 销售业务员


## 二、页面整体布局

WhatsApp 管理页面采用 **两栏式布局**：

| 区域 | 宽度 | 内容 |
| :--- | :--- | :--- |
| **左栏（会话列表）** | 固定 360px | 展示所有 WhatsApp 会话列表 |
| **右栏（详情面板）** | 自适应（剩余宽度） | 展示当前选中会话的完整消息详情、联系信息及 AI 智能洞察 |

右栏支持折叠，折叠后详情面板收起，列表区扩展至全宽。


## 三、页面顶部导航

**功能定位：** 账号切换与核心操作入口。

| 元素 | 说明 |
| :--- | :--- |
| **账号切换器** | 下拉选择框，展示当前用户绑定的 WhatsApp 账号。MVP 仅支持一个账号，该功能为后续多账号扩展预留。 |
| **搜索框** | 按联系人名称、企业名称、消息内容搜索会话。 |
| **刷新按钮** | 手动刷新会话列表，获取最新消息。 |


## 四、左栏：会话列表

**功能定位：** 展示所有 WhatsApp 会话，快速定位目标联系人。

### 4.1 列表字段

| 字段 | 说明 | 展示规则 |
| :--- | :--- | :--- |
| **联系人名称** | 显示发件人姓名或昵称 | 若联系人已归集至客户，显示客户侧联系人姓名；若未归集，显示 WhatsApp 显示名称 |
| **最后一条消息摘要** | 展示最近一条消息的正文预览 | 截取前 30 字符，超过截断加“...” |
| **时间** | 显示最近一条消息的收发时间 | 今天显示时分（如 14:20），昨天显示“昨天”，更早显示月/日 |
| **未读角标** | 展示该会话未读消息数量 | 红色圆点+数字，位于列表项右侧；无未读时不展示 |
| **联系人企业（辅助信息）** | 显示 AI 识别的联系人所属企业名称 | 若 AI 已提取企业信息，以小字展示在联系人名称下方；未提取则不展示 |

### 4.2 列表排序规则

- 按最近一条消息的收发时间**倒序排列**（最新消息置顶）
- 未读会话不单独置顶，遵循时间倒序即可

### 4.3 列表交互

- 点击会话列表项，右侧详情面板刷新为该会话的完整消息记录
- 选中的会话列表项呈现高亮选中态
- 鼠标悬停显示轻微底色反馈

### 4.4 会话列表空态

| 场景 | 展示内容 |
| :--- | :--- |
| 无任何会话 | 显示“暂无会话”占位图，并提示“暂无 WhatsApp 会话” |
| 搜索无结果 | 显示“未找到相关会话”，建议“请尝试其他关键词” |


## 五、右栏：详情面板

**功能定位：** 展示当前会话的完整消息历史、联系人信息及 AI 分析结果。

### 5.1 联系人基础信息区

**功能定位：** 展示当前会话联系人的核心身份信息。

| 字段 | 数据来源 | 示例 |
| :--- | :--- | :--- |
| **联系人姓名** | WhatsApp 显示名称 / 联系人对象 | Sarah Jenkins |
| **手机号码** | WhatsApp 账号 | +1 (555) 0123-4567 |
| **企业名称** | AI 提取 | Global Tech Solutions Inc. |
| **地理位置** | AI 提取 | California, USA |

> **联系人生成逻辑：**
> - 首次接收到该联系人消息时，系统自动创建联系人影子对象
> - 若该联系人已归集至客户，企业名称和地理位置优先展示客户侧数据
> - 联系人尚未归集客户时，企业名称和地理位置展示 AI 提取信息，仅供销售参考

### 5.2 消息记录区

**功能定位：** 展示当前会话的全部消息历史。

**展示形式：**
- 按时间正序排列（从上到下，最早的在上方）
- 客户消息右对齐（蓝色气泡），我方消息左对齐（白色/灰色气泡）
- 每条消息展示发送时间（仅显示时分）

**消息内容支持：**
| 类型 | 展示方式 |
| :--- | :--- |
| 文本消息 | 完整展示 |
| 图片消息 | 展示缩略图，支持点击放大查看 |
| 文件消息 | 展示文件名及大小，支持点击下载（MVP 支持常见文件类型） |

**消息加载机制：**
- 默认加载最近 30 条消息
- 滚动至顶部时自动加载更早 30 条（懒加载）
- 加载历史消息时显示“正在加载更多消息...”提示

**消息记录空态：**
- 会话无任何消息时，显示“暂无消息记录”

### 5.3 AI 智能洞察区

**功能定位：** 展示 AI 对当前会话联系人的分析结果。

#### 1）AI 意向摘要

- **展示内容：** AI 对会话内容的意向总结。
- **示例：** “该联系人在过去 48 小时内高频询问产品参数，采购动力显著增强，建议优先处理回复。”

#### 2）AI 摘要标签

- **展示内容：** AI 自动生成的摘要标签。
- **示例：** 高意向、采购主管、决策链顶端。
- **展示规则：** 以标签形式水平排列。
- **说明：** 此为消息/联系人层面的 AI 摘要标签，不同步至线索或客户对象，仅作为当前会话的辅助参考信息。

### 5.4 业务操作区

**功能定位：** 提供消息流转至 CRM 核心流程的操作入口。

| 操作按钮 | 触发逻辑 |
| :--- | :--- |
| **生成线索** | 点击后，系统校验当前会话是否已关联线索。<br>• 若未关联：弹出【确认生成线索】弹窗，预填 AI 提取的联系人姓名及企业名称，用户可修改后确认。<br>• 若已关联：提示“该会话已生成线索 [线索编号]”，点击跳转至线索详情。 |
| **查看线索详情** | 仅当会话已关联线索时显示。点击跳转至对应线索详情页。 |
| **查看客户** | 仅当会话对应的联系人已归集至客户时显示。点击跳转至客户详情页。 |
| **回复** | 唤起系统默认消息发送界面，进入当前会话的回复模式。 |


## 六、异常状态与兜底逻辑

### 6.1 AI 服务不可用

- **页面表现：** AI 智能洞察区显示灰色占位符，提示“AI 服务暂时不可用。”
- **业务影响：** 【生成线索】按钮依然可用，后台仅依赖规则识别。

### 6.2 消息加载失败

- **页面表现：** 消息记录区顶部显示“消息加载失败，请刷新重试”。
- **交互：** 提供【重新加载】按钮。

### 6.3 会话列表加载失败

- **页面表现：** 会话列表区显示“会话加载失败，请刷新重试”。
- **交互：** 提供【重新加载】按钮。


## 七、交互说明汇总

| 交互行为 | 说明 |
| :--- | :--- |
| 切换会话 | 点击左侧会话列表项，右侧详情面板刷新 |
| 搜索会话 | 顶部搜索框输入关键词，会话列表实时过滤 |
| 刷新会话 | 点击刷新按钮，会话列表重新加载最新数据 |
| 查看图片消息 | 点击图片缩略图，弹出大图预览 |
| 下载文件 | 点击附件消息的文件名，触发下载 |
| 生成线索 | 点击【生成线索】按钮，弹出确认弹窗，预填 AI 信息 |
| 跳转线索/客户 | 点击【查看线索详情】/【查看客户】，跳转至对应详情页 |
| 加载更多消息 | 滚动消息记录区至顶部，自动加载更早消息 |
