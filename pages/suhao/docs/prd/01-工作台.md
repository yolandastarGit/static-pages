## 一、模块概述

**模块名称：** 工作台

**模块定位：** 工作台是登录后的每日工作入口，核心回答“我今天需要处理什么”。页面聚焦待办事项、异常提醒、当前个人业务状态和高频入口，不承载经营趋势、渠道分析、客户价值分析等深度分析任务。

**面向角色：** 全部角色。

**核心价值：**
- 快速发现需要立即处理的消息、线索和跟进事项。
- 在不进入列表页的情况下掌握本人当前业务状态。
- 将异常风险提前暴露，并提供明确跳转入口。
- 提供高频页面快捷入口，减少每日操作路径。

**职责边界：**

| 工作台负责 | 不在工作台承载 |
| :--- | :--- |
| 待回复、未读、超时、待分配等行动提醒 | 销售漏斗、成交额趋势、目标达成 |
| 本人线索、客户、高意向、待跟进客户总数 | 来源渠道、国家/地区、产品、客户价值分析 |
| 异常状态与配置风险提醒 | 业务员排行、历史趋势、深度经营分析 |
| 高频业务入口 | 独立分析看板 |

> 经营与趋势类内容统一进入【分析中心】：销售经营、获客分析、客户经营。

## 二、角色视图概览

不同角色页面结构一致，但卡片数量、数据范围和跳转后的可操作权限不同。

| 角色 | 工作台聚焦 | 数据范围 | 站点筛选 |
| :--- | :--- | :--- | :--- |
| 业务员 | 本人待办、本人线索、本人客户、沟通入口 | 仅本人负责数据 + 本人绑定邮箱/WhatsApp | 不展示 |
| 运营专员 | 负责站点待办、公海待分配、站点范围异常、业务入口 | 负责站点内业务数据 | 展示，可选负责站点 |
| 协同人 | 授权站点待办查看、异常查看、分析入口 | 授权站点只读数据 | 展示，可选授权站点 |
| 系统管理员 | 全局待办、全局异常、系统管理入口 | 全部站点 + 全部数据 | 展示，可选全部站点 |

## 三、页面布局

工作台采用卡片式运营工作区，按“先处理、再观察、后跳转”的顺序组织。

| 区域 | 内容 | 说明 |
| :--- | :--- | :--- |
| 顶部筛选栏 | 站点筛选、时间筛选、刷新、当前角色提示 | 筛选后刷新下方全部卡片 |
| 今日待办 | 待回复线索、邮件未读、跟进超时、新增线索、待分配 | 首页优先级最高 |
| 我的业务状态 | 我的线索数量、我的高意向线索数量、我的客户数量、我的待跟进客户 | 仅展示当前状态总数，不做趋势 |
| 异常提醒 | 无效/丢失线索、超期回收线索、异常站点、AI 服务不可用 | 有风险才显著提示 |
| 快捷入口 | 沟通、线索、客户、合同、分析、系统管理等入口 | 按角色菜单权限展示 |

**桌面端布局建议：**
- 筛选栏固定为单行，站点、时间、日期范围、刷新按钮从左到右排列。
- 卡片区使用 4 列等宽栅格；不足 4 个时保持同尺寸，不拉伸成大卡。
- 各区块标题保持简短：今日待办、我的业务状态、异常提醒。

**移动端/窄屏布局建议：**
- 筛选项换行堆叠，日期范围保持起止日期同一组。
- 卡片区改为 1-2 列，确保数值和标题不截断。
- 快捷入口可按两列展示，避免横向滚动。

## 四、顶部筛选栏

| 筛选项 | 展示角色 | 默认值 | 生效范围 |
| :--- | :--- | :--- | :--- |
| 站点筛选 | 运营专员、协同人、系统管理员 | 全部可见站点 | 限定工作台所有卡片的数据范围 |
| 时间快捷筛选 | 全部角色 | 当天 | 影响“新增线索”等有时间范围口径的卡片 |
| 自定义起止日期 | 全部角色 | 当天至当天 | 选择后时间快捷筛选显示“自定义” |
| 刷新 | 全部角色 | - | 重新拉取/计算当前筛选条件下的工作台数据 |

> 工作台默认时间以“当天”为准，用于突出每日处理事项；分析中心仍按各分析页自身默认时间范围执行。

**时间筛选规则：**

| 选项 | 日期范围 |
| :--- | :--- |
| 当天 | 今日 00:00 至今日 23:59 |
| 本周 | 本周一至今日 |
| 本月 | 本月 1 日至今日 |
| 自定义 | 用户选择的开始日期至结束日期 |

**站点筛选规则：**
- 业务员仅关联 1 个站点，不展示站点筛选。
- 运营专员只能选择负责站点。
- 协同人只能选择授权站点，跳转后保持只读边界。
- 系统管理员可选择全部站点。

## 五、今日待办

今日待办区用于暴露“现在需要处理”的事项。卡片主数值可点击，跳转后自动带入筛选条件。

| 卡片 | 定义 | 时间口径 | 数据范围 | 点击跳转 | 适用角色 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 待回复线索 | 线索关联消息存在至少 1 条“未回复”，且线索非终态 | 与时间筛选无关 | 本人负责线索 | 线索列表，筛选待回复 | 全部 |
| 邮件未读 | 本人绑定邮箱收件箱中未读邮件数 | 与时间筛选无关 | 本人邮箱 | 邮件，筛选未读 | 全部 |
| 跟进超时 | 下次跟进时间早于今日且线索非终态 | 与时间筛选无关 | 权限范围内线索 | 线索列表，筛选下次跟进时间小于今日 | 全部 |
| 新增线索 | 创建时间在所选时间范围内的线索数 | 受时间筛选影响 | 本人负责线索 | 线索列表，筛选创建时间范围 | 全部 |
| 待分配 | 状态为“待分配”的线索数 | 与时间筛选无关 | 权限范围内公海/待分配线索 | 公海池 | 运营专员、协同人、系统管理员 |

**待回复与未读区分：**

| 维度 | 适用渠道 | 粒度 | 工作台卡片 |
| :--- | :--- | :--- | :--- |
| 已读/未读 | 仅邮件 | 单封邮件 | 邮件未读 |
| 已回复/未回复 | 邮件 + WhatsApp | 单封邮件/单个会话 | 待回复线索 |

> “已读”不等于“已回复”。待回复线索与下次跟进时间无关；跟进超时与回复状态无关。

## 六、我的业务状态

本区只展示当前用户本人业务状态总数，不展示环比、趋势、排行和漏斗。

| 卡片 | 定义 | 点击跳转 | 适用角色 |
| :--- | :--- | :--- | :--- |
| 我的线索数量 | 本人为负责人、非终态的线索总数 | 线索列表 | 全部 |
| 我的高意向线索数量 | 本人为负责人，当前跟进阶段为打样阶段、报价阶段或谈判阶段的线索数 | 线索列表，筛选高意向阶段 | 全部 |
| 我的客户数量 | 本人为负责人的客户总数 | 客户列表 | 全部 |
| 我的待跟进客户 | 本人名下存在“待跟进”或“跟进中”非终态线索的客户数 | 客户列表，筛选待跟进客户 | 全部 |

**展示要求：**
- 主数值使用大号数字，辅助文案只说明统计范围。
- 不展示成交金额、线索成交率、有效线索率、客户活跃趋势等分析指标。
- 点击卡片后跳转列表页，列表页继续按用户权限收敛数据。

## 七、异常提醒

异常提醒用于暴露需要关注但不一定立刻处理的风险。无权限访问目标页时，卡片可展示但跳转后仍按菜单和按钮权限控制。

| 卡片 | 定义 | 点击跳转 | 适用角色 | MVP 状态 |
| :--- | :--- | :--- | :--- | :--- |
| 无效/丢失线索 | 权限范围内状态为“无效”或“丢失”的线索数 | 线索列表，筛选无效/丢失 | 全部 | 保留 |
| 超期回收线索 | 权限范围内入池原因为“超期回收”的线索数 | 公海池 | 运营专员、协同人、系统管理员 | 二期/延后 |
| 异常站点 | 权限范围内状态非“启用”的站点数 | 站点管理 | 系统管理员可跳转；运营专员/协同人仅提示 | 保留提示 |
| AI 服务不可用 | AI 总开关关闭，或无可用 AI 服务商时展示提示 | AI 能力管理 | 全部可见提示；系统管理员可跳转 | 保留 |

**AI 服务不可用展示：**
- 不作为普通计数指标，主数值可显示“--”。
- 文案提示“请检查 AI 配置”。
- 非系统管理员看到提示但不可进入 AI 能力管理。

## 九、交互说明

| 交互行为 | 说明 |
| :--- | :--- |
| 切换时间 | 更新时间范围，并刷新受时间影响的卡片 |
| 切换站点 | 刷新全部卡片，所有数据限定在所选站点内 |
| 点击待办卡片 | 跳转对应列表或沟通页，并带入筛选参数 |
| 点击业务状态卡片 | 跳转对应业务列表，并带入本人/阶段/状态筛选 |
| 点击异常卡片 | 有目标页权限则跳转；无权限则只展示提示或拦截提示 |
| 点击快捷入口 | 进入对应模块首页 |
| 刷新 | 保留当前筛选条件，重新加载工作台数据 |

**建议跳转参数：**

| 卡片 | 目标页筛选参数 |
| :--- | :--- |
| 待回复线索 | `reply=pending` |
| 邮件未读 | `unread=1` |
| 跟进超时 | `overdue=1` 或 `nextFollowUpTimeEnd=昨日` |
| 新增线索 | `createdStart=开始日期&createdEnd=结束日期` |
| 待分配 | 公海池默认待分配视图 |
| 我的高意向线索数量 | `stage=highIntent` |
| 无效/丢失线索 | `statusGroup=invalidLost` |

## 十、空态、加载与异常

| 场景 | 处理方式 |
| :--- | :--- |
| 单卡无数据 | 主数值显示 `--`，辅助文案保留统计口径 |
| 单卡加载失败 | 该卡主数值显示 `--`，不影响其他卡片 |
| 页面整体加载中 | 卡片区域显示骨架屏 |
| 页面整体加载失败 | 显示“数据加载失败，请刷新重试”并提供刷新 |
| 无待办事项 | 今日待办卡片显示 `0` 或 `--`，不隐藏卡片 |
| 无异常提醒 | 异常卡片可显示 `0`；AI 服务可用时不展示 AI 异常提示 |
| 无目标页权限 | 保持工作台可见，跳转时提示“您没有该操作权限” |

## 十一、研发验收要点

| 验收项 | 标准 |
| :--- | :--- |
| 页面边界 | 工作台不出现销售漏斗、成交额趋势、有效线索率、线索成交率、客户活跃趋势 |
| 数据权限 | 各卡片数据必须按 §4.2 权限收敛，协同人只读 |
| 站点筛选 | 业务员不展示；多站点角色只展示可访问站点 |
| 时间默认 | 默认当天，切换后新增线索等时间口径卡片同步刷新 |
| 跳转筛选 | 点击卡片后目标页带入对应筛选，不扩大权限范围 |
| 异常独立 | 单个卡片失败不影响其他卡片展示 |
| 响应式 | 桌面 4 列，窄屏 1-2 列，文本不溢出 |
