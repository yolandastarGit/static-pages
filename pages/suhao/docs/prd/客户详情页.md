## 一、模块概述

**模块归属：** 客户中心 → 客户列表 → 点击客户名称进入

**模块定位：** 展示单条客户的完整信息，是企业客户资产的核心查看与操作页面。

**面向角色：** 销售业务员、销售主管


## 二、页面整体布局

客户详情页采用 **上下布局 + 标签页切换**：

| 区域 | 内容 |
| :--- | :--- |
| **顶部信息区** | 客户名称、客户编号、关联线索数、来源站点数、负责人姓名 |
| **标签页导航** | 客户信息 / 线索记录 / 联系人 / 合作合同 / AI客户画像 |


## 三、顶部信息区

**功能定位：** 展示客户核心摘要信息。

| 字段 | 说明 | 展示规则 |
| :--- | :--- | :--- |
| **客户名称** | 客户企业名称 | 大号字体突出显示 |
| **客户编号** | 系统自动生成，格式：CUS-YYYY-XXXX（CUS + 年份 + 4 位流水号） | 以标签形式展示在名称右侧 |
| **关联线索数** | 该客户下关联的线索总数 | 展示格式：N 条线索 |
| **来源站点数** | 该客户涉及的来源站点数量 | 展示格式：N 个站点 |
| **负责人** | 客户当前负责人 | 展示负责人姓名，格式：负责人 XXX |


## 四、标签页一：客户信息

**功能定位：** 展示客户的基础信息与业务属性。

### 4.1 展示字段

| 字段 | 说明 | 数据来源 |
| :--- | :--- | :--- |
| **公司名称** | 客户企业名称 | 客户核心属性 |
| **国家/地区** | 客户所在国家 | 客户核心属性 |
| **行业** | 客户所属行业 | 客户核心属性 |
| **官网** | 客户企业官网 | 客户扩展属性 |
| **联系方式** | 询盘人的电话 + 邮箱 | 取线索来源消息中询盘人的联系方式（发件人邮箱/电话），合并展示。无询盘人信息时显示"-" |
| **负责人** | 客户当前负责人 | 客户核心属性 |
| **创建时间** | 客户创建时间 | 系统记录 |

### 4.2 客户标签

**功能定位：** 展示该客户下的所有标签。

- 客户标签与 AI 标签分离：客户标签仅包含业务员手动添加的标签，AI 标签仅展示在线索详情页
- 多个标签水平排列
- 支持在客户信息标签页内直接管理标签（增删）
- 客户标签可删除

### 4.3 与基线的对应关系

| 截图字段 | 基线属性 |
| :--- | :--- |
| 公司名称 | 企业名称 |
| 国家/地区 | 国家 |
| 行业 | 行业 |
| 负责人 | 当前负责人 |
| 创建时间 | 创建时间 |
| 客户标签 | 客户标签 |


## 五、标签页二：线索记录

**功能定位：** 展示该客户关联的全部线索列表。

### 5.1 统计信息

| 统计项 | 说明 |
| :--- | :--- |
| 关联线索数 | 该客户下线索总数 |
| 来源站点数 | 该客户涉及的来源站点数量 |

### 5.2 筛选条件

| 筛选项 | 类型 | 说明 | 默认值 |
| :--- | :--- | :--- | :--- |
| **来源站点** | 下拉选择 | 按线索所属站点筛选 | 全部 |
| **处理状态** | 下拉选择 | 按线索状态筛选（状态值取自 7.3 定义） | 全部 |

### 5.3 表格字段

| 字段 | 说明 | 展示规则 |
| :--- | :--- | :--- |
| **线索编号** | 线索唯一编号 | 点击跳转至线索详情页 |
| **来源站点** | 线索所属站点 | — |
| **来源渠道** | 线索来源渠道 | 官网询盘 / WhatsApp / 展会 / 客户转介绍等 |
| **询盘时间** | 线索创建时间 | 展示完整日期时间 |
| **产品需求** | 线索意向产品 | 展示 AI 识别的产品类目 |
| **处理状态** | 线索当前状态 | 状态值：公海待分配 / 已分配 / 跟进中 / 高意向 / 已成交 / 无效 / 丢失 |
| **负责人** | 线索负责人 | 展示负责人姓名 |

### 5.4 交互规则

- 默认按询盘时间倒序排列
- 点击线索编号跳转至线索详情页
- 线索记录列表为空时，显示“暂无关联线索”


## 六、标签页三：联系人

**功能定位：** 管理客户下的全部联系人，支持新增、编辑、删除。

### 6.1 顶部操作

| 按钮 | 说明 |
| :--- | :--- |
| **新增联系人** | 弹出新增联系人弹窗 |

### 6.2 表格字段

| 字段 | 说明 | 展示规则 |
| :--- | :--- | :--- |
| **姓名** | 联系人姓名 | 点击可编辑；删除后展示为"-" |
| **职位** | 联系人职位 | — |
| **邮箱** | 联系人邮箱 | — |
| **电话** | 联系人电话 | — |
| **WhatsApp** | 联系人 WhatsApp | — |
| **联系角色** | 联系人在采购中的角色 | 决策人 / 采购经理 / 执行联系人 / 关键联系人 / 其他 |
| **主要联系人** | 是否为主要联系人 | 勾选标识（✅ / ⭐），仅可设置一位 |
| **AI识别** | 是否由系统从线索自动归集 | 带 AI 标记，仅展示不可编辑 |

### 6.3 新增/编辑联系人弹窗

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 姓名 | 文本输入 | 是 | 联系人姓名 |
| 职位 | 文本输入 | 否 | 联系人职位 |
| 邮箱 | 文本输入 | 否 | 联系人邮箱 |
| 电话 | 文本输入 | 否 | 联系人电话 |
| WhatsApp | 文本输入 | 否 | 联系人 WhatsApp |
| 联系角色 | 下拉选择 | 否 | 决策人 / 采购经理 / 执行联系人 / 关键联系人 / 其他 |
| 主要联系人 | 开关/复选框 | 否 | 标识是否为主要联系人 |

### 6.4 交互规则

- 新增联系人：点击【新增联系人】弹出弹窗，填写信息后提交
- 编辑联系人：点击联系人行编辑按钮，弹窗预填当前信息
- 删除联系人：二次确认后执行逻辑删除。删除后，历史关联数据中该联系人字段展示为"-"占位符
- AI识别标记为只读，不可编辑
- 主要联系人仅可设置一位；设置新主要联系人时，原主要联系人自动取消
- 支持手动选择联系人作为客户信息中默认展示的联系人（非主要联系人概念，仅用于详情页顶部"联系方式"的展示来源选择）


## 七、标签页四：合作合同

**功能定位：** 管理客户下的全部合同。

### 7.1 统计信息

| 统计项 | 说明 |
| :--- | :--- |
| 累计合同数 | 该客户下合同总数 |
| 累计合同金额 | 所有合同金额累计（按币种分别统计） |
| 最近签约时间 | 最近一份合同的签约日期 |
| 最近签约合同 | 最近一份合同的编号，点击跳转 |

### 4.2 顶部操作

| 按钮 | 说明 |
| :--- | :--- |
| **新增合同** | 弹出新增合同弹窗，自动关联当前客户（复用合同中心弹窗） |
| **AI 画像** | 弹出 AI 画像弹窗，只读展示从线索同步的企业信息（公司简介/主营业务/企业规模/目标市场/风险信息） |

### 4.3 表格字段

| 字段 | 说明 |
| :--- | :--- |
| **合同编号** | 合同编号，点击跳转合同详情 |
| **合同名称** | 合同名称 |
| **合同金额** | 含币种符号 |
| **签约时间** | 合同签约日期 |
| **合同状态** | 已签约 / 执行中 / 已完成 / 已终止 / 已作废 |
| **操作** | 【详情】跳转合同详情 |

### 7.4 交互规则

- 默认按签约时间倒序排列
- 点击新增合同复用合同中心弹窗，自动关联当前客户
- 点击合同编号/详情跳转合同详情页
- 合同列表为空时，显示“暂无合作合同”


## 八、标签页五：AI客户画像

**功能定位：** 展示从线索同步至客户的企业信息。

### 8.1 数据说明

- **数据来源：** 线索转客户时同步的 AI 企业信息（BR-036）
- **同步时机：** 线索转客户时同步一次，后续不自动更新
- **同步内容：** 企业信息（公司简介、主营业务、所属行业、企业规模、目标市场）及风险信息
- **展示形式：** 只读展示，不可编辑

### 8.2 展示内容

| 区域 | 字段 | 说明 |
| :--- | :--- | :--- |
| **企业画像** | 公司简介 | 企业背景描述 |
| 企业画像 | 主营业务 | 主要经营业务 |
| 企业画像 | 所属行业 | 所属行业 |
| 企业画像 | 企业规模 | 员工规模（如中型企业·200-500人） |
| 企业画像 | 目标市场 | 目标市场区域 |
| **风险信息** | 风险提醒 | AI 识别的企业风险信息（如有） |


## 九、异常状态与兜底逻辑

| 场景 | 处理方式 |
| :--- | :--- |
| 客户不存在 | 显示“客户不存在或已被删除”，提供返回列表按钮 |
| 数据加载失败 | 显示“数据加载失败，请刷新重试”，提供【重新加载】按钮 |
| 各标签页无数据 | 显示对应空态提示：“暂无关联线索”/“暂无联系人”/“暂无合作合同”/“暂无AI客户画像” |
| 新增联系人必填校验 | 姓名不可为空，提交时校验，提示“请输入联系人姓名” |
| 联系人已被删除 | 关联数据展示为"-"，不阻塞页面加载 |

## 十一、与其他模块的关系

| 关联模块 | 关系说明 |

| **跟进记录** | 新增/编辑/删除联系人、新增合同等操作自动生成跟进记录 |
| **AI能力** | AI客户画像数据来源为线索转客户时同步的AI企业信息 |


## 十二、客户标签管理

### 12.1 数据一致性

客户详情页中的标签管理与 [客户列表](file:///Users/yolandaw-%E2%80%8E/Documents/项目-苏豪 0.2/prd/06-客户列表.md) 的标签管理是**同一份数据**：

- **客户列表**：行操作列【打标】→ 标签管理弹窗
- **客户详情页**：客户信息 → 标签管理

两处实时双向同步，无任何时延。

### 12.2 标签类型

| 标签类型 | 来源 | 可编辑性 |
| :--- | :--- | :--- |
| AI 标签 | 系统根据客户信息、行为、邮件往来自动生成 | 仅系统可改（人工只读） |
| 手动标签 | 销售业务员或主管手动添加 | 手动可增/删/改 |
