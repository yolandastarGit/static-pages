## 2. 产品整体说明

### 2.1 产品定位

面向销售团队的客户关系管理平台，围绕「消息接入（邮件/WhatsApp） → 线索识别 → 跟进转化 → 客户沉淀 → 合同成交 → 经营分析」闭环，为业务员、运营专员、协同人、系统管理员提供分层协作能力。AI 作为平台能力嵌入各业务场景，不可用时系统按规则正常运行。

### 2.2 核心目标

1. 邮件与 WhatsApp 双渠道**消息**统一接入，实现多渠道集中管理，避免线索遗漏。
2. 线索全生命周期管理（识别、归属、跟进、转客户、回收）。
3. 客户档案与联系人持续沉淀，合同录入与成交追溯。
4. 灵活的业务规则配置与权限管理
5. 销售经营、获客、客户经营三维经营分析。

### 2.3 业务边界

- **覆盖：** 消息沟通（邮件/WhatsApp）、线索管理、客户/联系人/合同管理、经营分析、站点管理、系统管理。

### 2.4 对象关系

> **消息定义：** 消息指系统从沟通渠道接入的客户沟通内容，包含邮件消息和 WhatsApp 消息。消息用于沟通记录展示、AI 分析、线索识别、回复状态判断和客户互动沉淀。通知、站内信、钉钉推送不属于消息，统一归入通知体系。

**核心业务对象定义（术语唯一权威）：**

| 对象 | 定义 | 关键特点 |
| :--- | :--- | :--- |
| 询盘联系人 | 来自消息渠道（邮件、WhatsApp 等）主动发起沟通的人 | 是线索来源；**不属于客户资产**；可未识别所属企业 |
| 所属企业 | 系统根据询盘联系人信息识别出的企业或个体工商户名称 | 属**线索阶段信息**（字段名：所属企业/企业名称）；**不是独立业务对象**；**不等同于客户名称**；有值时可在转客户时作为客户名称来源 |
| 线索 | 销售机会对象 | 包含：询盘联系人、所属企业、询盘内容、AI 分析结果、跟进记录；**无线索名称字段，展示用线索编号** |
| 客户 | 已确认进入 CRM 客户管理体系的业务主体 | 客户资产的归属主体；归属一个站点；**MVP 客户唯一性 =（站点 + 客户名称）**；客户名称按统一规则生成（企业/个体工商户取企业名称，个人取询盘联系人姓名）；可拥有多个历史线索（客户 1:N 线索），线索转客户后保留历史关联 |
| 客户联系人 | 客户主体下维护的联系人 | 来源：线索转客户/关联时，**询盘联系人转换/归集为客户联系人**；**必属于且仅属于一个客户**（禁止独立联系人、禁止跨客户共享） |

> **客户名称口径：** 客户列表中的「客户名称」用于展示客户主体名称。若客户主体为企业或个体工商户，客户名称取企业名称；若客户主体为个人，客户名称取询盘联系人姓名。MVP **不强制**设置客户主体类型字段，系统按可识别到的企业名称或询盘联系人姓名生成客户名称；若无法识别客户名称，转客户时需人工补充。客户名称是客户档案的主要展示字段，**不等同于**线索阶段「所属企业 / 企业名称」。MVP **不设独立企业（Company）对象**，无企业管理菜单；跨站点同名多档案由「站点 + 客户名称」唯一性解释。

**对象层级（唯一权威）：
   ** 询盘联系人 → 线索（Lead）→ 关联客户（Customer）→ 客户联系人（Contact）；
   客户资产侧为 客户（Customer）→ 客户联系人（Contact）。

```
客户：**站点 + 客户名称 唯一确定一个客户**（同客户名称跨站点可多档案）
站点 1:N 消息（邮件/WhatsApp） / 线索 / 客户 / 客户联系人
邮箱账号两类：业务员个人邮箱（归业务员本人）、站点主邮箱（站点绑定运营人员、以其业务邮箱作主邮箱，一站点一主邮箱，见 §20.5、§24.3.1）；WhatsApp 账号仅业务员绑定、不绑固定站点（业务员恒单站点，见 §24.3.2）
线索 1:1 询盘联系人；线索含所属企业字段（线索阶段信息，非独立对象）
线索 N:1 客户（一线索至多关联一个客户）；线索 N:1 站点（线索创建即有明确站点，见 §27.1）；线索 1:N 消息（邮件/WhatsApp） / 跟进记录 / 合同
客户 1:N 客户联系人 / 合同 / 线索
询盘联系人 --转客户/关联线索归集--> 客户联系人
合同 N:1 客户 / 线索（合同不设独立站点，继承客户站点）
```

**站点术语（唯一权威，全系统只引用不重复定义）：** 站点是 CRM 中**业务运营与数据隔离的基础单位**（用于用户业务归属、业务数据归属、消息渠道配置、AI 能力配置、业务规则配置），**不是组织架构，不扩展部门/区域/公司组织树**。

| 术语 | 字段 | 归属对象 | 含义与用途 |
| :--- | :--- | :--- | :--- |
| 站点 | `site_id` | 线索 / 客户 / 合同 | 业务数据归属站点；**数据权限判断、业务管理、获客/经营分析的统一站点依据** |
| 授权站点 | `user_site_permissions` | 用户 | 用户拥有访问权限的站点范围；属**权限模型**，非业务字段 |
| 邮箱关联站点 | `mail_site_id` | 邮箱账号 | 邮箱对应站点：业务员个人邮箱=业务员唯一授权站点；站点主邮箱=站点配置中为站点绑定运营人员并选定其业务邮箱所对应的站点（一站点一主邮箱）；邮件线索站点取此站点（见 §20.5、§24.3.1、§27.1） |

> **关键口径：** ① 权限与统计均基于**站点**（`site_id`）；② 邮件线索：站点 = 收件邮箱对应站点；WhatsApp 线索：站点 = 业务员唯一授权站点；**线索创建即有明确站点（不存在站点待补充）**；③ 变更负责人/公海分配均不改站点；④ 合同继承客户站点；⑤ 邮箱对应站点 / WhatsApp 线索站点**不受页面站点筛选器影响**。

## 4. 角色与权限体系

> 本章为全系统**权限体系唯一权威定义**。系统权限按「**站点权限 + 数据负责人 + 角色操作权限**」共同控制。不同角色的数据范围**来源不同**（业务员=本人负责数据；运营专员=负责站点；协同人=授权站点范围**只读**；系统管理员=全部）。业务模块的数据权限统一引用 §4.2，仅保留模块特殊权限（如公海池分配）。
>
> **权限模型总原则：**
>
> | 权限维度 | 说明 |
> | :--- | :--- |
> | 站点权限 | 用户只能访问自己授权站点范围内的数据 |
> | 负责人权限 | 业务员仅管理本人负责的数据 |
> | 角色权限 | 不同角色拥有不同的数据查看、编辑、分配、管理能力 |
> | 消息查看权限 | 运营专员、协同人、系统管理员可查看授权范围内业务员的**客户沟通消息**（含邮件消息与 WhatsApp 消息），但**仅限只读**（见 §7.3 消息查看与操作边界） |

### 4.1 角色定义

| 角色 | 职责 |
| :--- | :--- |
| 业务员 | 一线销售执行人员，**仅关联 1 个站点**；负责本人线索/客户/合同的跟进与转化；可对本人负责数据增删改查、录入跟进、推进销售流程；可绑定**多个邮箱**与 **1 个 WhatsApp**，处理本人绑定账号下的消息（查看/回复/转发/删除）；不管理其他业务员及其他站点数据 |
| 运营专员 | 站点业务运营管理角色，可关联**多个站点**；管理其**负责站点**范围内的业务数据（分配线索、调整负责人、协助推进客户与合同、站点经营数据）；其绑定邮箱可作为**站点主邮箱**对外接收询盘；可**只读**查看负责站点内业务员的客户沟通消息（含邮件消息与 WhatsApp 消息；不可代发/回复/转发/删除）；可将站点主邮箱产生的线索**转派**给业务员 |
| 协同人 | **只读协作角色**，授权站点范围内**仅查看**线索/客户/合同/跟进记录，以及授权范围内业务员的客户沟通消息（含邮件消息与 WhatsApp 消息，只读）；**不允许**新增、编辑、删除、分配、转客户、修改状态、录入跟进或代发消息；默认可导出（见 §4.4）；禁止系统级配置与全局权限配置 |
| 系统管理员 | 全公司系统与数据管控（全部站点 + 全部数据 + 全部消息），负责站点/用户/角色/字典/AI/集成配置/系统参数/系统日志；查看业务员客户沟通消息仅限只读，不可代业务员发送或回复 |

### 4.2 数据权限粒度

数据权限由**角色、管理关系、站点归属、团队（未来可选）共同确定**；**当前版本团队管理未启用，以授权站点作为主要管理范围**，未来启用团队后运营专员改按所属团队计算，其余角色行为不变。适用范围：线索/客户/合同/跟进记录。

| 粒度 | 角色 | 判定规则（当前版本，团队未启用） | 未来启用团队 |
| :--- | :--- | :--- | :--- |
| 仅本人 | 业务员 | 本人为负责人，且数据站点在授权站点内 | 不受团队影响，仍为本人数据 |
| 负责站点 | 运营专员 | 负责站点（=授权站点）内所有业务员数据（含本人） | 所属团队内业务员数据 |
| 授权站点（只读） | 协同人 | 授权站点范围内所有运营专员/业务员数据，**仅可查看/导出，不可写** | 授权站点范围内数据（只读，保持不变） |
| 全部 | 系统管理员 | 全公司所有站点数据，不受授权站点限制 | 全部站点 + 全部团队 + 全部数据 |

- **站点定义：** 业务运营与数据隔离基础单位，用于邮件/线索/客户/合同的数据归属（术语权威见 §2.4）。
- **权限唯一站点依据：** 数据可见性 = 数据「站点」(`site_id`) ∈ 用户「授权站点」(`user_site_permissions`) + 角色/团队粒度。
- **团队定义：** 团队是**销售管理单元**，仅用于销售数据管理范围扩展；**不是**部门/区域/公司组织树等组织架构。团队为可选权限维度，当前版本仅预留字段，不提供团队管理菜单/新增页面/维护流程/配置后台。（属于二期二期）
- **角色定义：** 决定用户可执行的操作（查看/新增/编辑/分配/导出/配置），见 §4.4。
- **团队为空兜底规则：** 用户未配置所属团队时，系统按授权站点计算数据权限。二期功能

## 5. 登录与认证

### 5.1 模块概述

- **定位：** 统一身份认证入口，登录成功后加载菜单/页面/操作/数据权限并进入对应工作台。

### 5.2 页面结构


| 区域 | 内容 |
| :--- | :--- |
| 品牌区 | Logo + "AI 智能 CRM" |
| 登录方式 Tab | 账号密码登录 \| 钉钉扫码登录 |
| 表单区 | 账号输入框 + 密码输入框 + 记住账号复选框 |
| 操作区 | 【登录】按钮 + "忘记密码"链接（提示"请联系系统管理员重置密码"，不自助重置） |
| 底部 | 版权信息 |

### 5.3 登录方式与字段

#### 5.3.1 账号密码登录（默认）

| 字段 | 类型 | 必填 | 校验 / 错误提示 |
| :--- | :--- | :--- | :--- |
| 登录账号 | 文本 | 是 | 非空，"请输入登录账号" |
| 登录密码 | 密码（可显隐） | 是 | 非空，"请输入登录密码" |
| 记住账号 | 复选框 | 否 | 选中后下次自动填充账号（本地存储） |

#### 5.3.2 钉钉扫码登录

无需账号密码，依赖集成配置（钉钉应用配置）钉钉凭证发起 OAuth。

### 5.4 业务规则

- 登录流程：校验账号存在 → 校验密码 → 校验账号状态 → 生成 Token(JWT) → 初始化权限 → 记录登录日志 → 进入工作台。
- **【二期/延后】登录失败锁定：** MVP 不做失败锁定，登录失败仅按 §5.5 统一提示"用户名或密码错误"。二期方案：同一账号连续登录失败达「登录失败锁定次数」（默认 5 次，见 §23.6）时锁定账号，锁定期为「账号锁定时长」（默认 30 分钟）；锁定期内禁止登录，成功登录后失败计数清零；锁定期满自动解锁，系统管理员重置密码亦可解锁；参数为 0 时不启用锁定。
- 钉钉扫码：已绑定→自动登录；未绑定→提示"当前钉钉员工未绑定系统用户，请联系系统管理员"；账号停用→禁止登录。
- 登录成功初始化：用户信息（ID/姓名/角色/站点）+ 菜单/页面/操作/数据四类权限 + 按角色跳转工作台。
- 退出登录：清 Token/Session → 销毁前端状态 → 重定向登录页；无二次确认，有未保存编辑时轻提示。
- 会话：登录创建 Session + 返回 JWT，每次请求携带 Token 鉴权；Session 仅用于日志与超时。
- 登录超时：默认 1440 分钟（系统参数可配），超时弹"您的会话已超时，请重新登录"并跳登录页。
- 同时登录：允许多设备同时登录（**【二期/延后】单端登录限制**见 §23.6.1）。
- 密码策略：初始密码由系统管理员分配（默认 123456，首次登录不强制改）；长度 ≥ 8 位可配；字符类型可配。
- 登录日志：记录时间、账号、IP、方式、结果（含失败原因）、浏览器、操作系统（BR-055）。

### 5.5 登录失败提示

| 场景 | 提示 |
| :--- | :--- |
| 用户不存在 / 密码错误 | 用户名或密码错误（统一提示，防账号枚举） |
| 用户停用 | 当前账号已停用，请联系系统管理员 |
| 钉钉未绑定 | 当前钉钉员工未绑定系统用户，请联系系统管理员 |
| 网络异常 | 网络连接异常，请稍后重试 |

### 5.6 MVP 范围

- **支持：** 账号密码登录、钉钉扫码登录、登录日志、用户改密、系统管理员重置密码、登录超时退出、记住账号。

#### 7.5.2 操作按钮

| 操作 | 前置条件 | 结果 |
| :--- | :--- | :--- |
| 生成线索（人工兜底）| 邮件未关联线索（AI 未自动生成 / AI 不可用 / 无法识别时使用，非主流程）| 弹【确认生成线索】弹窗；**字段集与新增线索一致**（见 §13.2 / §13.2.1）；AI 成功有值则预填可改，**AI 不可用/失败则业务字段全部不预填**；站点/来源/负责人等**默认带入但可改**；确认后按表单所选归属；已关联则提示线索编号并跳转 |

| 回复 | — | 打开写邮件页签，规则见 §7.6.11 |
| 转发 | — | 打开写邮件页签，规则见 §7.6.12；**纯邮件协作，不变更线索负责人，不触发转派逻辑**（与【转派给业务员】独立） |
| 删除 | — | 移至垃圾箱，已生成线索不同步删除 |

**转派机制（用户无感知，BR-053）：**

- **用户侧：** 运营仅点击【转派给业务员】并选择业务员；**不出现**写邮件/转发页签，**不要求**运营二次手动转发；页面**不展示**「转发」「Fwd:」等投递相关标签。
- **系统侧：** 确认后自动完成邮件投递（底层转发至目标业务员绑定邮箱）+ 按 T1–T3 处理负责人；投递件内部标记为转派来源，**不触发 AI 再建线索**；若当前邮件**已关联线索**则投递件关联该线索（BR-021 例外，见 §7.10）；若**未关联线索**则投递件亦不建线索、不自动挂旧线索。
- **业务员侧：** 在本人绑定邮箱收件箱收到该邮件，与普通来件展示一致；可通过线索列表/详情中的**负责人**确认归属；**无线索的转派件**可在邮件上使用【生成线索（人工兜底）】成为有效线索。

**站点主邮箱转派规则 T1–T3（运营专员点击转派时系统自动判断）：**

| 规则 | 判断条件 | 系统动作 |
| :---: | :--- | :--- |
| T1 | 当前邮件**已关联线索**，且线索负责人为**当前运营** | 弹窗可选本站点业务员 → 投递邮件至目标业务员绑定邮箱 + 将该线索负责人变更为目标业务员；**不新建线索** |
| T2 | 当前邮件**已关联线索**，且线索负责人已是**业务员 A** | 转派目标**锁定为 A，不允许换成其他人** → 投递邮件至 A 绑定邮箱；负责人保持 A；**不新建线索**。弹窗展示说明：「该邮件已关联线索，负责人为 [A]，仅可转派给本人」 |
| T3 | 当前邮件**未关联线索** | **允许转派**（不因无线索拦截）→ **仅投递邮件**；**不新建线索**；**不按发件邮箱自动关联已有旧线索**。业务员收到后可自行【生成线索】。若按「站点 + 客户原始发件邮箱」匹配到已有线索且负责人为业务员 A：目标业务员**默认选中 A，允许运营改选其他人**（本邮件尚未挂线索，换人不改动旧线索负责人）；并提示「该发件人在本站点已有线索，负责人 [A]（仅提示，不自动合并）」 |

> **口径拆分（唯一权威，消除与「一邮件一线索」的歧义）：**
> - **建线索：** 有效询盘仍按 BR-021 **一封邮件一条线索**；**入库不做**按发件邮箱自动合并/去重。同发件人多封有效询盘 = 多条线索（可并存）。
> - **转派：** 管「邮件投递给谁」+「若本邮件已挂线索则如何处理负责人」；**不**用匹配键把未挂线索的邮件并进旧线索。
> - **「站点 + 客户原始发件邮箱」：** 仅用于转派时的**提示 / 默认选人（T3）/ 有线索时锁定同人（T2）**，**不是**建线索去重键，**不是**自动并线索条件。
> - **不使用**收件邮箱、运营人员作为「同人」匹配依据；客户名称仅作参考展示，不自动合并。
> - 一封邮件最多关联一条线索；一条线索可关联多封邮件（仅限：转派投递件挂回原线索、或人工关联线索等明确动作，对齐 BR-021、BR-022）。
> - 转派仅存在于邮箱渠道，WhatsApp 无转派。

##### 7.6.7 发件人规则

> **总原则：** 发件邮箱与「当前查看邮箱上下文」对齐，避免回复/转发时换通道导致客户回信落到另一邮箱、再触发新入站线索链。**空白写邮件**允许在本人可用邮箱中选择

| 场景 | 规则 |
| :--- | :--- |
| **回复 / 转发** | 发件人 = **当前正在查看的该邮件所属邮箱**（即来信所在邮箱 / 列表当前邮箱上下文）；**锁定不可切换**（只读展示） |
| **空白写邮件**（点【写邮件】，非回复/转发） | **不锁定**；可在底部发件人区域于**本人可用发件邮箱**中切换；默认优先当前列表已选邮箱，否则取默认/首个可用邮箱 |
| 当前用户仅绑定一个邮箱 | 空白写邮件时默认该邮箱，不展示切换或仅只读展示 |
| 当前用户无可用发件邮箱 | 禁用发送按钮，提示“暂无可用发件邮箱，请先绑定邮箱” |
| 发件邮箱异常 | 禁用发送按钮，提示“当前发件邮箱不可用，请检查邮箱配置” |
| 运营站点主邮箱 | 在**站点主邮箱上下文**下回复/转发时，发件人锁定为主邮箱；空白写邮件时，若用户对该主邮箱有发送权限，可将其选为发件人 |
| 查看他人邮箱 | 不允许使用他人邮箱发送，见 §7.3；只读查看时不提供回复/转发代发 |

> **与线索关系：** 回复/转发锁定发件邮箱，降低「换邮箱发出 → 客户回信进另一邮箱 → 新入站再建线索」的割裂风险。空白写信可选邮箱，仍须注意选错通道可能带来后续多线索；不因此自动合并线索（BR-021）。

### 7.7 附件规则

### 7.8 批量 AI 提炼

- 触发：列表多选 → 【批量 AI 提炼】；仅同一邮箱、单次≤50 封；已读/未读均可、同封可重复提炼（指可重复生成 PDF，**不等于**消息详情「重新分析」；MVP **不提供** AI 智能分析面板的重新分析入口）。
- 弹窗只读展示：已选邮件数、主题、发件人、发件时间、影响范围（"分析 N 封，预计 X 秒"）。
- 流程：确认 → Loading（不可重复提交）→ 生成 PDF 自动下载至本地（系统内不单独展示结果）；失败 Toast"AI 提炼失败，请重试"。
- **与客户画像：** 批量提炼过程中，若询盘联系人**尚无**客户画像则按 §7.9 生成并落在询盘联系人资料；若**已有**客户画像则跳过生成、使用存量（不覆盖）。意向产品/询盘标签仅在该消息有效询盘并生成线索时按 §7.9 / §25 写入线索。

### 7.9 AI 智能分析面板（右栏）

当系统**收到**邮件或 WhatsApp 消息时，对**当前消息自动执行一次** AI 分析，结果持久化后在右侧「AI 智能分析」面板展示。**打开消息详情仅展示已有分析结果，不因打开而重复调用 AI。** MVP **不提供**「重新分析」入口（批量 AI 提炼见 §7.8，与本面板重新分析无关）。

消息发起人统一称为**询盘联系人**。

AI 仅作为辅助能力。AI 未启用、调用失败、只生成部分结果、无法识别所属企业或客户画像部分为空时，**不阻断**消息查看、人工/自动生成线索、线索跟进、关联客户或转客户（BR-015）。

#### 分析内容（一级四项）

| 一级模块 | 说明 | 数据流转 |
| :--- | :--- | :--- |
| 意向总结 | 根据当前消息总结询盘联系人需求、采购意向与关注点（建议 100–200 字） | **仅属当前消息的 AI 分析结果**（随消息分析结果保存并展示）；**不写入线索字段，不写入客户** |
| 意向产品 | 识别当前消息中的意向产品 | 有效询盘并**生成线索时**写入线索「意向产品」字段；**不写入客户** |
| 客户画像 | **统一模块**，内含二级：① 企业工商信息 ② 过往参展信息 ③ 企业风险提示。三者**不是**相互独立的一级 AI 模块 | 分析结果落在**询盘联系人资料**（可关联所属企业信息）；线索详情**引用展示**（不另建可被消息反复覆盖的线索画像副本）。生成/复用规则见下 |
| 询盘标签 | 根据当前消息生成的标签（产品/需求/行业/采购阶段等；即原「AI 分析标签」） | 有效询盘并**生成线索时**按 §25 / BR-016a **追加**写入线索标签（同名去重；人工删过的同名禁止再追加）；**不写入客户** |

**客户画像 · 企业工商信息交互：** 识别所属企业工商资料；若按「当前站点 + 客户名称 / 邮箱域名」匹配到已有客户，可展示【查看已有客户档案】；**仅提示，不阻止生成线索、不创建重复客户**。

#### 客户画像字段
**① 企业工商信息**
企业名称、官网/企业域名、 linkedin公司主页，所属行业、成立年份、国家/地区、员工人数区间、年收入或收入区间

**② 过往参展信息**
参展次数、最近参展时间、参展年份、展会名称、活跃度/趋势

**③ 企业风险提示**
根据当前公司网上的风险信息


#### 客户画像生成与复用（询盘联系人侧，BR-016c）

1. 客户画像在**消息 AI 分析阶段**生成（不是等到转客户才首次生成）。
2. **首次**：该询盘联系人尚无客户画像时，AI 生成并保存到询盘联系人资料。
3. **存量优先**：只要该询盘联系人**过往已有**客户画像，后续消息分析**跳过生成、只用存量**，新结果**不覆盖**。
4. **人工优先**：业务人员已编辑过的客户画像，以人工为准，AI **不覆盖**（BR-017）。
5. 无法识别所属企业时，客户画像允许部分为空，不阻止生成线索或转客户。

#### 生成线索时同步

- 意向产品 → 线索「意向产品」字段。
- 询盘标签 → 追加写入线索标签（规则见 §25）。
- 客户画像 → 线索详情通过询盘联系人资料**关联展示**（带入可见的客户画像）；不把意向总结写入线索。

#### 转客户 / 关联已有客户时同步（BR-036）

- 询盘联系人归集为客户联系人（BR-035）。
- **客户侧尚无客户画像记录** → 将询盘联系人侧客户画像写入客户「客户画像」。
- **客户侧已有客户画像记录（只要有过即算）** → **整包不碰、绝不覆盖**（不做补空）。
- **国家/地区、所属行业（客户标准字段）预填：** 取自客户画像·企业工商信息中的「国家/地区」「所属行业」；写入前与字典（客户域→国家/地区、行业）对齐，**匹配不上则留空**（不写自由文本、不强制「其他」）。
  - **新建客户：** 预填上述两字段（无值则空）；随后可在客户列表【编辑】修改。
  - **关联已有客户：** 仅当客户该字段为空时预填；**已有值不覆盖**。
- 意向总结、意向产品、线索标签 / 询盘标签 → **不写入客户**。

权威字段同步总表见 §21.4；标签细则见 §25。

### 7.10 业务规则

- **AI 自动识别并生成线索（主流程，BR-019a/BR-020）：** 收到邮件 → AI 自动判定是否为有效询盘（标准见 §21.8）→ 有效则自动提取字段并**自动创建线索**（无需人工点击），无效不生成；每封邮件独立生成线索（一消息一线索 BR-021，同封多需求合并 BR-023）。
- **询盘联系人影子对象（BR-012）：** 首次收到发件人邮件时按姓名/邮箱自动创建询盘联系人影子对象（尚未归集为客户联系人），已有则关联不重复；归客户后可联动展示客户名；归集规则见 27.3。
- **生成线索归属（BR-008，两类邮箱）：** ① **业务员个人邮箱**：站点=该邮箱对应站点（业务员唯一授权站点），负责人=**业务员本人**，直接进入本人线索列表；② **站点主邮箱（运营人员业务邮箱）**：站点=该站点主邮箱对应站点（`mail_site_id`，见 §20.5），负责人=**该站点绑定的运营人员**（归属异常兜底见 §27.1），由运营人员点击【转派给业务员】一步完成投递与负责人变更、不进公海（转派 T1–T3、BR-053 见 §7.5.2）；**人工兜底生成归属当前操作人**；关闭 AI 时可人工生成线索。站点未绑定运营人员/主邮箱时其邮件不参与自动生成线索。
- **转派投递（BR-053）：** 站点主邮箱【转派给业务员】确认后，系统自动投递邮件至目标业务员绑定邮箱，并按 T1–T3 处理负责人（**未关联线索时仅投递、不建线索、不自动挂旧线索**）；已关联线索的投递件**不触发 AI 建线索**并关联原线索；页面不向用户展示「转发」相关标签。
- **消息回复状态（BR-019b，邮件实现）：** 按单条邮件消息维护，默认"未回复"；该邮件首次回复发送成功后仅当前邮件置"已回复"，不影响同一线索下其他未回复消息；用于工作台待回复判定（§27.6）。

### 7.11 异常处理

- **AI 能力异常：** 引用《全局异常处理规范》§29.7；AI 不可用时**不自动生成线索**，邮件消息保留，转由**人工兜底创建线索**；【生成线索】弹窗按 §13.2.1：**业务字段全部不预填**（含发件邮箱也不自动带入），由用户手填；站点/来源/负责人**默认带入但可改**，不阻断业务（见 §21.5、§21.8）。
- **加载/网络异常：** 引用 §29.2、§29.3（邮件加载失败提示"邮件加载失败，请刷新重试" + 【重新加载】）。
- **空态（模块特有）：** 收件箱"暂无邮件"、已发送"暂无已发送邮件"、草稿箱"暂无草稿"、垃圾箱"垃圾箱暂无邮件"、搜索"未找到相关邮件"。

## 8. 沟通中心 · WhatsApp

### 8.1 模块概述

- **定位：** 统一管理绑定的 WhatsApp 会话，提供会话查看回复、AI 智能分析面板；**线索由 AI 自动识别有效询盘后自动生成（主流程），人工生成为异常兜底**（见 §21.8、§27.1）。
- **面向角色与数据权限：** WhatsApp 账号**仅业务员可绑定**（1 个/人，见 §24.3.2）；**业务员**可查看并操作（回复/发送/删除）本人 WhatsApp 会话；**运营专员（负责站点）/协同人（授权站点）/系统管理员（全部）可只读查看**授权范围内业务员的 WhatsApp 会话，**不可代发/回复/删除**（消息查看与操作边界见 §7.3）。

### 8.2 页面结构

三栏布局：左栏联系人列表（选中高亮）、中栏会话正文（未选中显引导）、右栏详情面板（可折叠）。顶部：搜索框（按联系人名/企业名/消息内容；**输入防抖 300ms 后自动查询、无需按 Enter，清空搜索框恢复完整列表**）、最近消息时间范围筛选、刷新按钮。

#### 8.2.1 最近消息时间范围筛选

| 字段 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| 最近消息时间 | 日期区间（开始/结束） | 不限制 | 按会话最近一条消息时间筛选；允许只填开始或只填结束；开始不晚于结束；空值表示不限制该时间条件 |

- **业务含义：** 按会话**最近一条消息时间**筛选会话列表（不区分消息方向）。
- **叠加规则：** 与搜索框叠加生效；时间筛选只影响当前会话列表查询，不改变 WhatsApp 线索归属规则、不改变权限范围。
- **重置规则：** 点击重置时恢复默认时间范围（不限制）。

### 8.3 联系人列表字段

| 字段 | 展示规则 |
| :--- | :--- |
| 头像 | 缩略图 |
| 联系人名称 | WhatsApp 名称 |
| 消息摘要 | 最近一条前 30 字符，超长"..." |
| 时间 | 今天时分、昨天"昨天"、更早月/日、跨年 YYYY/MM/DD |

空态：无会话"暂无 WhatsApp 会话"；搜索无结果"未找到相关会话"。

### 8.4 会话正文（中栏）

- 按时间正序；客户消息右对齐（蓝气泡），我方左对齐；
- 消息类型：文本（完整）、图片（缩略图可预览）、文件（名+大小，可下载）。
- 输入区：多行输入（Enter 发送、Shift+Enter 换行）、发送按钮、上传图片（jpg/png）、上传文件（pdf/docx/xlsx/xls/pptx/ppt/zip/rar）。
- 加载：默认最近 30 条，滚动到顶懒加载更早 30 条；默认定位最新；无消息"暂无消息记录"。
- 中栏空态：未选联系人显"请从左侧选择一个会话开始沟通"。

### 8.5 详情面板（右栏）

#### 8.5.1 询盘联系人基础信息

姓名、手机号、企业名称（所属企业，AI 提取）、地理位置（AI 提取）。已归客户时企业名/地理位置优先展示客户侧数据。

#### 8.5.2 站点规则

WhatsApp 账号**不绑定固定站点**，仅绑定业务员（见 §24.3.2、§8.1）。因**业务员恒为单一授权站点**，WhatsApp 线索站点必定明确、不存在"站点待补充"。

AI 识别有效询盘后自动生成正式线索，站点规则如下：

| 场景 | 站点 | 进入位置 |
| :--- | :--- | :--- |
| 业务员（单一授权站点） | 业务员唯一授权站点 | 线索列表 |

**自动生成线索归属该 WhatsApp 账号绑定且启用的业务员**（归属异常兜底见 §27.1）；**人工兜底生成归属当前业务员**。WhatsApp 线索站点不受页面站点筛选器影响。

> **关键口径：** WhatsApp 线索负责人 = WhatsApp 绑定业务员，创建即"待跟进"、**不进公海池**、站点明确。

#### 8.5.3 AI 智能分析面板

与邮件侧 §7.9 **同一套口径**（一级四项：意向总结、意向产品、客户画像、询盘标签；客户画像含企业工商信息 / 过往参展信息 / 企业风险提示）。收到 WhatsApp 消息时分析一次并持久化；打开会话仅展示已有结果；MVP 不提供重新分析入口。客户画像落询盘联系人资料、首次生成后复用存量、人工编辑优先（BR-016c、BR-017）；询盘标签写入线索见 §25 / BR-016a。字段同步总表见 §21.4。

#### 8.5.4 业务操作

| 操作 | 前置条件 | 结果 |
| :--- | :--- | :--- |
| 生成线索（人工兜底）| 会话未关联线索（AI 未自动生成 / AI 不可用 / 无法识别时使用，非主流程）| 弹【确认生成线索】弹窗；**字段集与新增线索一致**（见 §13.2 / §13.2.1）；已关联则提示线索编号并跳转 |
| 查看线索详情 | 会话已关联线索 | **新开页签**跳转线索详情（§11） |

### 10.7 公海回收规则

#### 10.7.1 回收来源

> **MVP 仅保留「运营专员手动回收」**；`【二期/延后】` 自动回收（系统参数天数 + 站点回收规则 + 定时任务）延后。

| 优先级 | 来源 | 说明 |
| :--- | :--- | :--- |
| 1 | 系统参数「未跟进自动回收天数」**【二期/延后】** | 二期：系统级，对所有站点强制生效，站点配置不可覆盖 |
| 2 | 站点配置「公海规则」**【二期/延后】** | 二期：仅系统参数未配置时生效 |
| 3 | 运营专员手动回收（**MVP**） | 兜底，随时可执行；MVP 唯一回收来源 |

#### 10.7.2 回收状态范围与固定规则

- **手动回收状态范围（MVP）：** 除"已成交"外全部可手动回收（待跟进/跟进中/已转客户/无效/丢失）；仅"已成交"排除（成交已闭环，不再回收公海）。
- **【二期/延后】自动回收状态范围：** 仅"待跟进""跟进中"（自动回收为"超时未跟进"驱动，仅对活跃跟进状态有意义）；已转客户、已成交、无效、丢失不自动回收。
- **系统固定规则：** 超时计算基准=最近跟进时间（自动回收口径，二期）；回收后不允许再次分配给同一负责人。
- **回收动作（负责人清空）：** 线索进入公海执行 `owner = null`（清空负责人），**保留历史跟进记录与历史操作记录**；重新分配后由分配动作重新生成负责人（见 §10.6.3）。
- **【二期/延后】自动回收执行：** 定时任务每日一次（如凌晨 2:00）；仅扫描启用站点（停用站点自动回收暂停）；线索状态待跟进/跟进中且最近跟进时间超阈值；回收后生成跟进记录（备注"系统自动回收至公海"）。

### 10.8 异常处理

引用 §29.1、§29.3、§29.5（并发冲突）。模块特有：无数据"公海池暂无线索"；分配相关校验与失败提示（目标负责人停用、并发已分配、批量部分失败、站点无销售人员）见 10.6.3。

## 11. 线索详情页

### 11.1 模块概述

- **定位：** 展示单条线索完整信息，跟进/状态流转核心操作页；线索列表与公海池共用。
- **打开方式：** **新开页签**（工作区多页签）；不占侧边菜单。从列表、公海、消息「查看线索」等入口跳转时新开或激活已有同线索详情页签（同线索重复打开复用同一页签）。

### 11.2 页面结构

顶部基础信息区 + 操作按钮区 + 标签页内容区（跟进日志 / 关联合同 / 客户画像）。

### 11.3 基础信息字段

线索编号（只读）、询盘联系人、邮箱（无则"-"）、WhatsApp（无则"-"）、电话（无则"-"）、意向产品（无则"-"）、标签（逗号分隔）、关注点（逗号分隔，无则"-"）、备注（无则"-"）、状态阶段（格式"状态·阶段"，如"跟进中·谈判阶段"，无阶段"-"）、负责人（姓名+头像）、站点（数据归属与权限依据，只读）、来源渠道、创建时间、最近跟进（无则"-"）、下次跟进（无则"-"）。

### 11.4 操作按钮区

详情顶部**仅保留 3 个按钮**（顺序固定）：**录入跟进**、**查看关联消息**、**转客户**。不提供编辑、打标签、标记异常、变更负责人、回收至公海等入口（上述操作仅在线索列表 / 公海池，见 §13.1.1）。行为见第 13 章。**协同人仅只读查看详情（默认可导出），录入跟进 / 转客户不展示或置灰；查看关联消息可按消息只读权限进入。**

### 11.5 标签页内容

#### 11.5.1 跟进日志

时间轴/表格展示本线索全生命周期记录（创建、状态变更、跟进记录），默认按跟进时间倒序，支持列排序（跟进时间、下次跟进时间）与分页（20 条/页）；记录不可改不可删（BR-031）；点击【录入跟进】弹窗；新增记录同步全局跟进日志。**展示字段与 §13.4 录入跟进对齐**（跟进时间、跟进人、跟进方式、跟进内容、客户关注、当前阶段、下次跟进时间、跟进附件；附件可下载/预览）。全局跟进日志页字段补齐**本期暂缓**，仍按 §12.5。

#### 11.5.2 关联合同

合同编号（跳合同详情）、合同名称、金额（¥）、状态（已签约/失效）、签约时间；空显"暂无关联合同"。

#### 11.5.3 客户画像

### 11.6 异常处理

引用 §29.3。模块特有：线索不存在时"线索不存在或已被删除" + 返回列表。

## 12. 线索中心 · 跟进日志

### 12.3 页面结构

页签标题 + 筛选条件栏 + 表格区。

**【二期/延后】统计信息栏：** 跟进日志总数、今日跟进数、今日计划跟进数（下次跟进时间=今日，仅数字）。MVP 不展示。

### 12.4 筛选条件

| 字段 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| 跟进时间 | 日期区间 | 本月 | 按**实际录入跟进的时间**（`createdAt`）筛选；`followUpTimeStart`/`followUpTimeEnd`；允许只填开始或只填结束；空值不限制 |

| 客户 | 文本 | 空 | MVP：按关联客户名称模糊筛选 |

| 跟进方式 | 下拉 | 全部（取值见 §23.5 线索域→跟进方式） | — |
| 跟进状态（阶段） | 下拉 | 全部 | MVP 单选；**【二期/延后】** 多选 |
| 对象类型 | 下拉 | — | **【二期/延后】** 线索/客户/合同 |
| 对象关键词 | 文本 | — | **【二期/延后】** |

> **跟进时间与下次跟进时间不能混用**：跟进时间为实际跟进发生时间，下次跟进时间为计划时间；两字段独立筛选、各自携带开始/结束参数。重置时恢复默认（跟进时间=本月，下次跟进时间=不限制）。

### 12.5 表格字段

**MVP：** 跟进时间（倒序）、跟进人、关联线索编号（跳详情）、关联客户、跟进方式、跟进内容、当前跟进状态、下次跟进时间。分页 20 条/页，20/50/100。

> **与线索详情跟进日志差异：** 详情内跟进日志字段对齐 §13.4（含客户关注、附件），见 §11.5.1；**全局跟进日志页补齐客户关注/附件本期暂缓**。

### 12.6 业务规则

跟进记录仅新增，不修改历史、不删除（BR-031）；每次关键沟通或状态变化必须新增跟进记录（BR-030）。

### 12.7 异常处理

引用 §29.1、§29.3。模块特有：无记录"暂无跟进记录"、搜索无结果"未找到匹配的跟进记录"；导出上限 10000 条。

- **客户画像：** 编辑写入**询盘联系人侧**画像（线索侧引用展示，不另存可被消息覆盖的副本）；人工编辑后以人工为准，后续 AI **不覆盖**（BR-017、BR-016c）。无存量画像时可在编辑中首次补录。
- **负责人：** 可在编辑中修改；与列表「更多 → 变更负责人」**两入口并存**，共用权限与可选人范围。若负责人发生变化，**须填写变更说明**，并生成跟进记录（备注文案同变更负责人：`变更负责人：[说明]`）。
- **关注点：** 编辑可增可删；与录入跟进的「追加去重」规则并存（见 §13.4）。
- **校验：** 询盘联系人非空；邮箱/电话/WhatsApp 至少一个；站点不在本表单修改故不作站点空校验。
- **成功后：** 更新信息（所属企业变更不自动重匹配客户）+ 提示"线索信息已更新" + 对变更字段生成跟进记录（备注"[字段]由[原值]变更为[新值]"，仅记变更字段；负责人变更按上条专属文案；客户画像变更记"更新了客户画像"）。**手动修改字段后 AI 不再覆盖（BR-017）。**
- **线索状态：** 不在编辑中修改；仅通过录入跟进 / 转客户 / 合同联动 / 标记异常 / 公海回收与分配等动作变更。

### 13.5 打标签

> **入口：** 线索列表操作列 / 顶部批量；**详情页不提供打标签入口**。

- 单条弹窗：线索编号、当前标签（带删除）、添加标签（输入回车新增）、热门标签（当前站点 Top10，点击添加，见 25.2）。
- 批量弹窗：已选线索数、添加标签、热门标签；**仅批量新增，不支持删除**。
- 交互：中英文/数字/下划线，≤20 字；同一线索不可重复（"该标签已存在"）；单条删除悬停×二次确认。手动标签选项由字典（线索域→线索手动标签）维护，新输入同步写回字典。AI 来源、人工来源标签均可增删改。
- **AI 与人工规则：** 线索标签含 AI（询盘标签）与人工两类来源；追加/去重/禁止恢复已删同名标签等规则以 §25.2 为准（BR-016a、BR-017）。

### 13.6 转客户

- **定位：** 业务人员**人工主动**将线索沉淀为客户资产的业务动作（线索 → 客户 + 客户联系人 + 保留原线索），帮助业务人员快速完成"线索 → 客户资产"转换；与客户中心「变更负责人」完全隔离，不复用逻辑、不共用名称。
- **转客户规则（准入）：** 是否允许触发由**跟进阶段扩展属性「是否允许转客户」**（即转客户规则）决定（见 §23.5）；**仅"允许触发"，绝不自动转化**，最终是否转化由人工主动执行。**跟进阶段取值与"是否允许转客户"默认标记以字典 §23.5.4 为准（本模块不重复定义固定枚举）**。线索状态准入=待跟进/跟进中/无效/丢失（"已成交""已转客户""待分配"不支持；公海池"待分配"仅分配/标记异常）。单条：负责人本人/运营专员/系统管理员；批量：运营专员/系统管理员。**协同人只读，不可转客户。**
- **执行流程（点击【转客户】后）：**
  1. **生成客户名称并创建/关联客户：** 客户站点=线索站点。**客户名称生成规则：** ① 若线索已识别到企业名称或个体工商户名称（所属企业/企业名称有值），则客户名称取该企业名称；② 若未识别到企业名称，仅存在个人询盘联系人，则客户名称取询盘联系人姓名；③ 若企业名称与询盘联系人姓名**均为空**，则**不允许转客户**，需用户人工补充客户名称后重试。按生成后的**客户名称**在当前线索站点内**精确匹配**已有客户（命中则关联已有客户；未命中则创建新客户；历史多条时关联最近创建的一条，不弹窗；BR-034）。MVP 不强制区分企业 / 个体工商户 / 个人，不设置客户类型字段，不提供筛选或统计（见 §30.6）。
  2. **创建客户联系人：** 将询盘联系人转换/归集为该客户下的客户联系人（归集去重见 BR-035、§27.3）；若客户名称来源于询盘联系人姓名，该联系人仍作为客户联系人保存，不额外创建独立个人客户对象。
  3. **关联原线索：** 保留原线索并挂接到客户（客户 ├ 客户联系人 └ 历史线索）。
  4. **修改线索状态：** 跟进中 → 已转客户（**不再直接置"已成交"**；"已成交"由后续录入已签约合同并关联线索驱动，见 §26.1、BR-056）。
  5. **同步客户画像等 AI 信息**至客户（见 §7.9、§21.4、BR-036）：客户侧**尚无**客户画像记录时，将询盘联系人侧客户画像写入客户；客户侧**已有**客户画像记录时整包不碰、绝不覆盖；同时按 BR-036 将工商信息中的**国家/地区、所属行业**预填到客户标准字段（新建必预填、关联仅补空、字典匹配不上留空，可人工改）；意向总结/意向产品/线索标签不同步；无企业信息时工商信息可显"-" → 系统通知负责人。
- **系统记录（非跟进记录）：** **不强制填写跟进记录**；系统自动记录**客户转化事件**（示例："线索已转化为客户"｜操作人｜时间｜生成客户），属**系统操作记录**（写入操作日志 §23.9），不计入销售跟进记录。
- **结果：** 生成/关联客户 + 客户联系人 + 保留并关联原线索 + 线索状态→"已转客户"（状态流转见 §26.1）。客户名称校验通过后转客户为必然成功动作；客户名称为空时拦截并提示补充；其余异常按系统异常处理（BR-033）。
- **线索生命周期（方案 B）：** 转客户后**线索继续存在并作为业务跟进对象**，可查看历史信息、新增跟进记录、继续维护业务沟通；**系统不自动删除或关闭线索**（不存在"转客户后线索生命周期结束"）。
- **转客户后规则边界：** 允许——查看历史信息 / 查看 AI 分析结果 / 新增跟进记录 / 继续记录客户沟通；禁止——**状态回退（已转客户→跟进中/待跟进）** / **返回正常跟进准入链路** / **再次转客户（已转客户不再作为转化准入状态）**。（注：已转客户仍可经公海回收机制进入待分配，属异常回收，见 §10.7、§26.1，非跟进链路回退。）

### 13.7 标记异常

- **入口：** 线索列表更多、公海池操作列；**详情页不提供**。终态线索锁定，入口置灰。
- **字段：** 当前状态（只读）、异常类型（无效/丢失）、异常备注（必填）。
- **结果：** 状态变更为对应终态 + 生成跟进记录（备注"标记异常：[类型]-[备注]"）；公海池中标记后仍保留列表展示，不再支持分配。线索不物理删除（BR-037）。

### 13.8 变更负责人

- **定位：** 单条线索负责人 A→B；仅变更本条，不影响该客户其他历史线索及后续新线索归属。与公海分配不同（分配针对无负责人公海线索）。**站点主邮箱【转派给业务员】在投递邮件的同时按 T1–T3 处理负责人**（一步完成，BR-053；见 §7.5.2）；未关联线索也可转派（仅投递）；转派不进公海。
- **入口：** 仅线索列表更多；**详情页不提供**。
- **字段：** 线索编号（只读）、客户名称（只读）、分配状态（只读）、站点（只读）、当前负责人（只读）、新负责人（下拉必填，该线索站点销售人员，停用不展示）、变更备注（必填）。
- **校验：** 新负责人非空"请选择新负责人"；不可与当前相同"新负责人与当前负责人相同，无需变更"；备注非空。并发已变更"线索负责人已被其他操作变更，请刷新后重试"。
- **结果：** 更新负责人 + 状态不变 + 生成跟进记录（备注"由[原负责人]变更至[新负责人]"）+ 提示"线索负责人已变更为[新负责人]"（BR-025/BR-028）。

**负责人变更规则：**

| 场景 | 是否允许 | 说明 |
| :--- | :---: | :--- |
| 运营专员将自己负责的站点主邮箱线索转派给业务员 | 允许 | 负责人由运营变更为目标业务员 |
| 运营专员调整负责站点内线索负责人 | 允许 | 仅限运营负责站点范围内 |
| 运营专员转派**已关联线索**且负责人为业务员 A 的邮件 | 目标锁定为 A | 不允许换成其他人（转派 T2）；无线索转派不改动旧线索负责人（T3） |
| 业务员主动修改线索负责人 | 不允许 | 业务员仅维护本人负责数据 |
| 协同人修改负责人 | 不允许 | 协同人严格只读 |
| 系统管理员修改负责人 | 允许 | 全局权限 |

### 13.9 查看关联消息

> 线索可关联全部消息（含邮件与 WhatsApp）。入口文案统一为「**查看关联消息**」（详情顶部三按钮之一；列表更多同入口）。按线索已关联渠道跳转沟通中心对应页。

- **跳转规则：** 仅关联邮件 → 沟通中心·邮件（按线索关联消息过滤）；仅关联 WhatsApp → 沟通中心·WhatsApp（定位会话）；两者皆有 → **优先按来源渠道**（邮件→邮件页，WhatsApp→WhatsApp 页），同页内仍可再切另一渠道；皆无 → Toast「该线索暂无关联消息」。
- **关联邮件消息：** 顶部显"线索：[编号]-[名称]的邮件往来"，提供返回；无关联邮件显"该线索暂无关联邮件"；记录不存在提示"关联消息记录不存在或已被清理"（BR-022 一线索可关联多消息）。
- **关联 WhatsApp 消息：** 按会话 ID 定位并刷新详情；无会话显"该线索暂无关联 WhatsApp 会话"；不存在提示"关联的 WhatsApp 会话记录不存在或已被清理"。

| 行业 | 下拉 | 否 | 字典 §23.5（表单项文案可用「所属行业」）；转客户时可从工商画像预填（BR-036），支持【编辑】修改 |
| 客户潜质分级 | 下拉 | 否 | 取值字典 §23.5；未选默认"可跟进"；**仅人工选择，系统不自动赋值变更** |
| 站点 | 下拉 | 是 | 数据归属与权限依据 |
| 关联已有线索 | 下拉搜索 | 否 | 该站点下未关联客户、且状态≠无效/丢失的线索，防同名拆分 |
| 客户联系人姓名/职位/邮箱/电话 | 文本 | 否 | 写入客户联系人 |

提交：校验客户名称/站点非空；**MVP 客户唯一性 =（站点 + 客户名称）**；同一站点内客户名称相同视为同一客户，校验"该客户已存在，请勿重复创建"；不同站点即使客户名称相同也允许各自建档；客户名称按统一规则生成（企业/个体工商户取企业名称，个人取询盘联系人姓名）；精确匹配、不做模糊匹配；创建后当前业务员成为负责人；关联线索按 14.8（一）规则；生成跟进记录（备注"新建客户"）。

#### 14.7.2 编辑客户资料

客户名称、站点只读；国家/行业/**客户潜质分级**可编辑（潜质分级取值字典、仅人工改）；标签只读（增删由列表【打标签】弹窗）。**本弹窗不维护客户联系人**（新增见 §14.7.2a）。编辑生成跟进记录（备注"更新了客户信息：[字段]由[旧值]变更为[新值]"）。客户本身不可删除（BR-043）。

#### 14.7.2a 添加联系人

- **入口：** 客户列表操作「更多」→【添加联系人】（**不在客户详情页提供**）。
- **权限：** 与【编辑】一致——业务员（本人负责客户）、运营专员 / 系统管理员（权限范围内）；**协同人不可**。
- **字段：** 姓名（必填）、职位、邮箱、电话、WhatsApp、联系角色（决策人/采购经理/执行联系人/关键联系人/其他）、主要联系人（勾选；同客户仅一位，设新则原自动取消）。
- **规则：** 邮箱/手机去重仅同客户内提示不阻止；保存成功后 toast「联系人已新增」，**留在客户列表，不跳转详情**。

#### 14.7.3 变更负责人

- **定位：** 已有客户 A→B；后续该客户新生成线索自动分配新负责人，历史线索不变。与转客户完全隔离。
- **字段：** 客户名称（只读）、当前负责人（只读）、新负责人（下拉必填，该客户站点启用销售人员）、变更备注（必填）。
- **校验：** 新负责人启用（停用阻止"该负责人已不可用，请重新选择"）、不可与当前相同。
- **结果：** 更新负责人 + 后续新线索归新负责人 + 历史线索不变 + 生成跟进记录（备注"客户[名称]由[原负责人]变更为[新负责人]"）。批量：逐条执行，部分成功弹窗汇总。

#### 14.7.4 客户打标签

客户标签**仅人工标记、独立于线索标签**（规则见 §25.3、BR-016b），选项由字典（客户域→客户标签）维护。字段：客户名称（只读）、当前标签（带×移除）、选择标签（下拉多选，已添加置灰）。提交对比新增/移除，生成跟进记录（备注"更新了客户标签：新增[...]，移除[...]"）；字典无选项提示"暂无可选标签，请联系系统管理员在字典管理中维护"。

#### 14.7.5 客户导出

- **入口：** 客户列表顶部【导出】。
- **字段：** 导出范围（当前选中[默认]/当前筛选结果/当前页）、文件格式（只读 CSV）、导出上限（"最多 10000 条"）。
- **规则：** 导出内容为当前权限范围内客户列表字段；超 10000 条提示"数据量过大，请缩小筛选范围"；成功下载本地，失败 Toast"导出失败，请重试"。
- **权限：** 业务员（本人范围）、运营专员（负责站点）、协同人（授权站点只读，默认可导出）、系统管理员（全部）均可导出；记录操作日志（操作类型=导出，见 §23.9、BR-054）。

#### 14.7.6 客户导入

- **入口：** 客户列表顶部【批量导入】；先【下载导入模板】。**导入模板字段与客户列表/新建客户字段一致**（客户名称、国家/地区、行业、客户潜质分级、站点、客户联系人姓名/职位/邮箱/电话；表头不一致整单拒绝，提示"导入模板与系统字段不一致，请下载最新模板"）。
- **流程：** 下载模板 → 填写 → 上传（Excel/CSV）→ 系统校验 → 预览（错误行标红与原因）→ 确认导入。
- **匹配与归属：** 按客户唯一性（**站点 + 客户名称**，见 §14.7.1、§30.6 唯一约束）匹配已有客户——命中则跳过并在结果中标注"已存在，未重复创建"，未命中则新建；站点须在操作人权限范围内；负责人默认导入操作人，运营专员 / 系统管理员可在模板内指定权限范围内业务员；新建客户生成跟进记录（备注"批量导入客户"）。
- **结果汇总：** "成功 N 条、跳过 M 条（已存在）、失败 K 条（可下载失败明细）"；记录操作日志（操作类型=导入，见 §23.9、BR-054）。
- **权限：** 业务员（仅本人）、运营专员 / 系统管理员（权限范围内）可导入；**协同人不可（严格只读）**。

### 14.8 异常处理

引用 §29.1、§29.3、§29.5。模块特有：无数据"暂无客户"、搜索无结果"未找到匹配的客户"；同一站点内客户名称相同"该客户已存在，请勿重复创建"（不同站点允许同名各自建档）；批量变更部分成功弹窗汇总。

## 15. 客户详情页

### 15.1 模块概述

- **定位：** 展示单客户完整信息的**只读**查看页；客户资产增删改入口均在客户列表 / 合同中心等，**详情页不提供任何写操作按钮**。
- **打开方式：** **新开页签**（工作区多页签）；不占侧边菜单。从客户列表、线索详情关联客户、消息「查看客户」等入口跳转时新开或激活已有同客户详情页签（同客户重复打开复用同一页签）。
- **面向角色：** 全部角色（权限同 14.2；协同人只读查看）。

### 15.2 页面结构

基础信息区 + 标签页导航（客户信息 / 线索记录 / 客户联系人 / 合作合同 / AI 客户画像）。**无顶部操作工具栏**；各标签页默认只读，**例外：AI 客户画像支持人工编辑**（§15.4.5）。

### 15.3 基础信息字段

**客户名称**（大号；客户主体展示名称：企业/个体工商户取企业名称，个人取询盘联系人姓名，见 §2.4）、客户编号（CUS-YYYY-XXXX）、关联线索数、站点、负责人、国家/地区、行业、**客户潜质分级**（字典取值、人工维护，见 §26.3）、客户标签（水平排列超出"+N"，无则"-"）。

### 15.4 标签页内容

> **只读约定：** 客户信息 / 线索记录 / 客户联系人 / 合作合同仅展示数据；不提供【关联线索】【新增/编辑/删除联系人】【新增合同】【打标签】等操作（写操作入口见 §14.7、合同中心 §16）。**AI 客户画像**为例外，支持人工编辑（§15.4.5）。

#### 15.4.1 客户信息

客户名称（与列表口径一致，见 §2.4）、国家/地区、行业、客户潜质分级、官网、联系方式（取主要联系人电话+邮箱，无则"-"）、负责人、创建时间。**不提供【关联线索】**（关联线索在新建客户或线索转客户流程完成）。

#### 15.4.2 线索记录

筛选（站点、线索状态）；字段：线索编号（跳详情）、站点、来源渠道、询盘时间、线索状态、负责人；默认按询盘时间倒序；空显"暂无关联线索"。

#### 15.4.3 客户联系人

只读列表：客户主体下已有联系人（来源：线索转客户/关联时由询盘联系人归集，见 §27.3；后续新增见列表【添加联系人】§14.7.2a）。字段：姓名、职位、邮箱、电话、WhatsApp、联系角色、主要联系人（⭐）。空显"暂无联系人"。**详情内不提供新增/编辑/删除。**

#### 15.4.4 合作合同

只读展示与该客户有关联的合同记录（`customer_id` 匹配）。统计（累计合同数、累计合同金额、最近签约时间、最近签约合同，统计卡仅取状态=**已签约**，口径见 27.7）；列表字段：合同编号（可跳转合同中心/合同详情查看）、合同名称、金额（¥）、签约时间、合同状态（已签约/失效，按签约时间倒序，**失效合同仍展示**）；空显"暂无合作合同"。**详情内不提供【新增合同】及合同编辑/作废等写操作**（合同增删改查在合同中心 §16）。

#### 15.4.5 客户画像

客户资产阶段的「客户画像」，结构与消息/线索侧一致，统一包含：企业工商信息、过往参展信息、企业风险提示（展示细项可含企业名称/简称/工商信息/参展次数与展会名称/风险提示等）。数据来自转客户或关联客户时按 BR-036 / §7.9 同步：客户侧**尚无**画像记录则写入询盘联系人侧画像；客户侧**已有**画像记录则**整包不碰、绝不覆盖**。企业工商信息仅在识别到企业或个体工商户时展示，个人客户无则显"-"。**详情页支持人工编辑客户画像**（本页提供编辑入口）；编辑保存后以人工为准，后续 AI **不覆盖**（BR-017、BR-016c）。年营收不写入客户标准字段；不自动带入意向总结/意向产品/线索标签。手动创建且未关联 AI 分析、或客户侧尚无结构化画像时，显"**当前客户暂无客户画像**"（可附说明：暂无企业工商信息、过往参展信息与企业风险提示）；无存量时可在本页编辑中首次补录。**禁止**用意向总结 / `aiProfile` 类业务描述文案冒充客户画像，以免与结构化画像混淆。

### 15.5 异常处理

引用 §29.1、§29.3。模块特有：客户不存在"客户不存在或已被删除"；各标签页空态对应提示；联系人已删除关联数据显"-"不阻塞。

## 16. 客户中心 · 合同中心

### 16.1 模块概述

- **定位：** 统一管理销售合同，记录成交信息，支撑经营分析与成交追溯。合同必须关联客户，可选关联线索。**合同模块为「合同记录管理」（方案 A）：仅管理合同基本信息、合同金额、签约日期、合同状态；不含审批流程、OA 流程、合同签署流程（非合同审批系统）。不提供合同删除入口。**
- **面向角色：** 全部角色（**协同人仅只读查看合同，默认可导出，不可新增/编辑/录入跟进/作废/删除**）。

### 16.2 数据权限

数据范围遵循 §4.2，按合同负责人及站点判定（合同不设独立站点，继承客户站点），不按合同状态细分。**协同人仅只读查看合同（默认可导出），不可编辑、录入跟进、新增、作废/删除。**

### 16.3 页面结构

筛选条件栏 + 表格区（操作 + 分页）。

### 16.4 筛选条件

| 字段 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| 合同编号 | 文本 | 空 |
| 客户名称 | 文本 | 空（模糊搜客户名/邮箱） |
| 签约日期 | 日期区间 | 本月 | 按**合同业务发生时间**（`sign_date`）筛选；`signDateStart`/`signDateEnd`；允许只填开始或只填结束；空值不限制 |
| 创建时间 | 日期区间 | 不限制 | 按**合同录入系统时间**（`createdAt`）筛选；`createTimeStart`/`createTimeEnd`；允许只填开始或只填结束；空值不限制 |
| 合同金额 | 数字区间（最小/最大）| 不限制 | 按合同金额范围筛选；`amountMin`/`amountMax`；允许只填一端；最小不大于最大（违例提示"最小金额不能大于最大金额"）|
| 合同状态 | 下拉 | 全部 | 取值：已签约 / 失效（见 §26.2） |
| 负责人 | 下拉 | 全部 |

> **签约日期与创建时间口径区别：** 签约日期=合同业务发生时间（人工录入），创建时间=合同录入系统时间（系统生成）；两字段独立筛选、不混用。重置时恢复默认（签约日期=本月，创建时间=不限制）。

### 16.5 表格字段

合同编号（人工录入，跳详情）、合同名称、客户（展示客户名称，口径见 §2.4）、金额（¥，如 ¥45,200）、签约日期（YYYY-MM-DD）、合同状态（彩色徽章：已签约 / 失效）、负责人、附件（"N 个附件"悬停文件名）、操作（详情、编辑、更多→作废）。**失效合同仍保留在列表中展示**（可筛选）。列表顶部提供【导出】（CSV，按当前筛选/选中，见 §16.6.4）。分页：创建时间倒序，20 条/页，20/50/100。列表顶部提供【批量导入】（见 §16.6.5）。**不提供删除。**

### 16.6 合同操作

#### 16.6.1 新增合同

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 合同编号 | 文本 | 是 | 人工录入，系统不生成，提交后不可改；站点内唯一（合同继承客户站点，重复"该合同编号已存在，请重新输入"） |
| 合同名称 | 文本 | 是 | — |
| 客户 | 下拉搜索 | 是 | 权限内客户；展示**客户名称**（企业/个体工商户取企业名称，个人取询盘联系人姓名，见 §2.4） |
| 关联线索 | 下拉搜索 | 否 | 仅已选客户下线索；未选客户置灰；换客户清空；状态不限 |
| 金额 | 数字 | 是 | ¥，>0 |
| 签约日期 | 日期 | 是 | 默认今日 |
| 合同状态 | 下拉 | 是 | 默认**已签约**（仅已签约 / 失效；录入固定为已签约） |
| 合同附件 | 上传 | 否 | PDF/doc/docx/xls/xlsx/jpg/png；单个≤10MB，建议≤5 个 |

> **无「合同结束时间」字段。**

提交规则：
- 负责人默认当前用户；生成跟进记录（备注"录入合同[编号]"）。
- **线索联动（BR-056）：** 若填写了关联线索 → 该线索状态置为**已成交**（不论原状态为待跟进/跟进中/已转客户等；已是已成交则保持）。未关联线索则不改任何线索。
- 仅驱动**本条关联线索**，不影响同客户其他线索。

#### 16.6.2 编辑合同

- **字段口径：** 编辑表单字段与合同详情展示字段对齐（§16.8）。
- **只读（不可改）：** 合同编号、客户、关联线索。
- **可改：** 合同名称、金额、签约日期、合同状态（已签约 / 失效）、负责人、附件。
- **改为失效：** 须填写失效原因（同【作废】）；确认后合同状态→失效，并触发与 §16.6.3 / BR-056 相同的线索回退联动；【作废】入口保留（快捷作废，不替代编辑改状态能力）。
- **由失效改回已签约：** 允许；若合同有关联线索，按 BR-056 将该线索置为已成交（已是已成交则保持）。
- 保存后生成跟进/操作记录（备注含变更字段）。
- **列表/详情：** 失效合同仍留在列表，可查看详情；编辑入口不因失效单独置灰（与上条状态可回改一致）。

#### 16.6.3 作废

- **入口：** 操作列更多→【作废】（按钮文案为「作废」；状态展示为「失效」）。亦可在【编辑合同】中将合同状态改为失效（见 §16.6.2）。
- **准入：** 仅状态=**已签约**可作废；已失效不可再作废（可经编辑改回已签约后再作废）。
- **弹窗：** 作废原因（必填）；确认后合同状态→**失效**。
- **列表表现：** 失效合同**仍留在合同列表**（可筛选），不隐藏、不物理删除；合同中心**不提供删除**（BR-050）。
- **线索联动（BR-056）：**
  1. 合同**无关联线索** → 仅改合同状态，不动线索。
  2. 合同关联线索 A1，且 A1 当前=**已成交**：若 A1 下仍存在其他状态=**已签约**的合同 → A1 **保持已成交**；若 A1 下已无其他已签约合同 → A1 回退为**已转客户**。
  3. 仅回退本条关联线索 A1，不影响同客户其他线索。
  4. 当线索状态实际发生回退时，在**该线索**跟进记录中追加一条**系统执行**记录，文案示例：`系统：合同[编号]已失效，线索状态由「已成交」变更为「已转客户」`。
- 详情顶部可展示失效原因+时间+操作人；系统日志记录。

#### 16.6.4 合同导出

- **入口：** 合同列表顶部【导出】。
- **字段：** 导出范围（当前选中[默认]/当前筛选结果/当前页）、文件格式（只读 CSV）、导出上限（"最多 10000 条"）。
- **规则：** 导出内容为当前权限范围内合同列表字段（含已签约与失效；**不含合同结束时间**）；超 10000 条提示"数据量过大，请缩小筛选范围"；成功下载本地，失败 Toast"导出失败，请重试"。
- **权限：** 业务员（本人负责）、运营专员（负责站点）、协同人（授权站点只读，默认可导出）、系统管理员（全部）均可导出；记录操作日志（操作类型=导出，见 §23.9、BR-054）。

#### 16.6.5 合同导入

- **入口：** 合同列表顶部【批量导入】；先【下载导入模板】。
- **模板字段（与 §16.6.1 新增合同表单一致，不含附件）：** 合同编号（必填）、合同名称（必填）、客户（必填）、关联线索（可选）、金额（必填，>0）、签约日期（必填）、合同状态（必填，默认已签约；枚举仅已签约/失效）。**无合同结束时间。合同附件不支持批量导入**，导入成功后可在合同详情页单独上传。
- **流程：** 下载模板 → 填写 → 上传（Excel/CSV）→ 系统校验 → 预览（错误行标红与原因）→ 确认导入。
- **匹配与校验：** 客户按（站点 + 客户名称）在操作人权限范围内精确匹配已有客户，未命中整行失败；关联线索填写时须属于已匹配客户，否则行失败；合同编号在站点内唯一（继承客户站点），重复行失败；金额/日期格式校验；模板表头与系统字段不一致整单拒绝，提示"导入模板与系统字段不一致，请下载最新模板"。
- **线索联动：** 导入成功且关联线索、状态=已签约时，按 §16.6.1 / BR-056 将关联线索置为已成交。
- **归属：** 负责人默认导入操作人；系统参数「新增合同自动归属客户负责人」=是 时默认合同业务员=客户当前负责人；导入结果汇总"成功 N 条、失败 K 条（可下载失败明细）"。
- **权限：** 业务员（本人负责）、运营专员（负责站点）、系统管理员（全部）可导入；**协同人不可（严格只读）**；记录导入日志（操作类型=导入，见 §23.9、BR-054）。

### 16.7 合同状态流转

合同状态机权威定义见 §26.2（**仅已签约 / 失效**；录入默认已签约；可通过【作废】或【编辑】改为失效；可通过【编辑】由失效改回已签约）。不提供终止、执行中、已完成等状态。

### 16.8 合同详情页

- 入口：操作列【详情】或合同编号；独立页面（原型可为抽屉，字段口径以下表为准）。
- **展示字段（与编辑可编辑项对齐）：** 合同编号、合同名称、合同状态、负责人、客户、金额、签约日期、关联线索、附件；另有操作记录区。
- 布局：顶部信息区（编号/名称/状态徽章/负责人）、基本信息区（客户名跳详情、金额¥、签约日期）、关联信息区（关联线索跳详情）、附件区（下载预览）、操作记录区（**仅合同自身操作记录：创建/编辑/作废/附件操作，不含跟进记录**）。
- **编辑约束回顾：** 编号 / 客户 / 关联线索只读；其余业务字段可在【编辑】中修改（§16.6.2）。
- 操作记录字段：操作时间、操作人、操作类型、操作内容（变更前→后）、关联字段。

### 16.9 异常处理

引用 §29.1、§29.3、§29.5。模块特有：无数据"暂无合同"、搜索无结果"未找到匹配的合同"；合同编号重复拦截；关联客户已逻辑删除时客户名后标"(已注销)"。

### 17.2 页面结构

顶部筛选栏 + 指标卡区+ 分析区

### 17.4 指标卡

| 卡片 | 定义 | 变化量格式 |
| :--- | :--- | :--- |

| 高意向线索| 本人为负责人、当前跟进阶段属高意向（打样/报价/谈判）的线索数 | 线索列表（按阶段过滤） | 全部 |

> - **询盘总数** = 周期内创建的全部线索数量（历史产生量；含后续标记为无效/丢失的线索）。  
> - **有效线索数** = 当前状态不属于【无效、丢失】的线索数量（当前存量）。  
> - 二者时间口径与业务含义不同，**不得合并为同一指标**。邮件 / WhatsApp / 人工录入 / 导入进入线索列表时入库视为有效，无效/丢失仅在后续跟进中判定。  
> - 高意向阶段以字典「跟进阶段」中允许转客户的阶段（打样/报价/谈判）为准。  
> **与 §18 获客分析切分：** §18 同时含消息接入效率与全盘获客结构；其中「询盘总数 / 有效线索数」与 §17 **口径同源**，§18 用于渠道/地域/产品/站点等获客结构，§17 用于销售漏斗、人员表现与成交经营。§18 顶部「生成线索数」仅计消息链路产出，不等于询盘总数。

- **线索增长与成交金额趋势（双轴）：
根据顶部时间筛选条件动态展示趋势数据，支持按天、月、年三个时间粒度查看。

左轴展示线索数量（柱状图）；
右轴展示成交金额（折线图）；
按「年」筛选时，按月聚合展示趋势（如 1 月～当前月份）；
按「月」筛选时，按天聚合展示趋势（如 1 日～当前日期）；
按「天」筛选时，展示当天统计结果。
图表需保持与顶部时间筛选条件一致，确保用户查看的数据范围与统计维度统一。

| :--- | :--- |

### 18.7 意向产品分析

按线索字段**意向产品**统计**线索数**（前 N + 其他），反映市场产品需求热度。

### 18.8 客户关注点分析

### 18.9 站点价值分析

字段：站点名称、**询盘总数**（与 §17 同源）、**有效线索数量**（与 §17 同源：当前状态 ≠ 无效、≠ 丢失）、**转化率**（已成交客户数 / 询盘总数 ×100%；已成交客户数口径见 §27.7）、成交金额（口径见 §27.7）。排序支持按询盘总数/有效线索数/转化率/成交金额，默认成交金额倒序；每页 10 条。

> **站点统计口径：** 邮件线索按邮箱对应站点计入站点统计；WhatsApp 线索按业务员唯一授权站点计入站点统计；人工录入/导入线索按其线索站点计入。

| 高意向客户数 **【二期/延后】** | 潜质分级="可跟进"或名下存在高意向线索（打样/报价/谈判阶段）的客户数 |

| 活跃客户数 | 在「客户活跃度判定天数」内有跟进/消息互动的客户数（阈值默认 90 天，见系统参数 §23.7）|
| 沉默客户数 | 超过「客户活跃度判定天数」无跟进/消息互动的客户数（默认 90 天，见 §23.7）|
| 客户合同金额 | 客户名下合同金额合计（口径见 27.7）|

### 19.4 图表

- **客户潜质分级分布：** 按客户潜质分级（可跟进/潜在/一般客户）分布（饼图）。
- **国家/地区分布：** 按客户所属国家/地区分布（柱状/地图，前 10 + 其他）。
- **客户新增趋势：** 按月统计新增客户数，以**折线图**呈现（X 轴本年 1 月至当前月，固定本年、不受时间筛选影响），反映客户资产的增长节奏。
- **客户标签分布：** 按客户人工标签分布（前 N + 其他）。
- **客户关注点分布：** 按**客户关注点**（字典：客户域→客户关注选项，见 §23.5）统计**客户数**。口径：取客户名下**关联线索**的关注点，在同一客户内**聚合去重**后，该客户计入对应关注点；一客户可落入多个关注点桶。与获客分析 §18.8（线索维度）并存、维度不同。
- **客户活跃度：** 按最近互动时间分桶（活跃/沉默，阈值取系统参数「客户活跃度判定天数」，见 §23.7）分布，反映客户经营健康度。
- **客户成交分析：** 成交客户占比、客户合同金额分布/年度成交趋势（X 轴本年 1 月至当前月）。

### 19.5 客户画像

- **补充说明：** 客户画像用于**沉淀客户价值信息**，是客户资产分析的基础。单客户画像在【客户详情 → AI 客户画像】查看与**人工编辑**（§15.4.5）；本模块从客户资产视角聚合与呈现。数据初始来源 AI；人工编辑后以人工为准、AI 不覆盖（见 §15.4.5、BR-017）。
- **包含维度：** 企业工商信息、AI 参展资料、AI 风险提醒、客户类型（业务口径：企业/个体/个体工商户，见 §30.6，不作独立字段统计）、核心诉求、采购产品、采购量级、沟通语言、国家/地区。

### 19.6 业务员简化版视图

指标卡与全部图表、客户画像均展示，数据限定本人负责客户；不展示站点筛选；时间范围可用；布局与其他角色一致。

### 19.7 异常

## 20. 站点中心 · 站点管理

### 20.1 模块概述

### 20.5 字段与规则说明（原配置页字段权威，入口已并入 §20.3）

> **MVP 不提供独立「站点配置」页签/列表【配置】按钮。** 下列规则仍为字段权威；维护入口仅为【新增站点】/【编辑】。

#### 20.5.1 回收规则 **【二期/延后】**

> **MVP 不做站点级自动回收配置**，公海回收仅保留运营专员手动回收（见 §10.7）；新增/编辑弹窗内**预留展示**，本期不可编辑。

二期：配置该站点线索回收规则：启用自动回收（开关，默认关闭，仅系统参数未配置时生效）、回收超时时长（数字+单位，启用必填，默认 7 天）。**回收的完整线索生命周期与固定规则以线索中心为准（见 §10.7），本模块不重复定义。**

#### 20.5.2 AI 规则

启用 AI 识别（开关，默认开启，关闭后该站点消息不调用 AI 识别）。在新增/编辑弹窗中维护，**仅保留该开关**。

#### 20.5.3 运营人员与主邮箱

- **定位：** 为当前站点**绑定运营人员**，并以该运营人员的**业务邮箱**作为站点**主邮箱**对外接收询盘。业务关系为「**站点 → 绑定运营人员 → 运营人员绑定业务邮箱 → 邮箱接收消息 → AI 识别有效询盘 → 生成线索**」；**取消「邮箱直接绑定站点」的方式**。
- **面向角色：** 系统管理员。
- **创建必填：** 新增站点时运营人员与主邮箱均为必填，未选不可保存。

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 绑定运营人员 | 下拉 | 是 | 选择负责本站点的运营人员（仅"启用"运营专员可选；协同人不可选）|
| 主邮箱 | 下拉 | 是 | 取所选运营人员已绑定的业务邮箱；该运营人员有多个业务邮箱时选择其一作为本站点主邮箱 |
| 绑定时间 | 系统字段 | — | 只读 |

**规则：**

- **一站点一主邮箱**：当前站点仅能绑定一个运营人员与一个主邮箱；同一业务邮箱不可同时作为多个站点主邮箱、不可重复绑定。
- 保存后，该邮箱即为本站点主邮箱；后续该邮箱收到的有效询盘线索：站点 = 本站点，负责人 = 该运营人员，由运营人员转派给业务员（归属与转派规则见 §7.5.2、§27.1，本模块不重复定义）。
- 更换绑定运营人员/主邮箱仅影响后续新邮件生成的线索，不影响历史邮件与历史线索。
- 校验：邮箱已被占用→"该邮箱已绑定，不可重复绑定"；异常见 §29.11。

### 20.6 站点停用连锁影响

| 影响范围 | 停用后行为 |
| :--- | :--- |
| 消息接收 | 不再接收新消息（邮件消息与 WhatsApp 消息均停收），已有消息永久保留 |
| AI 识别 | 无新消息故不调用，历史结果保留 |
| 公海自动回收 | 暂停（含手动回收对该站点不可用），重新启用后恢复 |
| 现有线索/客户/合同 | 历史数据保留，已有业务可继续流转 |
| 站点筛选可见性 | 停用站点仍在各筛选下拉中（带「已停用」标记），可查历史数据 |
| 站点删除 | 不允许物理删除，仅停用（BR-009） |

### 20.7 异常

引用 §29.1、§29.3、§29.5。模块特有：无数据"暂无站点"、搜索无结果"未找到匹配的站点"；停用存在进行中线索允许停用；主邮箱重复绑定拦截（见 §20.5.3、§29.11）。

## 20A. 站点中心 · 站点运营数据



### 20A.1 模块概述

- **定位：** 按**站点 + 时间周期（月/周）**记录站点侧运营指标（广告、流量、询盘、GSC 等），为获客分析（§18）等提供数据来源。数据来源为**手动维护 + 批量导入**。
- **面向角色：** 系统管理员（新增/编辑/删除/导入/导出）；运营专员/协同人在授权站点范围内**只读查看**（菜单可见性见 §4.3）。业务员无该菜单。
- **写权限：** 仅系统管理员可新增/编辑/删除/导入；操作记录导入日志（BR-054）。
- **核心规则：** **列表展示字段 = 导入字段 = 导入模板字段，三者完全一致**（字段顺序、名称一一对应）。

### 20A.2 字段定义

| 序号 | 字段 | 建议列名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| — | 站点 | `site_id` | BIGINT | 是 | 维度；FK→站点；导入模板用站点名称/编码匹配 |
| — | 统计周期类型 | `period_type` | TINYINT | 是 | 维度；月 / 周 |
| — | 统计周期 | `period` | VARCHAR(16) | 是 | 维度；月=YYYY-MM，周=YYYY-Www（如 2026-W27） |
| 1 | 广告花费 | `ad_spend` | DECIMAL(14,2) | 否 | 允许空 |
| 2 | 网站浏览次数 | `website_visits` | BIGINT | 否 | 允许空 |
| 3 | 询盘量 | `inquiry_count` | BIGINT | 否 | 允许空 |
| 4 | 高潜询盘量 | `high_intent_inquiry_count` | BIGINT | 否 | 允许空 |
| 5 | 询盘转化率 | `inquiry_conversion_rate` | DECIMAL(7,4) | 否 | 允许空；百分比 |
| 6 | 单个询盘成本 | `cost_per_inquiry` | DECIMAL(14,2) | 否 | 允许空 |
| 7 | 高潜询盘成本 | `cost_per_high_intent_inquiry` | DECIMAL(14,2) | 否 | 允许空 |
| 8 | 平均访问时长 | `avg_visit_duration` | DECIMAL(10,2) | 否 | 允许空；秒 |
| 9 | 网站跳出率 | `bounce_rate` | DECIMAL(7,4) | 否 | 允许空；百分比 |
| 10 | GSC 曝光量 | `gsc_impressions` | BIGINT | 否 | 允许空 |
| 11 | GSC 点击次数 | `gsc_clicks` | BIGINT | 否 | 允许空 |
| 12 | GSC 出词数 | `gsc_keywords` | BIGINT | 否 | 允许空 |
| 13 | GSC 平均排名 | `gsc_avg_position` | DECIMAL(8,2) | 否 | 允许空 |

> **唯一约束：** `UNIQUE(site_id, period_type, period)`——同一站点同一周期仅一条记录。

### 20A.3 列表页

- **顶部筛选：** 站点（下拉，仅授权站点）、统计周期类型（月/周）、统计周期区间。
- **列表字段：** 与 §20A.2 完全一致（站点、周期类型、周期 + 13 项指标），末列操作（编辑、删除）；空值显"--"。
- **操作按钮：** 【新增】、【批量导入】、【下载导入模板】、【导出】。
- **分页：** 按周期倒序，20 条/页，20/50/100。

### 20A.4 新增/编辑弹窗

- 字段同 §20A.2；维度字段（站点、周期类型、周期）编辑时只读、新增时必填；13 项指标均允许空。
- 校验：站点/周期类型/周期非空；`(site_id, period_type, period)` 重复时拦截"该站点该周期数据已存在，请编辑或使用导入覆盖"；数值字段格式校验（非负、百分比 0~100）。
- 保存记录操作日志。

### 20A.5 批量导入

- **入口：** 列表页【批量导入】；先【下载导入模板】。**模板字段必须与列表字段完全一致**（顺序：站点、周期类型、周期 + 13 项指标）。
- **流程：** 下载模板 → 填写 → 上传（Excel/CSV）→ 系统校验 → 预览（含错误行标红与原因）→ 确认导入。
- **校验规则：** 站点须存在且在操作人权限范围内；周期类型/周期格式合法；数值格式合法；模板表头与系统字段一致，否则整单拒绝并提示"导入模板与系统字段不一致，请下载最新模板"。
- **重复处理：** 命中唯一键 `(site_id, period_type, period)` 时**按行覆盖**（更新已有记录），非重复行新增；导入结果汇总"成功 N 条、覆盖 M 条、失败 K 条（可下载失败明细）"。
- **权限：** 写操作仅系统管理员（见 §20A.1）；运营专员/协同人只读查看授权站点数据。

### 20A.6 权限

## 21. AI 能力管理

### 21.1 模块概述

- **定位：** 统一管理 AI 服务商及模型能力（新增/编辑/启停），定义 AI 输出结构、同步规则与降级规则。
- **面向角色：** 系统管理员。

### 21.2 AI 能力列表

字段：能力名称、AI 服务商、AI 模型、API 端点、状态（启用/停用）、创建时间、操作（编辑、启用/停用、删除[仅停用状态可删，启用置灰]）。搜索（能力名/服务商）、按状态筛选、创建时间倒序、每页 10 条、刷新。

### 21.3 AI 能力字段

| 字段 | 类型 | 必填 | 编辑可改 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| 能力名称 | 文本 | 是 | 是 | — |
| AI 服务商 | 下拉 | 是 | 是 | OpenAI/Azure OpenAI/通义千问/智谱 AI/DeepSeek/自定义；服务商+模型组合唯一 |
| AI 模型 | 下拉 | 是 | 是 | 随服务商联动收敛；"自定义"降级为文本输入 |
| API 端点 | 文本 | 是 | 是 | 合法 URL |
| API Key | 文本 | 是 | 是（脱敏 sk-****xxxx，改需重输） | ≥20 字符 |
| 启用状态 | 开关 | 是 | 是 | 保存立即生效 |

服务商模型枚举（系统硬编码）：OpenAI/Azure→GPT-4o、GPT-4o-mini；通义千问→Qwen-Plus、Qwen-Turbo；智谱→GLM-4、GLM-4-Flash；DeepSeek→DeepSeek-Chat、DeepSeek-Reasoner；自定义→人工录入。

### 21.4 AI 输出结构与同步

> 与 §7.9 面板结构对齐。权威交互与复用规则以 §7.9 为准；本节为对象级同步契约。

#### 21.4.1 输出结构（最小集）

- **意向总结**（消息级）
- **意向产品**
- **客户画像**（统一模块）：企业工商信息、过往参展信息、企业风险提示
- **询盘标签**（消息层 AI 标签；写入线索后成为线索标签的 AI 来源部分，见 §25）

#### 21.4.2 同步关系

| 对象 | 写入/展示 | 明确不同步 |
| :--- | :--- | :--- |
| 消息 | 意向总结、意向产品、客户画像（分析过程产出；画像落询盘联系人资料）、询盘标签 | — |
| 线索 | 意向产品（正式字段）；询盘标签→追加线索标签；客户画像经询盘联系人资料**引用展示** | **意向总结**（不写线索字段） |
| 客户 | 转客户/关联时：① 客户侧**无**画像记录则写入联系人侧客户画像；客户侧**已有**画像记录则整包不碰；② **国家/地区、所属行业**从工商画像预填客户标准字段（新建预填、关联仅补空；字典匹配不上留空；可人工改） | 意向总结、意向产品、询盘标签/线索标签（客户仅人工标签） |

#### 21.4.3 关键规则

1. **触发：** 消息接入时分析一次并持久化；打开详情不重跑；MVP 无重新分析入口（§7.9）。
2. **意向总结：** 仅消息 AI 结果，不进线索、不进客户。
3. **意向产品：** 生成线索时写入线索字段；不进客户（BR-036）。
4. **客户画像：** 存询盘联系人资料；首次生成、有则复用存量、人工编辑优先（BR-016c、BR-017）；转客户时客户无画像才写入、有则不覆盖（BR-036）。
4a. **国家/地区、所属行业：** 转客户/关联时从工商画像预填客户标准字段（新建预填、关联仅补空；字典对齐，匹配不上留空）；之后仅人工【编辑】修改，AI 不因后续消息自动改客户档案这两字段。
5. **询盘标签：** 生成线索时追加到线索标签；同名去重；人工删除过的同名禁止 AI 再追加（§25、BR-016a）。
6. **所属企业名：** AI 识别的企业名称可作为线索「所属企业」字段，与客户画像模块配合展示；年营收不写入客户标准字段。

### 21.5 AI 降级规则

AI 不可用/调用失败时**不自动识别询盘、不自动创建线索**，业务不中断（BR-015）；消息保留、进入人工处理，由人工兜底创建线索、归属当前操作人；AI 失败不影响 CRM 运行。**AI 属辅助能力，不阻断业务流程：AI 成功时自动识别有效询盘并自动创建线索、写入 AI 结果；AI 不可用/失败时消息保留、进入人工处理、由人工兜底创建线索（见 §21.8、§27.1）；一期不设规则引擎自动识别。**

### 21.6 业务关联（只读）

系统内置，不提供在线配置：邮件意向分析、WhatsApp 意向分析、AI 自动提取企业信息、批量 AI 提炼，均调用系统默认 AI 能力。AI 功能启用总开关在系统管理→系统参数（业务规则配置）控制（见 §23.7）。

### 21.7 操作与异常

新增/编辑按 21.3 字段校验（必填、URL 合法、Key≥20、服务商+模型唯一）；启用/停用立即生效无二次确认；删除仅停用状态可删+二次确认，历史记录保留审计。API 端点连续 N 次调用失败自动标记异常，不影响其他能力，系统管理员可手动停用。AI 运行期异常的用户提示与降级引用 §29.7。

### 21.8 AI 有效询盘识别与自动生成线索（业务流程）

**AI 是线索生产入口。** 消息进入系统后 → AI 自动分析 → 识别有效询盘 → 提取业务信息 → 自动创建线索 → 绑定负责人 → 进入线索中心。人工创建线索为异常兜底（见 §21.8 第 4 点）。

1. **有效询盘识别标准（AI 判定）：** 消息（邮件消息或 WhatsApp 会话消息）满足以下**任一类型**即判定为有效询盘 → 自动生成正式线索 → 进入线索列表。

   | 类型 | 判断说明 | 示例 |
   | :--- | :--- | :--- |
   | 产品咨询 | 询问某类产品、型号、规格、功能、参数 | "你们有 XX 产品吗？" |
   | 价格咨询 | 询问报价、价格区间、采购成本 | "请发一下报价单。" |
   | 样品需求 | 询问样品、打样、试用、寄样 | "可以寄样品给我们测试吗？" |
   | 采购需求 | 表达采购、订购、批量购买意向 | "我们想采购 500 件。" |
   | 交期咨询 | 询问交货时间、生产周期、库存 | "最快什么时候可以发货？" |
   | 定制需求 | 提到 OEM、ODM、Logo、包装、定制规格 | "能否按我们的尺寸定制？" |
   | 合作代理 | 询问代理、分销、区域合作 | "我们想成为代理。" |
   | 项目询盘 | 项目背景、采购计划、招投标需求 | "我们有项目需要采购。" |
   | 复购/新需求 | 老客户提出新的产品、数量、报价需求 | "想再采购另一款产品。" |
   | 明确联系方式交换 | 请求销售联系、会议、资料 | "请安排销售联系我。" |

   - **业务规则：** 命中上述任意一种类型 → 判定有效询盘 → 自动生成正式线索 → 进入线索列表。
   - **非有效消息（不生成线索）：** 自动回复、广告、无业务内容消息、垃圾信息。
2. **判断结果存储（方案 B，不落库）：** AI 判断过程**不持久化**；**不新增** AI 有效询盘标识 / AI 置信度 / AI 判断原因等字段；AI 仅输出最终业务结果（线索及其提取字段与 AI 分析信息），判定有效性仅作为流程分支依据。
3. **AI 自动提取字段：**
   - **联系信息：** 联系人姓名、邮箱、电话、WhatsApp 账号。
   - **企业信息：** 企业名称、国家/地区、企业基本信息；其中**国家/地区写入线索「来源国家/地区」**（`source_country`，供获客分析 §18，可人工改）。
   - **业务信息：** 意向产品、产品需求、采购描述。
   - **AI 分析信息：** 意向总结、意向产品、客户画像（企业工商信息/过往参展信息/企业风险提示）、询盘标签（字段与同步口径见 §7.9、§21.4、§25，不重复定义）。
4. **AI 失败兜底：** AI 成功→自动创建线索；AI 不可用/失败→消息保留→进入人工处理→人工创建线索（见 §21.5）；**AI 失败不阻断业务**。人工创建线索仅用于：AI 服务不可用、AI 无法识别、历史数据补录、特殊业务场景。
5. **负责人归属与异常：** 归属规则与"收件邮箱/账号未绑定、用户停用、用户不存在"的待分配兜底见 §27.1、BR-020。

**邮件 — AI 识别有效询盘后：**

- 自动生成正式线索，站点 = 收件邮箱对应站点。**两类邮箱：** ① 业务员个人邮箱→**负责人=业务员本人**，直接进入本人线索列表；② 站点主邮箱（运营人员业务邮箱，`mail_site_id`，见 §20.5）→**负责人=该站点绑定的运营人员**，由其手动**转派给业务员、不进公海**（转派 T1–T3 见 §7.5.2；无线索可只投递）。
- 站点未绑定运营人员/主邮箱：其邮件不参与自动生成线索。

**WhatsApp — AI 识别有效询盘后：**

| 场景 | 站点 | 负责人 | 进入位置 |
| :--- | :--- | :--- | :--- |
| 业务员（单一授权站点） | 业务员唯一授权站点 | WhatsApp 绑定业务员 | 线索列表 |

> WhatsApp 仅业务员可绑（见 §24.3.2），线索站点必定明确、创建即"待跟进"、**不进公海池**。

## 22. 通知管理

### 22.1 模块概述

- **定位：** 管理端配置**通知场景对象**与**推送渠道**（站内信 / 钉钉）：场景系统内置不可自定义新增，仅配置渠道、对象、模板内容、启停状态。
- **面向角色：** 系统管理员/运营人员（侧边菜单「通知管理 → 通知场景」）。
- **与接收端区分（唯一权威）：**
  - **配置端（本模块）：** 通知场景列表 + 编辑弹窗；配置「何时推、推给谁、走哪些渠道、模板文案」。
  - **接收端：** 实际通知内容在 Header **头像左侧铃铛**打开的站内信面板中查看；钉钉渠道另达钉钉客户端。
  - **无独立承接页面：** 不设「通知中心」业务列表页、不设侧边菜单收件箱路由；铃铛为站内信唯一入口。

### 22.2 通知场景配置页

#### 22.2.1 结构

顶部筛选（通知场景[模糊搜]、状态[全部/开启/关闭]）+ 表格。

#### 22.2.2 表格字段

通知场景（只读）、通知渠道（顿号分隔，如"站内信、钉钉"）、通知对象（顿号分隔）、状态（彩色徽章）、操作（编辑）。

### 22.3 编辑通知场景弹窗

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 通知渠道 | 多选 | 是 | 站内信、钉钉（同时启用同时送达，无优先级）；本字段即推送渠道配置 |
| 通知对象 | 多选 | 是 | 角色类型、创建人、分配人、站点运营专员、指定用户 |
| 通知内容 | 富文本 | 是 | **场景模板**（标题、正文），触发后按模板生成送达内容；**不是**铃铛里的历史通知列表 |
| 状态 | 开关 | 是 | 开启/关闭 |

校验：渠道≥1、对象≥1、内容非空；保存立即生效；关闭后不再触发推送。钉钉推送通道凭证在系统管理→集成配置（钉钉应用配置）维护，是否启用见 §23.7；站内信为系统内置通道。

### 22.4 通知场景清单（系统内置）

> 通知场景清单为以下 6 项，系统内置、不可增删。

| 通知场景 | 触发时机 | 默认通知对象 |
| :--- | :--- | :--- |
| 新线索生成 | **AI 自动识别有效询盘并生成线索时**（即时触发） | 当前负责人（业务员个人邮箱=业务员本人；站点主邮箱=运营专员；WhatsApp=绑定业务员） |
| 7 天邮件客户未回复提醒 | 邮件客户来信后超 7 天本方未回复 | 当前负责人 |
| 负责人变更 | 执行变更负责人（含运营专员转派站点主邮箱线索给业务员） | 新负责人、原负责人 |
| 邮件未读数量提醒 | 邮箱存在未读复邮件 | 当前邮箱归属用户 |
| 邮件未读超时提醒 | 邮件未读超时 | 当前邮箱归属用户 |
| 邮件未回超时提醒 | 邮件未回复超时 | 当前邮箱归属用户 |
| 邮件未回数量提醒 | 邮箱存在未读或未回复邮件 | 当前邮箱归属用户 |
| 跟进到期提醒 | 线索下次跟进时间到期（到达/超过今日） | 当前负责人 |

### 22.5 Header 铃铛（站内信接收，不占菜单）

- **入口：** 全局 Header 头像左侧铃铛图标；全部角色可见（本人收到的通知）。
- **形态：** 下拉/浮层面板展示站内信列表（标题、摘要、时间、已读/未读等）；**无独立路由页、无侧边菜单项**。
- **与 §22.2 关系：** 铃铛展示的是按场景配置触发后送达的**实例内容**；场景配置页不展示、不管理历史收件记录。

### 22.6 异常

同一场景仅一份配置；场景内置不可增删；对象含失效用户"通知对象中包含已失效的用户，请重新选择"；推送通道未配置时跳过并记日志。

新增（自动生成初始密码 123456，；编辑（登录账号不可改）；重置密码（二次确认，恢复 123456）；启用/停用（二次确认）。

#### 23.2.4 停用用户规则

### 23.3 角色管理（MVP 预置）

MVP 采用预置数据，内置**系统管理员 / 运营专员 / 业务员 / 协同人** 4 角色及权限，**不支持新增角色**（延二期完整 CRUD 与菜单权限可视化配置）。内置保护：系统管理员不可停用/删除/降权，系统至少保留一个启用的系统管理员。角色继承不提供。

#### 23.3.1 列表页

工具栏：新增（MVP 不可用）、修改、删除（预置角色不可删）、导出；筛选：角色名称/权限字符关键词、状态。

| 字段 | 说明 |

## 24. 个人中心

### 24.1 模块概述

- **定位：** 维护当前用户个人信息、账号绑定、密码修改。
- **面向角色：** 全部角色。
- **MVP：** 弹窗形态承载"我的资料（只读）+ 账号绑定 + 修改密码"；独立页面化布局延二期。

### 24.2 我的资料

字段：头像（jpg/png，建议 200×200，≤2MB）、姓名、手机号、邮箱（二期可编辑保存）。保存成功提示"个人信息已更新"。

### 24.3 账号绑定

#### 24.3.1 邮箱绑定（支持多个）

- **面向角色：** **业务员**（可绑定多个个人邮箱，收发处理本人邮件）与**运营专员**（可绑定多个邮箱，当其被绑定为某站点运营人员时，可由系统管理员在【站点管理·新增/编辑】选定其一为该站点主邮箱）均可在个人中心绑定邮箱账号；**协同人不绑定邮箱**（严格只读）。
- 新增绑定（输入邮箱）、删除绑定（二次确认）。**个人中心邮箱绑定不选择站点**；站点主邮箱由系统管理员在【站点管理·新增/编辑】通过「为站点绑定运营人员并选定其业务邮箱」的方式确定（见 §20.5）。

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 邮箱账号 | 输入框 | 是 | 绑定邮箱 |
| 绑定状态 | 系统字段 | 是 | 已绑定 / 已解绑 |

**规则：**

- 业务员 / 运营专员均可绑定**多个**邮箱账号；邮箱**归属绑定用户**。
- **业务员个人邮箱：** 有效询盘线索直接归业务员本人（站点=业务员唯一授权站点，见 §7.10、§27.1）。
- **运营专员被绑定为站点运营人员、其业务邮箱设为站点主邮箱后（一站点一主邮箱）：** 一个业务邮箱只能作为一个站点的主邮箱、一个站点只能有一个主邮箱；同一邮箱**不可重复作为多个站点主邮箱**（不可跨站点/跨用户重复）；站点主邮箱线索归该运营人员、转派业务员（见 §20.5、§27.1）。
- 校验：格式合法、不可重复绑定、同一邮箱不可被多用户绑定。列表字段：邮箱地址、类型（个人邮箱 / 站点主邮箱）、对应站点（站点主邮箱展示）、绑定时间、操作（删除）。

#### 24.3.2 WhatsApp 绑定（仅一个）

- **面向角色：** **仅业务员**可绑定 WhatsApp；运营专员/协同人/系统管理员**不支持**绑定 WhatsApp（见 §4.5、§8.1）。非业务员不展示 WhatsApp 绑定入口。
- 绑定（输入账号 + 授权）、解绑（二次确认）。校验：账号非空、不可被多用户绑定。

**规则：**

- 一个业务员仅允许绑定**一个** WhatsApp 账号。
- WhatsApp 只绑定业务员，**不绑定固定站点**；因业务员恒为单一授权站点，WhatsApp 线索站点 = 业务员唯一授权站点（见 §8.5.2、§27.1）。

### 24.4 修改密码

字段：新密码（≥8 位、含数字和大小写字母）、确认新密码（与新密码一致）。校验对应提示；成功后提示"密码已修改，请重新登录"并自动退出跳登录页。

### 24.5 异常

引用 §29.3、§29.5。模块特有：头像格式/大小超限提示；邮箱/WhatsApp 已被绑定提示；两次密码不一致拦截。

## 25. 标签体系

> 本章为全系统**标签体系唯一权威定义**。人工标签选项由字典管理维护（见 §23.5），AI 分析标签不进字典；业务模块只引用不重复定义。全系统标签统一为**三层**：询盘标签 / AI 分析标签（消息 AI 输出）→ 线索标签（业务使用）→ 客户标签（客户资产，人工独立）。

### 25.1 标签分类总览

| 标签类型 | 字段 | 归属/存储 | 来源 | 维护方式 | 同步规则 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 询盘标签（AI 分析标签） | `ai_analysis_tag` | 消息分析层，AI 面板「询盘标签」展示 | AI 分析生成 | AI 生成（面板只读） | 有效询盘生成线索时追加写入线索标签；同名去重；人工删过的同名禁止再追加；不进客户；不进字典 |
| 线索标签 | `lead_tag` | 线索 | AI（来自询盘标签）+ 人工 | 业务员增/改/删 | 追加不覆盖人工；转客户不写入客户 |
| 客户标签 | `customer_tag` | 客户 | 仅人工 | 客户管理增/删/改 | 独立维护；不继承线索标签；不受 AI 影响 |

> 人工标签（线索标签的人工部分、客户标签）选项由字典管理维护，业务侧新输入同步写回字典；询盘标签（AI 分析标签）不进字典。全系统不单设"标签管理"菜单。

### 25.2 线索标签

- **来源（两类，混排为统一业务标签）：** ① AI 来源——询盘标签在有效询盘生成线索时自动写入/追加（蓝底+圆角+AI 图标）；② 人工——灰底+圆角，选项来自字典线索域→线索手动标签。
- **AI 追加策略（BR-016a）：** 生成线索（或同线索下后续有效消息写入标签）时，询盘标签**追加**到线索标签；**同名（完全一致）去重**；**不删除**已有标签；**不覆盖**人工修改；**人工删除过的同名标签禁止 AI 再追加**（对齐 BR-017）。MVP 无消息「重新分析」入口。
- **人工维护：** 支持新增/修改/删除（AI 来源、人工来源标签均可增删改）；人工修改后视为业务标签，AI 不再覆盖。
- 展示：每线索最多 3 个，超出"+N"（悬停全部）；点击标签快速筛选；筛选区多选逻辑=OR。
- 热门标签：站点下使用频率最高 Top10，仅展示未被当前线索添加的；操作详见 13.5。
- 标签输入：中英文/数字/下划线，≤20 字，同线索不可重复。

### 25.3 客户标签

仅人工标记，无 AI 标签，选项由字典客户域→客户标签维护。**独立于线索标签：线索转客户不自动继承线索标签（BR-016b），且不受消息 AI / 询盘标签追加影响**。维护入口为客户列表【打标签】；客户详情仅只读展示同一份标签数据。超 3 个"+N"，悬停全名。操作详见 14.7.4。

### 25.4 询盘标签（消息 · AI 分析标签）

- **定义：** AI 对消息内容分析输出的标签（如价格敏感、关注某产品、采购意向强、交期关注），展示于 AI 智能分析面板「询盘标签」（只读），不是长期业务管理标签。
- **写入线索（BR-016a）：** 有效询盘生成线索时自动追加为线索标签的 AI 来源部分；同名去重；人工删过的同名禁止再追加；不覆盖人工标签。
- **不作为客户资产标签：** 不进客户中心、不进字典；客户标签始终人工维护。

## 26. 状态流转

> 本章为全系统**状态机唯一权威定义**（线索/合同/客户潜质/站点/用户）。业务模块只展示当前状态并引用本章，不重复描述状态流转。

### 26.1 线索状态机

> **线索状态机与跟进阶段分别维护、互不映射。** 线索状态描述线索的**业务生命周期**；跟进阶段（见 §23.5）描述销售**推进过程**；"转客户"是业务人员**人工主动动作**（见 §13.6），三者不混用。跟进阶段不映射为线索状态，也不自动改变线索状态。

**状态值：** 待分配（公海专用）、待跟进、跟进中、已转客户、已成交、无效、丢失。

**核心流转链路：** 待跟进 → 跟进中 → 已转客户 → 已成交；**已成交可因关联合同全部失效回退为已转客户**（见 BR-056）。

**状态含义与操作角色：**

| 状态 | 含义 | 进入条件 | 操作角色 |
| :--- | :--- | :--- | :--- |
| 待分配 | 公海池中等待分配的线索 | 公海回收（自动/手动）或标记异常后保留公海 | 运营专员 / 系统管理员（协同人只读不可分配） |
| 待跟进 | 已归属负责人、尚未开始跟进 | 渠道生成线索 / 手动新建 / 公海分配 | 负责人本人 |
| 跟进中 | 负责人已开始跟进推进 | 首次录入跟进（BR-038a） | 负责人本人 |
| 已转客户 | 已沉淀为客户资产、线索保留 | 人工主动执行【转客户】成功（BR-038b/033）；或已成交因关联合同全部失效回退（BR-056） | 负责人本人 / 运营专员（协同人只读不可）；回退由系统执行 |
| 已成交 | 已有关联合同且状态=已签约 | 录入/导入已签约合同并关联本线索（BR-056；不要求原状态必须为已转客户） | 由合同动作驱动 |
| 无效 / 丢失 | 异常终态（无跟进价值/流失） | 标记异常 | 负责人本人 / 运营专员（协同人只读不可） |

> **禁止主动回退：** 禁止人工将已转客户→跟进中/待跟进、跟进中→待跟进等；**公海回收（→待分配）为异常回收机制、非链路回退**（见 §10.7）。**唯一允许的成交回退：** 关联合同作废为失效且该线索下已无其他已签约合同时，系统将已成交→已转客户（BR-056）。数据权限口径见 §4.2。

| 当前状态 | 可流转至 | 触发动作 |
| :--- | :--- | :--- |
| 待分配 | 待跟进 | 公海池分配 |
| 待分配 | 无效 / 丢失 | 标记异常（保留在公海池列表展示） |
| 待跟进 | 跟进中 | 首次录入跟进（BR-038a） |
| 待跟进/跟进中 | 已转客户 | 人工主动执行【转客户】成功（BR-038b、BR-033） |
| 待跟进/跟进中 | 无效 / 丢失 | 标记异常 |
| 待跟进/跟进中 | 待分配 | 公海回收（自动/手动） |
| 待跟进/跟进中/已转客户 | 已成交 | 录入/导入已签约合同并关联本线索（BR-056） |
| 已转客户/无效/丢失 | 待分配 | 公海**手动**回收（除"已成交"外均可，自动回收不触发） |
| 已转客户 | 无效 / 丢失 | 标记异常 |
| 无效 / 丢失 | 已转客户 | 重新激活并执行【转客户】成功 |
| 已成交 | 已转客户 | 关联合同作废为失效，且该线索下已无其他已签约合同（BR-056，系统执行并写线索跟进记录） |
| 已成交 | — | 不可再次转客户；仍可录入跟进/更新阶段；不可公海回收 |

**关键规则：**
- **不自动转化：** 进入打样/报价/谈判等阶段仅表示"允许触发"（转客户规则），是否转化由人工主动执行；系统不因阶段变化自动创建客户。
- **转客户准入** = 待跟进/跟进中/无效/丢失（"已成交""已转客户""待分配"不可）；转化成功→"已转客户"（客户/客户联系人已沉淀、保留原线索）。
- **已成交**由「录入已签约合同且关联本线索」驱动（BR-056），不由转客户动作直接产生；原线索即使尚未转客户，关联合同后也可直接置已成交。
- **合同失效回退：** 仅当合同有关联线索、该线索当前=已成交、且该线索下已无其他已签约合同时，回退为已转客户；一线索多合同且仍有已签约合同则保持已成交；未关联线索的合同作废不影响任何线索；只动本条关联线索。
- **公海回收：** 手动回收适用除"已成交"外全部状态（待跟进/跟进中/已转客户/无效/丢失）；**【二期/延后】** 自动回收（超时未跟进）仅针对待跟进/跟进中（MVP 不做，见 §10.7）；"已成交"不回收。
- **公海回收 ≠ 状态回退：** 已转客户经手动回收进入"待分配"是**异常回收机制**（数据回到公海重新分配，见 §10.7），不属于跟进链路回退。
- 已成交/已转客户录入跟进仅追加记录（已成交回退除外由系统改状态）；转客户为必然成功动作，异常按系统异常处理（BR-033）。

### 26.2 合同状态机

**状态值：** **已签约**、**失效**。录入默认「已签约」，无「待签约 / 执行中 / 已完成 / 已终止 / 已作废」。无「合同结束时间」字段。

| 当前状态 | 允许流转至 | 触发 |
| :--- | :--- | :--- |
| 已签约 | 失效 | 更多→【作废】；或【编辑合同】将状态改为失效（均须填原因） |
| 失效 | 已签约 | 【编辑合同】将状态改回已签约（关联线索按 BR-056 置已成交） |

> 改为失效须填原因；合同中心不提供删除（BR-050）。线索联动见 BR-056、§16.6.2、§16.6.3。编辑时合同编号 / 客户 / 关联线索只读，其余字段可改（§16.6.2）。

### 26.3 客户潜质分级（设计说明）

**CRM 定位为客户资料库，客户不设独立"客户状态"字段**（一客户可绑多线索，各线索状态独立，不宜聚合成单一客户生命周期状态）；**系统不设"潜在客户/合作中/流失"等客户生命周期状态**。

经营情况由 **客户潜质分级** 反映：

- **字典维护：** 取值来自字典「客户域 → 客户潜质分级」（§23.5，如可跟进 / 潜在 / 一般客户）；选项由系统管理员在字典管理中维护，业务表单以下拉引用，不硬编码写死。
- **仅人工维护：** 在【新建客户】/【编辑客户】中由业务人员选择或调整；**系统不因线索状态、合同签约、跟进行为等自动变更**。
- **入口：** 仅客户新建/编辑表单中的「客户潜质分级」字段；不单列独立「调整潜质」操作。
- **潜质分级 ≠ 客户状态：** 仅表示经营价值/跟进优先级判断，不代表客户生命周期阶段。
- **不展示「关联线索状态分布」：** 客户详情不以线索状态计数作为客户经营标识；线索过程仍可在「线索记录」页签按条查看。

### 26.4 站点状态机

启用 ↔ 停用。站点不允许物理删除仅停用（BR-009）；停用停止接收新消息、暂停公海自动回收，历史数据保留、已有业务可继续；停用站点在筛选下拉仍可见（带「已停用」）。连锁影响见 20.6。

### 26.5 用户状态机

启用 ↔ 停用。用户不允许物理删除仅停用（BR-004e）；停用后无法登录、名下数据保持原状态；停用用户待跟进/跟进中线索超时按公海回收释放，可手动重新分配/转移；不可作为分配/转移目标。连锁影响见 23.2。

> 消息不设独立状态机：消息接收 → AI 识别有效询盘 → **有效则自动生成线索（主流程）**；AI 不可用/无法识别时人工兜底生成；无效消息不生成。相关规则见 §21.8、§27.1、第 28 章。

## 27. 数据流转与同步规则

> 本章为**业务对象生命周期与数据流转唯一权威定义**（消息→线索→客户→合同 的创建来源、创建条件、归属规则、同步规则、状态变化影响）。业务模块只描述入口与页面行为，数据流转以本章为准；AI 字段级同步以 §21.4 为准。

### 27.1 线索生成与归属

- **生成时序（主流程）：** 消息 → **AI 分析识别有效询盘**（标准见 §21.8）→ 自动提取业务信息 → **自动创建线索** → 绑定负责人 → 进入线索中心；无效消息不生成线索。AI 不可用时消息保留、转人工兜底创建（见 §21.5、§21.8）。
- **站点归属口径：** 站点（`site_id`）= 线索数据归属、权限判断、经营/获客分析的统一依据；**线索创建即有明确站点**。
- **负责人归属（两类邮箱）：** ① **业务员个人邮箱**线索负责人 = **业务员本人**，直接进入本人线索列表；② **站点主邮箱（运营人员业务邮箱）**线索负责人 = **该站点绑定的运营人员**，由运营人员点击【转派给业务员】一步完成投递与负责人变更（T1–T3、BR-053，见 §7.5.2）、不进公海；WhatsApp 线索负责人 = WhatsApp 绑定业务员；人工兜底生成归属当前操作人（BR-008）。
- **负责人异常兜底：** 站点主邮箱对应运营专员**已停用/缺失**、业务员个人邮箱对应业务员停用、或 WhatsApp 绑定业务员**已停用/不存在**时，**线索仍然创建**（负责人为空、状态"待分配"、进入公海池），由运营专员 / 系统管理员分配（协同人只读不可）；**禁止因负责人无法匹配导致线索创建失败**。

**邮件线索 — 业务员个人邮箱（AI 自动生成）：**

```text
来源渠道 = 邮件
站点 = 该邮箱对应站点（业务员唯一授权站点）
负责人 = 业务员本人
线索状态 = 待跟进
进入位置 = 本人线索列表（无需转派）
```

**邮件线索 — 站点主邮箱（运营人员业务邮箱，AI 自动生成）：**

```text
来源渠道 = 邮件
站点 = 站点主邮箱对应站点（mail_site_id，新增/编辑站点时绑定运营人员选定其业务邮箱）
负责人 = 该站点绑定的运营人员（运营点击【转派给业务员】一步完成投递与负责人处理，T1–T3，不进公海，BR-053）
线索状态 = 待跟进
进入位置 = 线索列表
```

**WhatsApp 线索（AI 自动生成，仅业务员）：**

```text
来源渠道 = WhatsApp
站点 = 业务员唯一授权站点
负责人 = WhatsApp 绑定业务员
线索状态 = 待跟进
进入位置 = 线索列表
```

- 邮件：每封邮件独立生成线索（BR-021，**入库不做按发件邮箱合并**）、同封多需求合并（BR-023）；关闭 AI 由人工维护；无效消息可人工转线索（BR-008/BR-038）。站点未绑定运营人员/主邮箱时其邮件不参与自动生成线索。转派参考匹配键（站点 + 客户原始发件邮箱）**代码写死**，仅用于 T2/T3 选人提示，见 §7.5.2；不进系统参数页。
- WhatsApp：仅业务员可绑（见 §24.3.2），**不绑定固定站点**；线索站点 = 业务员唯一授权站点，创建即明确、不进公海池。

### 27.2 AI 信息同步

详见 §7.9、§21.4。客户画像落询盘联系人、首次/存量/人工优先（BR-016c）；转客户时客户无画像才写入、有则不覆盖；国家/地区与所属行业从工商画像预填客户标准字段（新建预填、关联仅补空，字典匹配不上留空，可人工改）（BR-036）；意向总结不进线索/客户；客户仅人工标签；年营收不写入客户标准字段（BR-017）。

### 27.3 联系人归集（BR-035）

**询盘联系人 → 客户联系人**：手动关联线索时归集；**转客户动作触发时同步归集**（创建客户联系人）。询盘联系人转客户成功后，归集为该客户下的客户联系人；若客户名称来源于询盘联系人姓名，该联系人仍作为客户联系人保存，**不**额外创建独立个人客户联系人对象。保持：客户联系人必须归属于一个客户；不允许独立联系人；不允许跨客户共享联系人。

1. **去重边界：** 仅同一客户内去重；跨客户（含跨站点）不去重；客户匹配范围限当前线索站点（BR-034，按站点 + 客户名称精确匹配客户）。**一个客户可拥有多个联系人**。
2. **匹配字段（关联 or 新建）：** 关键资料（邮箱 / 手机号 / 姓名）与客户已有联系人**一致**则**自动关联已有联系人**；关键资料**存在冲突（不一致）**则**创建新的客户联系人**；关键资料均缺失直接新增。
3. **合并策略：** 关联已有时合并（以客户已有为主，补充缺失字段不覆盖非空）；无匹配或关键资料冲突则新增；归集的客户联系人保留「AI 识别」标记（只读）。
4. **主要联系人：** 归集不自动变更客户已有主要联系人，新归集默认非主要。
5. **询盘联系人影子对象（BR-012）：** 消息进入时创建询盘联系人影子对象（尚未归集为客户联系人），已有同源则关联；手动关联线索时按上述规则归集为客户联系人。

### 27.4 客户负责人变更同步

- 变更后客户后续新生成线索自动分配新负责人，历史线索不变。
- 系统参数"客户负责人变更是否同步线索负责人"：是→新线索归新负责人；否→新线索按原归属规则；无论取值历史线索不变。

### 27.5 合同归属

新增合同负责人默认当前用户；系统参数"新增合同自动归属客户负责人"开启时默认合同业务员=客户当前负责人。

### 27.6 回复状态与待回复判定

| 维度 | 适用渠道 | 粒度 | 含义 | 工作台卡片 |
| :--- | :--- | :--- | :--- | :--- |
| 已读/未读 | 仅邮件 | 单封邮件 | 是否查看过 | 邮件未读 |
| 消息回复状态（已回复/未回复） | 消息（邮件消息 + WhatsApp 消息） | 逐条消息（邮件按封、WhatsApp 按会话实现） | 是否回复过 | 待回复线索 |

- 两维度独立："已读"≠"已回复"。
- **消息回复状态**（BR-019b）：按单条消息维护，默认"未回复"；首次回复发送成功后**仅更新当前消息**为"已回复"，不影响同一线索下其他未回复消息。
- 待回复判定（BR-019c）：线索关联消息存在≥1条回复状态为"未回复"且线索非终态即计入；与下次跟进时间无关；所有消息已回复或线索终态则清除。
- 与跟进超时区别：待回复基于关联消息未回复（与时间无关），跟进超时基于下次跟进时间<今日（与消息回复状态无关）。

### 27.7 统一取数口径（最近签约合同 / 成交金额）

适用：客户列表「最近签约合同」、客户详情合作合同统计、分析中心外销合同数/合同金额/已成交客户数/站点成交金额。

- 筛选范围：合同状态=**已签约**（**排除失效**）。
- 排序：按签约时间倒序。
- 取数：最近 1 份（「最近签约合同」）或全部累计（「合同金额 / 外销合同数」）；已成交客户数=周期内至少有 1 份已签约合同的客户去重。
- 边界：无任何合同或全部为失效→「—」；多份则取签约时间最近一份。

### 27.8 时间范围筛选统一交互规则

适用于所有列表页/管理页/日志页/分析页的时间范围筛选项（各页面具体时间字段见对应章节）。

1. 支持开始时间和结束时间两个字段。
2. 允许只选择开始时间（不限制结束）。
3. 允许只选择结束时间（不限制开始）。
4. 开始时间不能晚于结束时间（前端校验，违例提示"开始时间不能晚于结束时间"）。
5. 时间为空时表示不限制该时间条件（不参与过滤）。
6. 时间比较口径：开始时间含当天起始、结束时间含当天结束（按日期粒度，YYYY-MM-DD）。
7. 点击重置时恢复该页面默认时间范围（见各页面说明）；**重置不改变用户权限范围、不改变站点/角色权限**。
8. 时间筛选只影响当前页面查询，不影响全局站点、用户权限、角色权限。
9. 时间范围展示格式与项目已有日期组件风格保持一致；API 参数格式与项目现有时间参数格式保持一致（YYYY-MM-DD）。
10. **不同业务含义的时间字段不能混用**（创建时间、更新时间、入池时间、签约日期、跟进时间、下次跟进时间、最近跟进时间、邮件时间、最近消息时间、登录时间、操作时间、变更时间、统计时间各自独立）。

**默认值建议（按业务合理性设置，不强行统一）：**

| 页面类型 | 默认时间范围 |
| :--- | :--- |
| 工作台 / 分析中心 | 本月（客户经营为本年，见 §19.2） |
| 线索列表 | 创建时间=本月，其余时间字段=不限制 |
| 公海池 | 入池时间=本月 |
| 跟进日志 | 跟进时间=本月，下次跟进时间=不限制 |
| 邮件 / WhatsApp | 不限制 |
| 系统日志 | 近 7 天 |
| 客户 / 合同 | 创建时间=不限制；合同签约日期=本月 |
| 站点 / 用户 / 字典管理 | 不限制 |

## 29. 异常处理与兜底规范

> 本章为全系统**异常处理唯一权威定义**（空态、加载、错误、权限、并发、系统级状态页、AI 异常、删除、审计、合规）。业务模块只引用本章，并仅保留"该模块发生异常时的业务影响与特殊处理"。

### 29.1 通用空态

列表无数据显对应空态占位图（"暂无线索/客户/合同"等）；搜索无结果"未找到匹配的 XX，请调整筛选条件后重试"；指标卡/数值无数据"--"；图表无数据保留坐标系显"暂无数据"。

### 29.2 通用加载态

列表首次加载 5 行骨架屏；筛选变化 3 行骨架；全局路由切换顶部进度条（不阻挡）；表单骨架；异步任务提交按钮 Loading + Toast"任务已提交"。

### 29.3 通用错误态

加载失败/网络异常/服务端 5xx："数据加载失败，请刷新重试" + 【重新加载】（5xx 可显"系统繁忙，请稍后再试"）；表单提交失败 Toast 具体原因并保留内容；接口超时"请求超时，请稍后重试"。

### 29.4 权限校验失败

无菜单权限→403 页面；无按钮权限→按钮置灰/隐藏，拦截时 Toast"您没有该操作权限"；数据权限越界→列表自动收敛不展示越权数据。

### 29.5 并发与冲突

公海线索并发分配"该线索已被其他操作分配，请刷新后重试"；变更负责人新负责人已停用阻止并提示；合同编号重复拦截；同一站点内客户名称相同客户拦截；站点主邮箱重复绑定拦截（见 §20.5.3、§29.11）。

### 29.6 系统级状态页

系统级状态页唯一权威定义（各模块只引用，不再单列）：

| 状态页 | 展示与操作 |
| :--- | :--- |
| 404 | 页面不存在 + 返回首页 |
| 403 | 无权限 + 返回首页（隐藏"返回上一页"） |

### 29.8 数据删除规则（软删除唯一权威定义）

### 30.5 对象规则与异常流程（引用）

以下为已确认规则的统一固化，细则以各唯一来源为准，本节不重复定义：

- **客户创建方式：** ① 线索转客户（§13.6）；② 客户中心直接新建客户（§14.7.1）。
- **AI 分析失败（不阻断业务）：** AI 成功→自动识别有效询盘并自动创建线索；AI 不可用/失败→消息保留→进入人工处理→人工创建线索；AI 仅辅助不阻断（见 §21.5、§21.8、§29.7、BR-015）。
- **客户匹配异常：** 生成线索时匹配到已有客户仅提示、不阻止创建、不创建重复客户，转客户时关联已有客户（见 BR-034、§13.6）。
- **联系人匹配异常：** 按姓名 + 关键联系资料一致自动匹配归集；任一关键资料不一致则新建联系人（见 BR-035、§27.3）。
