## 2. 产品整体说明

### 2.1 产品定位

面向销售团队的客户关系管理平台，围绕「消息接入 → 线索识别 → 跟进转化 → 客户沉淀 → 合同成交 → 经营分析」闭环，为业务员、运营专员、协同人、系统管理员提供分层协作能力。AI 作为平台能力嵌入各业务场景，不可用时系统按规则正常运行。

### 2.2 核心目标

1. 邮件 / WhatsApp 双渠道消息统一接入，避免线索遗漏。
2. 线索全生命周期管理（识别、归属、跟进、转客户、回收）。
3. 客户档案与联系人持续沉淀，合同录入与成交追溯。
4. 站点级隔离配置（公海规则、AI 规则）+ 多层级数据权限。
5. 销售经营、获客、客户经营三维经营分析。

### 2.3 业务边界

- **覆盖：** 邮件/WhatsApp 沟通、线索管理、客户/联系人/合同管理、经营分析、站点配置、系统管理。

### 2.4 对象关系

**核心业务对象定义（术语唯一权威）：**

| 对象 | 定义 | 关键特点 |
| :--- | :--- | :--- |
| 企业（Company） | 客户所属的工商主体（企业名称/工商信息识别得到）| **逻辑分组维度，无独立管理菜单**；企业信息沉淀自线索所属企业；一个企业可在不同站点存在多个客户档案 |
| 询盘联系人 | 来自邮件/WhatsApp 等渠道主动发起沟通的人 | 是线索来源；**不属于客户资产**；可未识别所属企业 |
| 所属企业 | AI 或人工根据询盘联系人信息识别出的企业信息 | 属**线索阶段信息**；转客户时作为客户所属企业（企业ID）沉淀 |
| 线索 | 销售机会对象 | 包含：询盘联系人、所属企业、询盘内容、AI 分析结果、跟进记录；**无线索名称字段，展示用线索编号** |
| 客户 | 已确认进入 CRM 客户管理体系的业务主体 | 客户资产的归属主体；归属一个企业（企业ID）与一个所属站点；**（所属站点 + 企业）唯一**；可拥有多个历史线索（客户 1:N 线索），线索转客户后保留历史关联 |
| 客户联系人 | 客户主体下维护的联系人 | 来源：线索转客户/关联时，**询盘联系人转换/归集为客户联系人**；**必属于且仅属于一个客户**（禁止独立联系人、禁止跨客户共享） |

**对象层级（唯一权威）：** 企业（Company）→ 客户（Customer）→ 客户联系人（Contact）；询盘联系人 → 线索（Lead）→ 关联客户（Customer）→ 客户联系人（Contact）。

```
企业 1:N 客户（同企业跨站点可多档案；站点+企业 唯一确定一个客户）；客户 N:1 企业
站点(所属) 1:N 【邮件/whatsapp 会话】 / 线索 / 客户 / 客户联系人
线索 1:1 询盘联系人；线索 N:1 所属企业（线索阶段信息）
线索 N:1 客户（一线索至多关联一个客户）；线索 N:1 所属站点 / 来源站点；线索 1:N 【邮件/whatsapp 会话】 / 跟进记录 / 合同
客户 1:N 客户联系人 / 合同 / 线索
询盘联系人 --转客户/关联线索归集--> 客户联系人
合同 N:1 客户 / 线索（合同不设独立站点，继承客户所属站点）
```

**站点术语（唯一权威，全系统只引用不重复定义）：** 站点是 CRM 中**业务运营与数据隔离的基础单位**（用于用户业务归属、业务数据归属、消息渠道配置、AI 能力配置、业务规则配置），**不是组织架构，不扩展部门/区域/公司组织树**。

| 术语 | 字段 | 归属对象 | 含义与用途 |
| :--- | :--- | :--- | :--- |
| 所属站点 | `owner_site_id` | 线索 / 客户 / 合同 | 业务数据当前归属站点；**数据权限判断、业务/销售管理的唯一站点依据** |
| 来源站点 | `source_site_id` | 线索（客户来源经线索追溯） | 业务数据产生来源站点；**仅用于获客/来源/渠道分析，不参与权限判断** |
| 授权站点 | `user_site_permissions` | 用户 | 用户拥有访问权限的站点范围；属**权限模型**，非业务字段 |
| 绑定站点 | `bind_site_id` | 邮箱 / WhatsApp 账号 | 渠道账号归属站点；作为该账号生成线索的**来源站点**来源 |

> **关键口径：** ① 权限只基于**所属站点**，来源站点绝不参与权限；② 线索 `owner_site_id` 默认 = `source_site_id`，MVP 无站点转移流程（变更负责人/公海分配均不改站点），二者相等；③ 合同继承客户所属站点；④ "当前站点"仅表示操作上下文或"该记录所属站点"，不单独存储。

## 4. 角色与权限体系

> 本章为全系统**权限体系唯一权威定义**。数据权限范围 = **角色 + 管理关系 + 站点归属 + 团队（未来可选）**（角色、数据范围、菜单权限、操作权限、站点权限均以本章为准）。不同角色的数据范围**来源不同**（业务员=本人；运营专员=负责站点；协同人=经管理运营专员间接获得的区域站点范围；系统管理员=全部）。业务模块的数据权限统一引用 §4.2，仅保留模块特殊权限（如公海池分配）。

### 4.1 角色定义

| 角色 | 职责 |
| :--- | :--- |
| 业务员 | 一线销售执行人员，负责本人线索/客户/合同的跟进与转化（本人数据，不受团队影响）；可查看/新建/编辑本人数据、录入跟进、推进销售流程；不管理其他业务员及其他站点数据 |
| 运营专员 | 站点业务运营管理角色，管理其**负责站点**（当前版本=授权站点）范围内的业务数据（客户/线索、线索分配、销售过程、站点经营数据）；未来启用团队后按所属团队管理 |
| 协同人 | **区域/多站点管理角色**，管理其所辖运营专员及对应负责站点（当前版本经**多站点授权**表达区域范围）；在管理范围内具备业务数据**管理权限**（查看、协调线索分配、管理业务推进、查看区域经营数据）；**禁止**管理范围外数据、系统级配置与全局权限配置 |
| 系统管理员 | 全公司系统与数据管控（全部站点 + 全部团队 + 全部数据），负责站点/用户/规则/AI/系统配置 |

### 4.2 数据权限粒度

数据权限由**角色、管理关系、站点归属、团队（未来可选）共同确定**；**当前版本团队管理未启用，以授权站点作为主要管理范围**，未来启用团队后运营专员改按所属团队计算，其余角色行为不变。适用范围：线索/客户/合同/跟进记录。

| 粒度 | 角色 | 判定规则（当前版本，团队未启用） | 未来启用团队 |
| :--- | :--- | :--- | :--- |
| 仅本人 | 业务员 | 本人为负责人，且数据所属站点在授权站点内 | 不受团队影响，仍为本人数据 |
| 负责站点 | 运营专员 | 负责站点（=授权站点）内所有业务员数据（含本人） | 所属团队内业务员数据 |
| 本区域 | 协同人 | 区域授权站点内所有运营专员/业务员数据（区域范围=所辖运营专员负责站点并集，经多站点授权表达） | 授权站点范围内数据（跨站点管理，保持不变） |
| 全部 | 系统管理员 | 全公司所有站点数据，不受授权站点限制 | 全部站点 + 全部团队 + 全部数据 |

- **站点定义：** 业务运营与数据隔离基础单位，用于邮件/线索/客户/合同的数据归属（术语权威见 §2.4）。
- **权限唯一站点依据：** 数据可见性 = 数据「所属站点」(`owner_site_id`) ∈ 用户「授权站点」(`user_site_permissions`) + 角色/团队粒度；**「来源站点」仅用于分析，绝不参与权限判断**。
- **团队定义：** 团队是**销售管理单元**，仅用于销售数据管理范围扩展；**不是**部门/区域/公司组织树等组织架构。团队为可选权限维度，当前版本仅预留字段，不提供团队管理菜单/新增页面/维护流程/配置后台。
- **角色定义：** 决定用户可执行的操作（查看/新增/编辑/分配/导出/配置），见 §4.4。
- **团队为空兜底规则：** 用户未配置所属团队时，系统按授权站点计算数据权限。
- **管理关系口径（MVP 落地）：** 协同人对运营专员的"管理关系"在当前版本**经多站点授权表达**——将协同人授权至其所辖运营专员的负责站点集合，即获得区域管理范围；**不新增"协同人-运营专员"映射表、不新增组织架构/部门/区域管理模块**；未来可引入显式管理关系或团队维度，届时行为向后兼容。
- **协同人统一口径（管理角色）：** 协同人为**区域管理角色**，在其管理范围（区域授权站点）内的线索/客户/合同/跟进日志具备**管理类操作权限**：查看、录入跟进、编辑、分配、协调调整负责人、标记异常、转客户、导出；**不含新建**（新增线索/新建客户由业务员/运营专员执行）。**管理范围外数据不可见/不可操作**；**系统级配置（站点/用户/角色/规则/AI）禁止**。各模块具体操作以模块权限矩阵为准（见 §13.1.2 等）。
- **公海池数据权限适用独立规则**（业务员按授权站点只读、运营专员/系统管理员/协同人可分配），详见第 10 章。

### 4.3 菜单权限

| 角色 | 可见菜单 |
| :--- | :--- |
| 业务员 / 运营专员 / 协同人 | 工作台、沟通中心、线索中心、客户中心、分析中心、通知管理、个人中心¹ |
| 系统管理员 | 以上全部 + 站点中心、AI 能力管理、系统管理 |

> ¹ 个人中心入口在头像下拉，不占侧边菜单。三类业务角色菜单相同，差异在数据权限粒度与操作权限。
>

### 4.4 操作权限类型

新建、编辑、删除、分配、转客户、导出（CSV）、配置。各模块具体操作权限矩阵见对应章节。

### 4.5 站点权限

- 用户仅可查看已授权站点数据；站点筛选下拉仅展示授权站点（系统管理员为全部），**含「已停用」站点（带标记），可查看停用站点历史数据**。
- **业务员仅可被授权单个站点**；运营专员/协同人/系统管理员可授权多个站点。

## 5. 登录与认证

### 5.1 模块概述

- **定位：** 统一身份认证入口，登录成功后加载菜单/页面/操作/数据权限并进入对应工作台。
- **面向角色：** 全部角色。

### 5.2 页面结构


| 区域 | 内容 |
| :--- | :--- |
| 品牌区 | Logo + "AI 智能 CRM" |
| 登录方式 Tab | 账号密码登录 \| 钉钉扫码登录 |
| 表单区 | 账号输入框 + 密码输入框 + 记住账号复选框 |
| 操作区 | 【登录】按钮 + "忘记密码"链接（提示"请联系系统管理员重置密码"，不自助重置） |
| 底部 | 版权信息 |

### 5.3 登录方式与字段

#### 5.3.1 账号密码登录（默认）

| 字段 | 类型 | 必填 | 校验 / 错误提示 |
| :--- | :--- | :--- | :--- |
| 登录账号 | 文本 | 是 | 非空，"请输入登录账号" |
| 登录密码 | 密码（可显隐） | 是 | 非空，"请输入登录密码" |
| 记住账号 | 复选框 | 否 | 选中后下次自动填充账号（本地存储） |

#### 5.3.2 钉钉扫码登录

无需账号密码，依赖服务协议配置 Tab3 钉钉凭证发起 OAuth。

### 5.4 业务规则

- 登录流程：校验账号存在 → 校验密码 → 校验账号状态 → 生成 Token(JWT) → 初始化权限 → 记录登录日志 → 进入工作台。
- 钉钉扫码：已绑定→自动登录；未绑定→提示"当前钉钉员工未绑定系统用户，请联系系统管理员"；账号停用→禁止登录。
- 登录成功初始化：用户信息（ID/姓名/角色/站点）+ 菜单/页面/操作/数据四类权限 + 按角色跳转工作台。
- 退出登录：清 Token/Session → 销毁前端状态 → 重定向登录页；无二次确认，有未保存编辑时轻提示。
- 会话：登录创建 Session + 返回 JWT，每次请求携带 Token 鉴权；Session 仅用于日志与超时。
- 登录超时：默认 1440 分钟（系统参数可配），超时弹"您的会话已超时，请重新登录"并跳登录页。
- 同时登录：允许多设备同时登录。
- 密码策略：初始密码由系统管理员分配（默认 123456，首次登录不强制改）；长度 ≥ 8 位可配；字符类型可配。
- 登录日志：记录时间、账号、IP、方式、结果（含失败原因）、浏览器、操作系统（BR-055）。

### 5.5 登录失败提示

| 场景 | 提示 |
| :--- | :--- |
| 用户不存在 / 密码错误 | 用户名或密码错误（统一提示，防账号枚举） |
| 用户停用 | 当前账号已停用，请联系系统管理员 |
| 钉钉未绑定 | 当前钉钉员工未绑定系统用户，请联系系统管理员 |
| 网络异常 | 网络连接异常，请稍后重试 |

### 5.6 MVP 范围

- **支持：** 账号密码登录、钉钉扫码登录、登录日志、用户改密、系统管理员重置密码、登录超时退出、记住账号。

### 7.5 邮件正文（中栏）

#### 7.5.1 基础信息

#### 7.5.2 操作按钮

| 操作 | 前置条件 | 结果 |
| :--- | :--- | :--- |
| 生成线索（人工兜底）| 邮件未关联线索（AI 未自动生成 / AI 不可用 / 无法识别时使用，非主流程）| 弹确认弹窗（预填企业名称、联系人、邮箱、电话、whatsapp、来源站点、来源渠道、意向产品，支持业务修改，并补充线索标签），确认后按 BR-008 归属当前业务员及站点；已关联则提示线索编号并跳转 |
| 回复 | — | 打开写邮件页签，带入原文、收件人、主题（前缀"Re:"）。 |
| 转发 | — | 带入原文与附件，收件人留空，主题前缀"Fwd:"，不影响原邮件回复状态 |
| 删除 | — | 移至垃圾箱，已生成线索不同步删除 |

### 7.6 写邮件页签

#### 7.6.1 字段

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 发件人 | 下拉 | — | 已绑定邮箱，默认当前；回复/转发选原邮件邮箱 |
| 收件人 | 文本 | 条件 | 多个以分号分隔；发送时收件人/主题/正文至少一项非空 |
| 抄送 / 密送 | 文本 | 否 | 默认收起，多个以分号分隔 |
| 主题 | 文本 | 否 | 回复"Re:"、转发"Fwd:"前缀 |
| 正文 | 富文本 | 否 | 支持基本排版 |
| 附件 | 上传 | 否 | 单附件≤100MB，数量上限9 个，格式不限；展示文件名/大小/图标，可移除；失败可重传 |

#### 7.6.2 发送 / 保存 / 取消规则

- **发送：** 收件人/主题/正文至少一项非空 → 发送 → Toast"发送成功"并关闭；失败"发送失败，请重试"，保留内容。
- **保存草稿：** 收件人/主题/正文/附件任一非空 → 保存 → Toast"草稿已保存" → 关闭。仅手动保存，系统不自动保存。
- **取消：** 关闭页签，未保存内容不保留。
- 发送失败：网络异常保留内容；收件人格式错高亮"收件人邮箱格式不正确"；发件人异常提示切换邮箱；附件中断保留已传。

### 7.7 附件规则

展示于正文下方（文件名/大小/图标）；单附件≤100MB，数量上限9 个；点击文件名下载；仅收件人/发件人/抄送人可下载；系统日志记录"附件下载"。下载失败按链接失效/文件丢失/网络超时（可重试）/类型不支持分别提示。

### 7.8 批量 AI 提炼

- 触发：列表多选 → 【批量 AI 提炼】；仅同一邮箱、单次≤50 封；已读/未读均可、同封可重复提炼。
- 弹窗只读展示：已选邮件数、主题、发件人、发件时间、影响范围（"分析 N 封，预计 X 秒"）。
- 流程：确认 → Loading（不可重复提交）→ 生成 PDF 自动下载至本地（系统内不单独展示结果）；失败 Toast"AI 提炼失败，请重试"。

### 7.9 AI 智能分析面板（右栏）

AI 仅辅助，不阻塞业务。含：① AI 意向总结（100-200 字）② 意向产品（生成线索时写入线索字段）③ AI 企业情报库（识别询盘联系人的**所属企业**工商信息，属线索阶段信息；已存在客户显【查看已有客户档案】，按"当前站点+企业名称/邮箱域名"匹配；**匹配到已有客户仅提示，不阻止生成线索、不创建重复客户**）④ AI 参展资料 ⑤ AI 风险提醒 ⑥ AI 分析标签（本面板只读展示；标签产生与写入线索规则见 §25）。AI 字段同步规则见 §21.4、§25。
- **AI 企业信息沉淀：** ③企业工商信息、④参展资料、⑤风险提醒属询盘联系人资料沉淀，首次生成后复用、不重复生成/不覆盖；标签按追加策略。规则见 BR-016c、BR-016a、§21.4。

### 7.10 业务规则

- **AI 自动识别并生成线索（主流程，BR-019a/BR-020）：** 收到邮件 → AI 自动判定是否为有效询盘（标准见 §21.8）→ 有效则自动提取字段并**自动创建线索**（无需人工点击），无效不生成；每封邮件独立生成线索（一消息一线索 BR-021，同封多需求合并 BR-023）。
- **询盘联系人影子对象（BR-012）：** 首次收到发件人邮件时按姓名/邮箱自动创建询盘联系人影子对象（尚未归集为客户联系人），已有则关联不重复；归客户后可联动展示客户名；归集规则见 27.3。
- **生成线索归属（BR-008）：** 来源站点=邮箱绑定站点（系统自动判定），所属站点默认=来源站点；**自动生成线索归属"收件邮箱绑定且启用的用户"**（归属异常兜底见 §27.1）；**人工兜底生成归属当前业务员**；关闭 AI 或其他邮件可人工转线索。
- **回复状态（BR-019b）：** 逐邮件标记，默认"未回复"；该邮件首次回复发送成功后仅该邮件置"已回复"；用于工作台待回复判定（27.7）。

### 7.11 异常处理

- **AI 能力异常：** 引用《全局异常处理规范》§29.7；AI 不可用时**不自动生成线索**，邮件消息保留，转由**人工兜底创建线索**（【生成线索】字段不预填、由用户手动输入），不阻断业务（见 §21.5、§21.8）。
- **加载/网络异常：** 引用 §29.2、§29.3（邮件加载失败提示"邮件加载失败，请刷新重试" + 【重新加载】）。
- **空态（模块特有）：** 收件箱"暂无邮件"、已发送"暂无已发送邮件"、草稿箱"暂无草稿"、垃圾箱"垃圾箱暂无邮件"、搜索"未找到相关邮件"。

### 8.5 详情面板（右栏）

① AI 意向摘要 ② 意向产品（生成线索写入线索）③ AI 企业情报库 ④ AI 参展资料 ⑤ AI 风险提醒 ⑥ AI 分析标签（本面板只读展示；标签产生与写入线索规则见 §25）。③④⑤为询盘联系人资料沉淀，首次生成后复用、不重复生成（BR-016c）；标签按追加策略（BR-016a）。AI 字段同步见 §21.4。

### 10.6 操作按钮

#### 10.6.1 操作

| 操作 | 前置条件 | 结果 |
| :--- | :--- | :--- |
| 分配（单/批量） | 状态="待分配" | 见 10.6.3 分配逻辑 |
| 标记异常 | 非终态 | 标记无效/丢失，见 13.7；标记后仍保留在公海池列表，状态展示对应终态 |
| 查看详情 | 无条件 | 跳线索详情 |

#### 10.6.2 分配弹窗字段

单个（线索编号、线索联系人、来源站点[只读]、目标负责人[下拉必填]、分配备注[可选]）；批量（已选线索数、目标负责人、分配备注）。目标负责人可选该线索所属站点下状态"可用"的销售人员。

#### 10.6.3 分配逻辑
1. 校验目标负责人"可用"（停用/离职阻止，提示"该负责人已不可用，请重新选择"）。
2. 校验线索仍在公海池（并发已分配则"该线索已被其他操作分配，请刷新后重试"）。
3. 成功后：负责人更新为目标人 → 状态"待分配"→"待跟进" → 移出公海池 → 生成跟进记录（备注"由[操作人]从公海分配至[负责人]"）。
4. 无可用销售人员：下拉为空，提示"该站点暂无可用销售人员，请先在用户管理中配置"。
5. 批量部分失败：弹窗汇总成功/失败数量及原因。

### 10.7 公海回收规则

#### 10.7.1 三种回收来源（优先级高→低）

| 优先级 | 来源 | 说明 |
| :--- | :--- | :--- |
| 1 | 系统参数「未跟进自动回收天数」 | 系统级，对所有站点强制生效，站点配置不可覆盖 |
| 2 | 站点配置「公海规则」 | 仅系统参数未配置时生效 |
| 3 | 运营专员手动回收 | 兜底，不受系统参数/站点配置控制，随时可执行 |

#### 10.7.2 回收状态范围与固定规则

- **手动回收状态范围：** 除"已成交"外全部可手动回收（待跟进/跟进中/已转客户/无效/丢失）；仅"已成交"排除（成交已闭环，不再回收公海）。
- **自动回收状态范围：** 仅"待跟进""跟进中"（自动回收为"超时未跟进"驱动，仅对活跃跟进状态有意义）；已转客户、已成交、无效、丢失不自动回收。
- **系统固定规则：** 超时计算基准=最近跟进时间；回收后不允许再次分配给同一负责人。
- **回收动作（负责人清空）：** 线索进入公海执行 `owner = null`（清空负责人），**保留历史跟进记录与历史操作记录**；重新分配后由分配动作重新生成负责人（见 §10.6.3）。
- **自动回收执行：** 定时任务每日一次（如凌晨 2:00）；仅扫描启用站点（停用站点自动回收暂停）；线索状态待跟进/跟进中且最近跟进时间超阈值；回收后生成跟进记录（备注"系统自动回收至公海"）。

### 10.8 异常处理

引用 §29.1、§29.3、§29.5（并发冲突）。模块特有：无数据"公海池暂无线索"；分配相关校验与失败提示（目标负责人停用、并发已分配、批量部分失败、站点无销售人员）见 10.6.3。

### 13.5 打标签

- 单条弹窗：线索编号、当前标签（带删除）、添加标签（输入回车新增）、热门标签（当前站点 Top10，点击添加，见 25.2）。
- 批量弹窗：已选线索数、添加标签、热门标签；**仅批量新增，不支持删除**。
- 交互：中英文/数字/下划线，≤20 字；同一线索不可重复（"该标签已存在"）；单条删除悬停×二次确认。手动标签选项由字典（线索域→线索手动标签）维护，新输入同步写回字典。AI 来源、人工来源标签均可增删改。
- **AI 与人工规则：** 线索标签含 AI 与人工两类来源，AI 重新分析仅追加、不覆盖人工修改；来源与追加规则以 §25.2 为准（BR-016a、BR-017）。

### 13.6 转客户

- **定位：** 销售**主动**将线索沉淀为客户资产的业务动作（线索 → 客户 + 客户联系人 + 保留原线索），帮助销售快速完成"线索 → 客户资产"转换；与客户中心「变更负责人」完全隔离，不复用逻辑、不共用名称。
- **转客户规则（准入）：** 是否允许触发由**跟进阶段扩展属性「是否允许转客户」**（即转客户规则）决定（见 §23.5）；**仅"允许触发"，绝不自动转化**，最终是否转化由人工主动执行。**跟进阶段取值与"是否允许转客户"默认标记以字典 §23.5.4 为准（本模块不重复定义固定枚举）**。线索状态准入=待跟进/跟进中/无效/丢失（"已成交""已转客户""待分配"不支持；公海池"待分配"仅分配/标记异常）。单条：负责人本人/运营专员/协同人（管理范围内）；批量：运营专员/协同人（管理范围内）/系统管理员。
- **执行流程（点击【转客户】后）：**
  1. **创建客户：** 依线索「所属企业+企业信息」创建客户资料，客户所属站点=线索所属站点；按"企业名称+所属站点"精确匹配客户（0 条创建新客户；1 条关联；N 条关联最近创建的一条，不弹窗；BR-034）。客户类型支持企业 / 个体 / 个体工商户（流程口径，字段落地见后续版本）。
  2. **创建客户联系人：** 将询盘联系人转换/归集为客户联系人（归集去重见 BR-035、§27.3）。
  3. **关联原线索：** 保留原线索并挂接到客户（客户 ├ 客户联系人 └ 历史线索）。
  4. **修改线索状态：** 跟进中 → 已转客户（**不再直接置"已成交"**；"已成交"由后续关联合同签约驱动，见 §26.1）。
  5. **同步 AI 信息**至客户（同步/不同步字段范围见 §21.4、BR-036）→ 系统通知负责人。
- **系统记录（非跟进记录）：** **不强制填写跟进记录**；系统自动记录**客户转化事件**（示例："线索已转化为客户"｜操作人｜时间｜生成客户），属**系统操作记录**（写入操作日志 §23.10），不计入销售跟进记录。
- **结果：** 生成/关联客户 + 客户联系人 + 保留并关联原线索 + 线索状态→"已转客户"（状态流转见 §26.1）。转客户为必然成功动作，异常按系统异常处理（BR-033），不设业务失败分支。
- **线索生命周期（方案 B）：** 转客户后**线索继续存在并作为业务跟进对象**，可查看历史信息、新增跟进记录、继续维护业务沟通；**系统不自动删除或关闭线索**（不存在"转客户后线索生命周期结束"）。
- **转客户后规则边界：** 允许——查看历史信息 / 查看 AI 分析结果 / 新增跟进记录 / 继续记录客户沟通；禁止——**状态回退（已转客户→跟进中/待跟进）** / **返回正常跟进准入链路** / **再次转客户（已转客户不再作为转化准入状态）**。（注：已转客户仍可经公海回收机制进入待分配，属异常回收，见 §10.7、§26.1，非跟进链路回退。）

### 13.7 标记异常

- **入口：** 线索列表更多、详情顶部、公海池操作列。终态线索锁定，入口置灰。
- **字段：** 当前状态（只读）、异常类型（无效/丢失）、异常备注（必填）。
- **结果：** 状态变更为对应终态 + 生成跟进记录（备注"标记异常：[类型]-[备注]"）；公海池中标记后仍保留列表展示，不再支持分配。线索不物理删除（BR-037）。

### 13.8 变更负责人

- **定位：** 单条线索负责人 A→B；仅变更本条，不影响该客户其他历史线索及后续新线索归属。与公海分配不同（分配针对无负责人公海线索）。
- **字段：** 线索编号（只读）、客户名称（只读）、分配状态（只读）、所属站点（只读）、当前负责人（只读）、新负责人（下拉必填，该线索所属站点销售人员，停用不展示）、变更备注（必填）。
- **校验：** 新负责人非空"请选择新负责人"；不可与当前相同"新负责人与当前负责人相同，无需变更"；备注非空。并发已变更"线索负责人已被其他操作变更，请刷新后重试"。
- **结果：** 更新负责人 + 状态不变 + 生成跟进记录（备注"由[原负责人]变更至[新负责人]"）+ 提示"线索负责人已变更为[新负责人]"（BR-025/BR-028）。

### 13.9 关联邮件 / 关联 WhatsApp

- **关联邮件：** 跳转沟通中心→邮件，按线索关联消息 ID 过滤，顶部显"线索：[编号]-[名称]的邮件往来"，提供返回按钮；无关联邮件显"该线索暂无关联邮件"，关联消息不存在提示"关联消息记录不存在或已被清理"（BR-022 一线索可关联多消息）。
- **关联 WhatsApp：** 跳转沟通中心→WhatsApp，按会话 ID 自动定位选中并刷新详情；无会话显"该线索暂无关联 WhatsApp 会话"，不存在提示"关联的 WhatsApp 会话记录不存在或已被清理"。
- 若无关联消息，点击时跳转对应模块首页，不附加筛选。

### 21.2 AI 能力列表

### 21.4 AI 输出结构与同步

#### 21.4.1 输出字段（最小集）

意向总结、意向产品、企业信息、企业画像、参展资料、风险信息、AI 标签。

#### 21.4.2 同步关系

| 对象 | 同步字段 | 不同步 |
| :--- | :--- | :--- |
| 消息 | 意向总结、意向产品、企业信息、参展资料、风险信息、AI 分析标签 | — |
| 线索 | 企业信息、企业画像、参展资料、风险信息、AI 分析标签（写入/追加线索标签）、意向总结、意向产品 | — |
| 客户 | 企业信息、企业画像、参展资料、风险信息（转客户时） | 意向总结、意向产品、AI 分析标签/线索标签（客户仅人工标签） |

#### 21.4.3 关键规则

**AI 信息归属：** AI 生成的企业信息即线索阶段「所属企业」，随线索存储、不进客户中心；仅转客户时按 BR-036 同步为客户侧企业画像。转客户不同步意向总结/意向产品/线索标签（BR-036）；意向产品为线索正式字段但不同步至客户；年营收不写入客户标准字段；**AI 分析标签在线索识别有效时写入（追加）线索标签、不进客户**；AI 重新分析追加不覆盖、手动编辑优先级高于 AI（BR-016a、BR-017）；**企业工商信息/参展/风险为询盘联系人资料沉淀，首次 AI 分析生成后复用，AI 重新分析不重复生成（BR-016c）**。

### 21.5 AI 降级规则

AI 不可用/调用失败自动降级为规则引擎识别，业务不中断（BR-015）；关闭 AI 时邮件转线索由人工维护归属本人；AI 失败不影响 CRM 运行。**AI 属辅助能力，不阻断业务流程：AI 成功时自动识别有效询盘并自动创建线索、写入 AI 结果；AI 不可用/失败时消息保留、进入人工处理、由人工兜底创建线索（见 §21.8、§27.1）。**

### 21.6 业务关联（只读）

系统内置，不提供在线配置：邮件意向分析、WhatsApp 意向分析、AI 自动提取企业信息、批量 AI 提炼，均调用系统默认 AI 能力。总开关与各业务开关在系统管理→系统配置控制。

### 21.7 操作与异常

新增/编辑按 21.3 字段校验（必填、URL 合法、Key≥20、服务商+模型唯一）；启用/停用立即生效无二次确认；删除仅停用状态可删+二次确认，历史记录保留审计。API 端点连续 N 次调用失败自动标记异常，不影响其他能力，系统管理员可手动停用。AI 运行期异常的用户提示与降级引用 §29.7。

### 21.8 AI 有效询盘识别与自动生成线索（业务流程）

**AI 是线索生产入口。** 邮件/WhatsApp 收到消息 → AI 自动分析 → 识别有效询盘 → 提取业务信息 → 自动创建线索 → 绑定负责人 → 进入线索中心。人工创建线索为异常兜底（见 §21.8 第 4 点）。

1. **有效询盘识别标准（AI 判定）：**
   - **有效询盘：** 产品咨询、价格询问、采购需求、样品需求、合作咨询、明确业务意向。
   - **非有效消息（不生成线索）：** 自动回复、广告、无业务内容消息、垃圾信息。
2. **判断结果存储（方案 B，不落库）：** AI 判断过程**不持久化**；**不新增** AI 有效询盘标识 / AI 置信度 / AI 判断原因等字段；AI 仅输出最终业务结果（线索及其提取字段与 AI 分析信息），判定有效性仅作为流程分支依据。
3. **AI 自动提取字段：**
   - **联系信息：** 联系人姓名、邮箱、电话、WhatsApp 账号。
   - **企业信息：** 企业名称、国家/地区、企业基本信息。
   - **业务信息：** 意向产品、产品需求、采购描述。
   - **AI 分析信息：** AI 意向总结、AI 标签、企业情报、风险提示、参展数据（字段与同步口径见 §21.4、§25，不重复定义）。
4. **AI 失败兜底：** AI 成功→自动创建线索；AI 不可用/失败→消息保留→进入人工处理→人工创建线索（见 §21.5）；**AI 失败不阻断业务**。人工创建线索仅用于：AI 服务不可用、AI 无法识别、历史数据补录、特殊业务场景。
5. **负责人归属与异常：** 归属规则与"收件邮箱/账号未绑定、用户停用、用户不存在"的待分配兜底见 §27.1、BR-020。

### 23.4 菜单管理（MVP 预置）

## 25. 标签体系

> 本章为全系统**标签体系唯一权威定义**。人工标签选项由字典管理维护（见 §23.5），AI 分析标签不进字典；业务模块只引用不重复定义。全系统标签统一为**三层**：AI 分析标签（AI 输出）→ 线索标签（业务使用）→ 客户标签（客户资产，人工独立）。

### 25.1 标签分类总览

| 标签类型 | 字段 | 归属/存储 | 来源 | 维护方式 | 同步规则 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| AI 分析标签 | `ai_analysis_tag` | 消息（邮件/WhatsApp）分析层，AI 面板展示 | AI 分析生成 | AI 生成（面板只读） | 线索识别有效时自动写入（追加）线索标签；不进客户；不进字典 |
| 线索标签 | `lead_tag` | 线索 | AI（来自 AI 分析标签）+ 人工 | 业务员增/改/删 | AI 重新分析追加不覆盖、不覆盖人工；转客户不写入客户 |
| 客户标签 | `customer_tag` | 客户 | 仅人工 | 客户管理增/删/改 | 独立维护；不继承线索标签；不受 AI 影响 |

> 人工标签（线索标签的人工部分、客户标签）选项由字典管理维护，业务侧新输入同步写回字典；AI 分析标签不进字典。全系统不单设"标签管理"菜单。

### 25.2 线索标签

- **来源（两类，混排为统一业务标签）：** ① AI 来源——AI 分析标签在线索识别有效时自动写入/追加（蓝底+圆角+AI 图标）；② 人工——灰底+圆角，选项来自字典线索域→线索手动标签。
- **AI 追加策略（BR-016a）：** AI 重新分析邮件/WhatsApp 时，新标签**追加**到线索标签；**不删除**已有标签；**不覆盖人工修改**过的标签（对齐 BR-017）。
- **人工维护：** 支持新增/修改/删除（AI 来源、人工来源标签均可增删改）；人工修改后视为业务标签，AI 不再覆盖。
- 展示：每线索最多 3 个，超出"+N"（悬停全部）；点击标签快速筛选；筛选区多选逻辑=OR。
- 热门标签：所属站点下使用频率最高 Top10，仅展示未被当前线索添加的；操作详见 13.5。
- 标签输入：中英文/数字/下划线，≤20 字，同线索不可重复。

### 25.3 客户标签

仅人工标记，无 AI 标签，选项由字典客户域→客户标签维护。**独立于线索标签：线索转客户不自动继承线索标签（BR-016b），且不受 AI 重新分析影响**。客户列表【打标签】与客户详情标签管理为同一份数据，实时双向同步。超 3 个"+N"，悬停全名。操作详见 14.7.4。

### 25.4 AI 分析标签（邮件 / WhatsApp）

- **定义：** AI 对消息（邮件/WhatsApp）内容分析输出的标签（如价格敏感、关注某产品、采购意向强、交期关注），属 AI 分析能力输出，展示于沟通中心 AI 面板（只读），不是长期业务管理标签。
- **写入线索（BR-016a）：** 线索识别有效时，AI 分析标签自动写入（追加）线索标签，成为线索标签的 AI 来源部分；AI 重新分析仅追加、不覆盖人工。
- **不作为客户资产标签：** 不直接进入客户中心、不进字典；客户标签始终人工维护。

## 26. 状态流转

> 本章为全系统**状态机唯一权威定义**（线索/合同/客户潜质/站点/用户）。业务模块只展示当前状态并引用本章，不重复描述状态流转。

### 26.1 线索状态机

> **线索状态机与跟进阶段分别维护、互不映射。** 线索状态描述线索的**业务生命周期**；跟进阶段（见 §23.5）描述销售**推进过程**；"转客户"是销售**主动动作**（见 §13.6），三者不混用。跟进阶段不映射为线索状态，也不自动改变线索状态。

**状态值：** 待分配（公海专用）、待跟进、跟进中、已转客户、已成交、无效、丢失。

**核心流转链路（单向不可回退）：** 待跟进 → 跟进中 → 已转客户 → 已成交。

**状态含义与操作角色：**

| 状态 | 含义 | 进入条件 | 操作角色 |
| :--- | :--- | :--- | :--- |
| 待分配 | 公海池中等待分配的线索 | 公海回收（自动/手动）或标记异常后保留公海 | 运营专员 / 协同人（区域范围）/ 系统管理员 |
| 待跟进 | 已归属负责人、尚未开始跟进 | 渠道生成线索 / 手动新建 / 公海分配 | 负责人本人 |
| 跟进中 | 负责人已开始跟进推进 | 首次录入跟进（BR-038a） | 负责人本人 |
| 已转客户 | 已沉淀为客户资产、线索保留 | 人工主动执行【转客户】成功（BR-038b/033） | 负责人本人 / 运营专员 / 协同人（区域范围） |
| 已成交 | 已签约成交（终态） | 已转客户关联合同签约（口径见 §27.7） | 由合同动作驱动 |
| 无效 / 丢失 | 异常终态（无跟进价值/流失） | 标记异常 | 负责人本人 / 运营专员 / 协同人（区域范围） |

> **禁止回退：** 正常跟进链路单向流转，禁止主动回退（如 已转客户 → 跟进中 / 待跟进、跟进中 → 待跟进）；**公海回收（→待分配）为异常回收机制、非链路回退**（见 §10.7）。数据权限口径见 §4.2。

| 当前状态 | 可流转至 | 触发动作 |
| :--- | :--- | :--- |
| 待分配 | 待跟进 | 公海池分配 |
| 待分配 | 无效 / 丢失 | 标记异常（保留在公海池列表展示） |
| 待跟进 | 跟进中 | 首次录入跟进（BR-038a） |
| 待跟进/跟进中 | 已转客户 | 人工主动执行【转客户】成功（BR-038b、BR-033） |
| 待跟进/跟进中 | 无效 / 丢失 | 标记异常 |
| 待跟进/跟进中 | 待分配 | 公海回收（自动/手动） |
| 已转客户/无效/丢失 | 待分配 | 公海**手动**回收（除"已成交"外均可，自动回收不触发） |
| 已转客户 | 已成交 | 关联合同签约（成交口径见 §27.7） |
| 已转客户 | 无效 / 丢失 | 标记异常 |
| 无效 / 丢失 | 已转客户 | 重新激活并执行【转客户】成功 |
| 已成交 | — | 终态，锁定不可回退；仍可录入跟进/更新阶段；不可再次转客户 |

**关键规则：**
- **不自动转化：** 进入打样/报价/谈判等阶段仅表示"允许触发"（转客户规则），是否转化由人工主动执行；系统不因阶段变化自动创建客户。
- **转客户准入** = 待跟进/跟进中/无效/丢失（"已成交""已转客户""待分配"不可）；转化成功→"已转客户"（客户/客户联系人已沉淀、保留原线索）。
- **已成交**由"已转客户"关联合同签约驱动，不由转客户动作直接产生。
- **公海回收：** 手动回收适用除"已成交"外全部状态（待跟进/跟进中/已转客户/无效/丢失）；自动回收（超时未跟进）仅针对待跟进/跟进中；"已成交"不回收。
- 终态录入跟进仅追加记录，不变更状态；转客户为必然成功动作，异常按系统异常处理（BR-033）。

### 26.2 合同状态机

**状态值：** 已签约、执行中、已完成、已终止、已作废。**录入即"已签约"，无"待签约"。**

| 当前状态 | 允许流转至 | 触发 |
| :--- | :--- | :--- |
| 已签约 | 执行中 | 编辑→状态 |
| 已签约 | 已作废 | 更多→作废 |
| 执行中 | 已完成 | 编辑→状态 |
| 执行中 | 已终止 | 更多→终止 |
| 执行中 | 已作废 | 更多→作废 |
| 已完成/已终止/已作废 | — | 终态 |

> "已签约"不可直接终止（终止仅限"执行中"）；作废需填原因，作废=逻辑删除、历史保留、不物理删除（BR-050、§29.8）。


### 26.3 客户潜质分级（设计说明）

**CRM 定位为客户资料库，客户不设独立"客户状态"字段**（一客户绑多线索，各线索状态独立无法聚合）；**系统不设"潜在客户/合作中/流失"等客户生命周期状态**。经营情况由：**客户潜质分级**（字典属性，取值见 §23.5）+ 关联线索状态分布（客户详情展示）共同反映。

- **客户潜质分级为客户经营标识**（如可跟进/潜在/一般客户等，取值以字典 §23.5 为准），由**业务人员人工维护、人工调整，不由系统自动变化**。
- **潜质分级 ≠ 客户状态**：潜质分级仅表示经营价值判断，不代表客户生命周期阶段。

### 26.4 站点状态机

启用 ↔ 停用。站点不允许物理删除仅停用（BR-009）；停用停止接收新消息、暂停公海自动回收，历史数据保留、已有业务可继续；停用站点在筛选下拉仍可见（带「已停用」）。连锁影响见 20.6。

### 26.5 用户状态机

启用 ↔ 停用。用户不允许物理删除仅停用（BR-004e）；停用后无法登录、名下数据保持原状态；停用用户待跟进/跟进中线索超时按公海回收释放，可手动重新分配/转移；不可作为分配/转移目标。连锁影响见 23.2。

> 消息不设独立状态机：邮件/WhatsApp 接收 → AI 识别有效询盘 → **有效则自动生成线索（主流程）**；AI 不可用/无法识别时人工兜底生成；无效消息不生成。相关规则见 §21.8、§27.1、第 28 章。

## 27. 数据流转与同步规则

> 本章为**业务对象生命周期与数据流转唯一权威定义**（消息→线索→客户→合同 的创建来源、创建条件、归属规则、同步规则、状态变化影响）。业务模块只描述入口与页面行为，数据流转以本章为准；AI 字段级同步以 §21.4 为准。

### 27.1 线索生成与归属

- **生成时序（主流程）：** 消息（邮件/WhatsApp）→ **AI 分析识别有效询盘**（标准见 §21.8）→ 自动提取业务信息 → **自动创建线索** → 绑定负责人 → 进入线索中心；无效消息不生成线索。AI 不可用时消息保留、转人工兜底创建（见 §21.5、§21.8）。
- **站点归属口径：** 来源站点(`source_site_id`)=生成时确定的产生来源站点；所属站点(`owner_site_id`)=数据归属与权限依据，默认=来源站点；**MVP 无站点转移流程**（变更负责人/公海分配均不改站点），二者始终相等；来源站点仅用于分析，不参与权限。
- **负责人归属（邮件收件业务员归属规则）：** 收件邮箱 → 匹配系统用户邮箱 → 绑定用户作为线索负责人（如 sales_japan@example.com → 张三，则线索负责人=张三）；WhatsApp 同理按账号绑定用户。自动生成归属该收件邮箱/账号绑定且启用的用户；人工兜底生成归属当前业务员（BR-008）。
- **负责人异常兜底：** 收件邮箱/账号**未绑定用户、用户已停用、用户不存在**时，**线索仍然创建**（负责人为空、状态"待分配"、进入公海池），由运营专员 / 协同人分配；**禁止因负责人无法匹配导致线索创建失败**。
- 邮件：来源站点=邮箱绑定站点，所属站点默认=来源站点；每封邮件独立生成线索（BR-021）、同封多需求合并（BR-023）；关闭 AI 由人工维护；无效消息可人工转线索（BR-008/BR-038）。
- WhatsApp：来源站点=账号绑定站点（个人中心维护，取授权站点其一，默认第一个），所属站点默认=来源站点；绑定站点变更仅影响后续新消息。

### 27.2 AI 信息同步

详见 21.4。AI 字段对消息/线索/客户的同步与不同步、客户仅人工标签、年营收不写入客户标准字段、手动编辑优先级高于 AI（BR-036/BR-017）。

### 27.3 联系人归集（BR-035）

**询盘联系人 → 客户联系人**：手动关联线索时归集；**转客户动作触发时同步归集**（创建客户联系人）。

1. **去重边界：** 仅同一客户内去重；跨客户（含跨站点）不去重；客户匹配范围限当前线索所属站点（BR-034）。**一个客户可拥有多个联系人**。
2. **匹配字段（关联 or 新建）：** 关键资料（邮箱 / 手机号 / 姓名）与客户已有联系人**一致**则**自动关联已有联系人**；关键资料**存在冲突（不一致）**则**创建新的客户联系人**；关键资料均缺失直接新增。
3. **合并策略：** 关联已有时合并（以客户已有为主，补充缺失字段不覆盖非空）；无匹配或关键资料冲突则新增；归集的客户联系人保留「AI 识别」标记（只读）。
4. **主要联系人：** 归集不自动变更客户已有主要联系人，新归集默认非主要。
5. **询盘联系人影子对象（BR-012）：** 消息进入时创建询盘联系人影子对象（尚未归集为客户联系人），已有同源则关联；手动关联线索时按上述规则归集为客户联系人。

### 27.4 客户负责人变更同步

- 变更后客户后续新生成线索自动分配新负责人，历史线索不变。
- 系统参数"客户负责人变更是否同步线索负责人"：是→新线索归新负责人；否→新线索按原归属规则；无论取值历史线索不变。

### 27.5 合同归属

新增合同负责人默认当前用户；系统参数"新增合同自动归属客户负责人"开启时默认合同业务员=客户当前负责人。

### 27.6 回复状态与待回复判定

| 维度 | 适用渠道 | 粒度 | 含义 | 工作台卡片 |
| :--- | :--- | :--- | :--- | :--- |
| 已读/未读 | 仅邮件 | 单封邮件 | 是否查看过 | 邮件未读 |
| 已回复/未回复 | 邮件+WhatsApp | 逐邮件/逐会话 | 是否回复过 | 待回复线索 |

- 两维度独立："已读"≠"已回复"。
- 回复状态（BR-019b）：默认"未回复"，首次回复发送成功后仅该消息置"已回复"。
- 待回复判定（BR-019c）：线索关联消息存在≥1条"未回复"且线索非终态即计入；与下次跟进时间无关；所有消息已回复或线索终态则清除。
- 与跟进超时区别：待回复基于关联消息未回复（与时间无关），跟进超时基于下次跟进时间<今日（与回复状态无关）。

### 27.7 统一取数口径（最近签约合同 / 成交金额）

适用：客户列表"最近签约合同"、客户详情合作合同统计、分析中心成交金额/成交客户/来源站点成交金额。

- 筛选范围：合同状态∈{已签约, 执行中, 已完成}（排除已终止、已作废）。
- 排序：按签约时间倒序。
- 取数：最近 1 份（"最近签约合同"）或全部累计（"成交金额"）。
- 边界：无任何合同或全部为已终止/已作废→"—"；多份则取签约时间最近一份。

无菜单权限→403 页面；无按钮权限→按钮置灰/隐藏，拦截时 Toast"您没有该操作权限"；数据权限越界→列表自动收敛不展示越权数据。

公海线索并发分配"该线索已被其他操作分配，请刷新后重试"；变更负责人新负责人已停用阻止并提示；合同编号重复拦截；同站点同企业客户拦截；站点编码重复拦截。

### 29.8 数据删除规则（软删除唯一权威定义）

**核心业务对象统一采用软删除（逻辑删除），任何情况下不物理删除。** 适用对象：线索、客户、客户联系人、合同。

| 对象 | 删除方式 | 删除后表现 | 关联规则 |
| :--- | :--- | :--- | :--- |
| 线索 | 软删除（关闭/标记异常，不提供删除入口） | 数据不可见，历史记录保留 | BR-037 |
| 客户 | 软删除（客户本身不可删，仅停用/注销标记） | 数据不可见，历史记录保留；关联处显"(已注销)" | BR-043 |
| 客户联系人 | 二次确认后逻辑删除 | 列表不再展示，历史关联显"-" | BR-035、§15.4.3 |
| 合同 | 作废=逻辑删除 | 列表不可见，历史记录保留 | BR-050、§16.6.4 |

**统一规则：** ① 删除后数据不可见但记录保留可审计；② 不做物理清除（消息永久保存见 BR-010，邮件草稿箱/垃圾箱 30 天清理为独立机制见 §7.4）；③ 站点/用户为停用非删除（BR-009/BR-004e）；④ 个人数据删除请求走匿名化（见 §29.10 GDPR）。

### 29.9 操作日志与审计

登录、业务操作、配置变更、导出、附件下载均记录；敏感字段脱敏（见 23.10）；日志保留 1 年。

### 29.10 GDPR 合规

预留数据匿名化能力：用户提出删除个人数据请求时，系统管理员可匿名化（脱敏姓名/邮箱/电话），保留业务数据完整性。MVP 预留。

## 30. 对象字段模型

> 本章为**核心业务对象字段的契约层数据字典**，供研发建模建表与测试对照使用；字段展示与交互以各业务模块为准，字段取值规则以对应唯一来源章节为准（状态见 §26，标签见 §25，字典见 §23.5，数据流转见 §27）。**所有核心对象统一软删除（见 §29.8）。**

### 30.1 线索 Lead

### 30.5 对象规则与异常流程（引用）

以下为已确认规则的统一固化，细则以各唯一来源为准，本节不重复定义：

- **客户创建方式：** ① 线索转客户（§13.6）；② 客户中心直接新建客户（§14.7.1）。
- **AI 分析失败（不阻断业务）：** AI 成功→自动识别有效询盘并自动创建线索；AI 不可用/失败→消息保留→进入人工处理→人工创建线索；AI 仅辅助不阻断（见 §21.5、§21.8、§29.7、BR-015）。
- **客户匹配异常：** 生成线索时匹配到已有客户仅提示、不阻止创建、不创建重复客户，转客户时关联已有客户（见 BR-034、§13.6）。
- **联系人匹配异常：** 按姓名 + 关键联系资料一致自动匹配归集；任一关键资料不一致则新建联系人（见 BR-035、§27.3）。
