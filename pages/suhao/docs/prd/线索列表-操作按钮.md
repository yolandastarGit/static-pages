## 一、编辑

### 1.1 操作概述

**操作名称：** 编辑

**功能定位：** 修改线索的基础信息，支持销售对 AI 识别结果进行人工修正。

**触发入口：**

| 入口 | 位置 | 前置条件 |
| :--- | :--- | :--- |
| 入口一 | 线索列表 → 操作列【编辑】按钮 | 单条线索操作 |
| 入口二 | 线索详情 → 顶部【编辑】按钮 | 当前线索详情页 |

**执行权限：** 当前线索负责人本人 或 销售主管（含以上）


### 1.2 弹窗字段定义

| 字段 | 组件类型 | 是否可编辑 | 说明 |
| :--- | :--- | :--- | :--- |
| 线索编号 | 只读展示 | 否 | 系统自动生成，不可修改 |
| 企业名称 | 文本输入 | 是 | 线索关联的企业名称 |
| 联系人姓名 | 文本输入 | 是 | 主要联系人姓名 |
| 联系人邮箱 | 文本输入 | 是 | 主要联系人邮箱 |
| 联系人电话 | 文本输入 | 是 | 主要联系人电话 |
| 来源站点 | 下拉选择 | 是 | 可选当前用户有权限的站点 |
| 来源渠道 | 下拉选择 | 是 | 官网询盘 / 自然询盘 / WhatsApp / 展会 / 客户转介绍 / 其他 |
| 意向产品 | 文本输入 / 下拉选择 | 是 | 支持手动输入或从已有产品类目中选择 |


### 1.3 弹窗交互说明

**打开弹窗：**
- 标题：`编辑线索`
- 所有字段自动加载当前线索数据
- 可编辑字段为白色输入框，只读字段为灰色背景

**提交校验：**

| 校验项 | 校验规则 | 错误提示 |
| :--- | :--- | :--- |
| 企业名称 | 不可为空 | “请输入企业名称” |
| 来源站点 | 不可为空 | “请选择来源站点” |

**提交成功：**
1. 线索信息更新
2. 若企业名称变更，系统**不自动重新匹配**客户，仅更新线索侧信息
3. 关闭弹窗，页面刷新，成功提示：`线索信息已更新`
4. 自动生成跟进记录：
   - 跟进方式：备注
   - 跟进内容：“[字段名] 由 [原值] 变更为 [新值]”（仅记录变更字段）
   - 跟进人：系统自动记录为操作人
   - 跟进时间：当前时间

**提交失败：**
- 提示：`更新失败，请稍后重试`
- 保留用户已填内容


### 1.4 编辑内容与 AI 的关系

| 场景 | 处理方式 |
| :--- | :--- |
| 用户手动修改字段 | 该字段后续不再被 AI 自动覆盖（BR-017） |
| 用户未修改字段 | 保持 AI 自动识别结果 |
| 用户将字段改回原值 | 视为用户手动修改，AI 不再覆盖 |

> 当用户手动编辑后，字段值被锁定，后续即使 AI 识别到新信息也不再自动覆盖，确保人工干预的有效性。


## 二、标记异常

### 2.1 操作概述

**操作名称：** 标记异常

**功能定位：** 将一条或多条线索标记为“异常”状态，用于处理识别错误、非目标客户、低质量线索等场景。

**触发入口：**

| 入口 | 位置 | 前置条件 |
| :--- | :--- | :--- |
| 入口一 | 线索列表 → 操作列【更多】下拉 → 【标记异常】 | 单条线索操作 |
| 入口二 | 线索列表 → 顶部【标记异常】按钮 | 选中至少一条线索 |

**执行权限：** 当前线索负责人本人 或 销售主管（含以上）

**执行结果：** 线索状态变更为“无效”


### 2.2 弹窗字段定义

#### 2.2.1 单条标记异常

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 线索编号 | 只读展示 | — | 展示当前线索编号 |
| 线索名称 | 只读展示 | — | 展示当前线索名称 |
| 异常类型 | 下拉选择 | 是 | 选项见下方说明 |
| 异常备注 | 多行文本 | 是 | 记录异常原因，便于追溯 |

**异常类型选项：**

| 选项 | 说明 |
| :--- | :--- |
| 识别错误 | AI 或规则误判，非有效商机 |
| 非目标客户 | 客户画像不符合目标市场 |
| 低质量线索 | 联系方式无效、无明确需求 |
| 重复线索 | 该客户已有活跃线索 |
| 已联系过 | 历史已跟进，无需重复创建 |
| 其他 | 其他原因 |

#### 2.2.2 批量标记异常

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 已选线索数 | 只读展示 | — | 展示已选中的线索数量 |
| 异常类型 | 下拉选择 | 是 | 所有选中线索共用 |
| 异常备注 | 多行文本 | 是 | 统一异常原因，所有选中线索共用 |


### 2.3 弹窗交互说明

**提交校验：**

| 校验项 | 校验规则 | 错误提示 |
| :--- | :--- | :--- |
| 异常类型 | 不可为空 | “请选择异常类型” |
| 异常备注 | 不可为空 | “请输入异常备注” |

**提交成功：**
1. 线索状态变更为“无效”
2. 线索自动移出公海池（若在公海中）
3. 关闭弹窗，页面刷新，成功提示：
   - 单条：`线索已标记为异常`
   - 批量：`成功标记 N 条线索为异常`
4. 自动生成跟进记录：
   - 跟进方式：备注
   - 跟进内容：“标记为异常：[异常类型] - [异常备注]”
   - 跟进人：系统自动记录为操作人
   - 跟进时间：当前时间


### 2.4 业务规则

| 规则编号 | 规则说明 |
| :--- | :--- |
| BR-037 | 线索不得物理删除，仅允许关闭、标记无效或丢失 |


## 三、关联邮件往来

### 3.1 操作概述

**操作名称：** 关联邮件往来（查看该线索的往来邮件）

**功能定位：** 快速查看当前线索关联的所有邮件往来记录，并跳转至邮件中心定位查看完整邮件内容。

**触发入口：**

| 入口 | 位置 | 前置条件 |
| :--- | :--- | :--- |
| 入口一 | 线索列表 → 操作列【更多】下拉 → 【查看邮件】 | 单条线索操作 |
| 入口二 | 线索详情 → 左栏【沟通记录】标签页 → 邮件筛选 | 当前线索详情页 |

**执行权限：** 当前线索负责人本人 或 销售主管（含以上）


### 3.2 交互说明

**入口一（从线索列表进入）：**

| 元素 | 说明 |
| :--- | :--- |
| 页面跳转 | 跳转至沟通中心 → 邮件页面 |
| 自动筛选 | 按当前线索关联的消息 ID 自动过滤，仅展示该线索的邮件 |
| 筛选条件展示 | 顶部显示当前筛选条件：“线索：[线索编号] - [线索名称] 的邮件往来” |
| 返回入口 | 页面提供【返回线索列表】按钮 |

**入口二（从线索详情沟通记录进入）：**

| 元素 | 说明 |
| :--- | :--- |
| 页面跳转 | 跳转至沟通中心 → 邮件页面 |
| 自动筛选 | 按当前线索关联的消息 ID 自动过滤 |
| 筛选条件展示 | 顶部显示“线索：[线索编号] - [线索名称] 的邮件往来” |
| 返回入口 | 页面提供【返回线索详情】按钮 |


### 3.3 跳转后邮件页面行为

| 行为 | 说明 |
| :--- | :--- |
| 列表过滤 | 仅展示该线索关联的邮件，邮件列表数据源为线索的关联消息 |
| 无邮件时 | 显示“该线索暂无关联邮件” |
| 点击邮件 | 右侧详情面板正常展示邮件完整内容 |
| 返回 | 点击返回按钮回到原页面 |


### 3.4 业务规则

| 规则编号 | 规则说明 |
| :--- | :--- |
| BR-022 | 一个线索可关联多个消息 |

## 五、打标

### 5.1 操作概述

**操作名称：** 打标

**功能定位：** 为线索添加或删除标签，支持用户自定义标签，也支持 AI 推荐标签供用户选择。

**触发入口：**

| 入口 | 位置 | 前置条件 |
| :--- | :--- | :--- |
| 入口一 | 线索列表 → 操作列【打标】按钮 | 单条线索操作 |
| 入口二 | 线索列表 → 顶部【打标】按钮 | 选中至少一条线索（批量打标） |
| 入口三 | 线索详情 → 右栏信息面板 → 标签区域【编辑】按钮 | 当前线索详情页 |

**执行权限：** 当前线索负责人本人 或 销售主管（含以上）


### 5.2 弹窗字段定义

#### 5.2.1 单条打标

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 线索编号 | 只读展示 | — | 展示当前线索编号 |
| 线索名称 | 只读展示 | — | 展示当前线索名称 |
| 当前标签 | 标签展示区 | — | 展示该线索已拥有的全部标签，每个标签带删除按钮 |
| 添加标签 | 文本输入 + 回车确认 | 否 | 用户手动输入标签，回车即新增 |
| AI 推荐标签 | 标签展示区 | — | 展示 AI 推荐的标签列表，点击即添加 |
| 热门标签 | 标签展示区 | — | 展示当前站点下使用频率最高的 10 个标签，点击即添加 |

#### 5.2.2 批量打标

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 已选线索数 | 只读展示 | — | 展示已选中的线索数量 |
| 添加标签 | 文本输入 + 回车确认 | 否 | 用户手动输入标签，回车即新增 |
| AI 推荐标签 | 标签展示区 | — | 展示 AI 推荐的标签列表，点击即添加 |
| 热门标签 | 标签展示区 | — | 展示当前站点下使用频率最高的标签 |

> 说明：批量打标**不支持删除标签**，仅支持批量新增，避免误删已有标签。


### 5.3 弹窗交互说明

**打开弹窗：**
- 标题：`打标 - [线索名称]`（单条）/ `批量打标（共 N 条）`（批量）

**添加用户自定义标签：**
- 输入标签名称后按回车键，标签立即新增
- 支持中英文、数字、下划线，长度不超过 20 个字符
- 标签不可与已有标签重复（同一线索下）
- 若标签已存在，提示“该标签已存在”

**添加 AI 推荐标签：**
- 点击 AI 推荐标签，标签立即新增
- 新增后该标签从推荐区消失

**删除标签（仅单条）：**
- 鼠标悬停标签，显示关闭按钮（×）
- 点击关闭按钮，标签立即删除
- 二次确认：`确认删除标签“XXX”吗？`


### 5.4 AI 推荐标签

**数据来源：**

| 来源 | 说明 |
| :--- | :--- |
| AI 提取 | 基于邮件/会话内容自动提取的关键词或主题 |
| AI 企业画像 | 基于企业信息自动生成的行业/规模标签 |
| AI 意向分析 | 基于客户意向自动生成的需求标签 |

**展示规则：**
- 推荐标签数量上限 10 个
- 标签按 AI 置信度排序（高 → 低）
- 仅展示未被用户添加的推荐标签
- 若线索已有足够标签（≥ 10 个），不再展示推荐标签区域


### 5.5 热门标签

**数据来源：** 当前站点下所有线索中使用频率最高的标签（Top 10）

**展示规则：**
- 热门标签与 AI 推荐标签并列展示
- 热门标签按使用频率排序（高 → 低）
- 仅展示未被当前线索添加的热门标签


### 5.6 业务规则

| 规则编号 | 规则说明 |
| :--- | :--- |
| BR-016 | AI 输出意向总结、企业信息、企业画像、风险信息、AI 标签 |

**标签类型说明：**

| 标签类型 | 来源 | 是否可删除 | 说明 |
| :--- | :--- | :--- | :--- |
| AI 标签 | AI 自动生成 | 不可删除 | 由系统自动管理，用户不可删除 |
| 手动标签 | 用户手动添加 | 可删除 | 用户自定义，可自由增删 |


### 5.7 打标后页面刷新

| 操作 | 页面反馈 |
| :--- | :--- |
| 添加标签 | 线索详情/列表的标签列实时更新 |
| 删除标签 | 标签从当前线索移除，列表实时更新 |
| 批量打标 | 所有选中线索新增该标签，列表刷新，成功提示“已为 N 条线索添加标签 [标签名]” |
