## 一、模块概述

**模块归属：** 分析中心 → 获客分析

**模块定位：** 提供渠道效能与来源站点的分析数据，帮助管理者了解不同渠道和站点的获客能力与转化效率。

**面向角色：** 销售主管、区域负责人


## 二、页面整体布局

获客分析页面采用 **顶部筛选栏 + 渠道效能卡片区 + 来源站点表格区** 布局：

| 区域 | 内容 |
| :--- | :--- |
| **顶部筛选栏** | 时间快捷选项 + 站点筛选（仅管理员/区域负责人可见） |
| **渠道效能分析** | 邮件 + WhatsApp 两个渠道效能卡片 |
| **来源站点价值分析** | 站点维度数据表格，含分页 |


## 三、顶部筛选栏

**功能定位：** 统一控制页面所有图表和表格的数据范围。

| 筛选项 | 类型 | 说明 | 默认值 |
| :--- | :--- | :--- | :--- |
| **时间快捷选项** | 按钮组 | 今日 / 本周 / 本月 / 本季度 / 自定义 | 本月 |
| **站点筛选** | 下拉选择 | 按站点过滤数据，仅管理员/区域负责人可见 | 全部站点 |

**交互规则：**
- 点击时间快捷选项，页面所有卡片和表格数据同步刷新
- 自定义时间支持选择起始日期和结束日期
- 站点筛选切换后，所有数据按站点过滤


## 四、渠道效能分析

**功能定位：** 展示邮件和 WhatsApp 两个渠道的消息处理与识别效能。

### 4.1 布局

一行展示 2 个卡片，左为邮件，右为 WhatsApp。

### 4.2 邮件效能卡片

| 指标 | 说明 | 示例 |
| :--- | :--- | :--- |
| **消息总量** | 该渠道在统计周期内的消息总数 | 4,520 |
| **AI 识别数量** | 经 AI 识别处理的消息数量 | 3,096 |
| **有效消息比例** | 有效消息数 / 消息总量 × 100% | 68.5% |
| **增长** | 较上一统计周期的消息量增长率 | +12% |

### 4.3 WhatsApp 效能卡片

| 指标 | 说明 | 示例 |
| :--- | :--- | :--- |
| **消息总量** | 该渠道在统计周期内的消息总数 | 8,912 |
| **AI 识别数量** | 经 AI 识别处理的消息数量 | 7,316 |
| **有效消息比例** | 有效消息数 / 消息总量 × 100% | 82.1% |
| **增长** | 较上一统计周期的消息量增长率 | +24% |

### 4.4 数据说明

- **有效消息**：经系统规则识别或 AI 识别，满足有效条件并生成线索的消息
- **增长**：较上一统计周期（与当前筛选周期对应，如上月）的消息量增长率


## 五、来源站点价值分析

**功能定位：** 展示各站点的获客数量与转化效率，帮助识别高价值与低效站点。

### 5.1 表格字段

| 字段 | 说明 | 示例 |
| :--- | :--- | :--- |
| **站点名称** | 来源站点名称 | 官方网站 |
| **渠道** | 该站点主要来源渠道 | 邮件 / WhatsApp / 混合 |
| **消息数量** | 该站点在统计周期内的消息总数 | 12,450 |
| **线索数量** | 该站点在统计周期内生成的线索总数 | 842 |
| **转化率** | 线索数量 / 消息数量 × 100% | 6.76% |
| **成交金额** | 该站点关联合同的总金额 | ¥1,245,000 |

**渠道列取值说明：**
- **邮件**：该站点 80% 以上消息来自邮件
- **WhatsApp**：该站点 80% 以上消息来自 WhatsApp
- **混合**：邮件和 WhatsApp 消息均占一定比例（均超过 20%）

### 5.2 交互规则

| 交互行为 | 说明 |
| :--- | :--- |
| **排序** | 支持按消息数量、线索数量、转化率、成交金额升序/降序排列，默认按成交金额倒序 |
| **分页** | 每页 10 条，支持翻页 |
| **点击站点名称** | 跳转至站点中心查看该站点详情（预留） |

### 5.3 分页

| 配置项 | 说明 |
| :--- | :--- |
| 默认每页条数 | 10 条 |
| 总条数展示 | 显示“共 N 个站点” |


## 六、数据更新频率

| 数据范围 | 说明 |
| :--- | :--- |
| 渠道效能卡片 | 按所选时间范围实时统计 |
| 来源站点表格 | 按所选时间范围实时统计 |


## 七、空态与异常态

| 场景 | 处理方式 |
| :--- | :--- |
| 无任何数据 | 各卡片显示“0”或“暂无数据”，表格显示空状态占位图 |
| 部分渠道无数据 | 该渠道卡片显示“暂无数据” |
| 数据加载失败 | 显示“数据加载失败，请刷新重试”，提供【重新加载】按钮 |


## 八、角色数据权限

| 角色 | 数据范围 |
| :--- | :--- |
| 销售主管 | 查看本团队全部数据 |
| 区域负责人 | 查看本区域全部数据 |


## 九、与其他模块的关系

| 关联模块 | 关系说明 |
| :--- | :--- |
| **沟通中心** | 渠道效能数据来源于邮件和 WhatsApp 的消息记录 |
| **线索中心** | 线索数量、转化率数据来源于线索生成记录 |
| **合同中心** | 成交金额数据来源于关联合同的金额汇总 |
| **站点中心** | 来源站点数据来源于站点配置 |
