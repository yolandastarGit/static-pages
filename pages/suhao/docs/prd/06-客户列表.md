## 一、模块概述

**模块归属：** 客户中心 → 客户列表

**模块定位：** 统一管理所有客户档案，是企业客户资产的集中管理页面。

**面向角色：** 销售业务员、销售主管、区域负责人


## 二、页面整体布局

客户列表页面采用 **单页表格布局**，从上至下分为三个区域：

| 区域 | 内容 |
| :--- | :--- |
| **筛选条件栏** | 多维度筛选条件，支持精细化查询 |
| **表格区** | 客户数据表格，含操作按钮及分页 |


## 三、筛选条件栏

**功能定位：** 提供多维度筛选条件，支持精细化查询。

| 筛选项 | 类型 | 说明 | 默认值 |
| :--- | :--- | :--- | :--- |
| **客户关键词** | 文本输入 | 模糊搜索客户名称、客户编号 | 空 |
| **国家/地区** | 下拉选择 | 按客户所在国家/地区筛选 | 全部 |
| **行业** | 下拉选择 | 按客户所属行业筛选 | 全部 |
| **客户状态** | 下拉选择 | 按客户当前状态筛选 | 全部 |
| **负责人** | 下拉选择 | 按客户负责人筛选，可选当前用户有权限查看的团队成员 | 全部 |
| **站点** | 下拉选择 | 按来源站点筛选 | 全部 |

**操作按钮：**

| 按钮 | 说明 |
| :--- | :--- |
| **查询** | 按当前筛选条件刷新表格数据 |
| **重置** | 将所有筛选条件恢复为默认值 |
| **导出客户** | 将当前筛选结果导出为 CSV 文件，导出范围为当前筛选条件下的全部数据（不限当前页），受操作权限控制。导出失败时提示"导出失败，请重试" |
| **高级筛选** | 展开更多筛选条件 |

## 五、新建客户弹窗

**功能定位：** 支持销售直接创建客户，不依赖线索转化。

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| **客户名称** | 文本输入 | 是 | 客户企业名称 |
| **国家/地区** | 下拉选择 | 否 | 客户所在国家 |
| **行业** | 下拉选择 | 否 | 客户所属行业 |
| **联系人姓名** | 文本输入 | 否 | 主要联系人姓名 |
| **联系人职位** | 文本输入 | 否 | 主要联系人职位 |
| **联系人邮箱** | 文本输入 | 否 | 主要联系人邮箱 |
| **联系人电话** | 文本输入 | 否 | 主要联系人电话 |
| **来源站点** | 下拉选择 | 是 | 客户所属站点 |

**提交逻辑：**
1. 校验客户名称不可为空
2. 校验来源站点不可为空
3. 校验当前站点下是否已存在同名客户，若存在则提示"该客户已存在，请勿重复创建"
4. 跨站点同名客户允许创建，不做拦截
5. 提交后创建客户，当前用户自动成为客户负责人
6. 联系人信息写入客户下的联系人对象
7. 自动生成一条跟进记录（跟进方式：备注，内容："新建客户"）
8. 客户状态初始为空，由业务员在客户详情页或编辑时手动设置


## 六、AI 画像弹窗

**功能定位：** 展示从线索同步至客户的企业信息（BR-036）。

| 展示内容 | 说明 |
| :--- | :--- |
| **企业画像** | AI 提取的企业画像信息 |
| **企业信息** | 企业名称、行业、年营收、员工规模、总部所在地等 |
| **风险信息** | AI 识别的企业风险信息 |
| **AI 标签** | AI 生成的客户标签（与客户手动标签独立，仅展示不可编辑） |

> 说明：AI 画像数据来源于线索转客户时同步的 AI 信息，为只读展示，不可编辑。
>
> **空态兜底：** 若客户通过手动创建（无关联线索），AI 画像弹窗展示"暂无 AI 画像数据，该客户通过手动创建，未关联 AI 分析"。


## 七、客户转移（负责人转移）

**功能定位：** 将客户的负责人从当前负责人变更为其他销售人员，后续该客户新生成的所有线索自动分配给新负责人。

**触发入口：**
- 表格顶部【客户转移】按钮（批量）
- 操作列【更多 → 转移】（单条）

**执行权限：** 当前客户负责人本人 或 销售主管

### 7.1 弹窗字段

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 客户名称 | 只读展示 | — | 展示当前客户名称 |
| 当前负责人 | 只读展示 | — | 展示当前客户负责人 |
| 新负责人 | 下拉选择 | 是 | 可选当前站点下所有状态为"在职"的销售人员 |
| 转移备注 | 多行文本 | 是 | 记录转移原因，必填 |

### 7.2 转移逻辑

1. 选择新负责人后，点击确认执行转移
2. 转移前校验：
   - 校验新负责人当前状态是否为"在职"，若已被禁用或离职，阻止转移并提示"该负责人已不可用，请重新选择"
   - 校验新负责人与当前负责人不同，若相同则提示"新负责人与当前负责人相同，无需转移"
3. 转移成功后：
   - 客户负责人更新为新负责人
   - 该客户后续新生成的所有线索自动分配给新负责人
   - 历史线索负责人保持不变（不因本次转移而变更）
   - 自动生成跟进记录（跟进方式：备注，内容："客户 [客户名称] 由 [原负责人] 转移至 [新负责人]"）
4. 转移失败时，弹窗提示错误原因

### 7.3 批量转移

- 选中多条客户，点击顶部【客户转移】按钮
- 统一选择新负责人，所有选中客户一并转移
- 转移备注共用
- 逐条执行，全部成功时提示"成功转移 N 条"；部分成功时弹窗汇总展示成功和失败数量及原因；全部失败时展示错误原因


## 八、编辑客户

**功能定位：** 编辑客户信息及联系人，支持增删改联系人。

**触发入口：** 操作列【更多 → 编辑】

**执行权限：** 当前客户负责人本人 或 销售主管

### 8.1 弹窗功能

| 区域 | 说明 |
| :--- | :--- |
| **客户信息** | 客户名称、来源站点只读不可改；国家、行业、客户状态、客户标签可编辑 |
| **联系人列表** | 展示该客户下的所有联系人 |
| **新增联系人** | 点击新增一行空白联系人表单（姓名、职位、邮箱、电话、是否主要联系人） |
| **编辑联系人** | 点击联系人行上的编辑按钮，字段变为可编辑状态 |
| **删除联系人** | 点击联系人行上的删除按钮，二次确认后删除（逻辑删除） |

**提交逻辑：**
- 编辑客户信息时自动生成跟进记录，记录变更内容："更新了客户信息：[字段名] 由 [旧值] 变更为 [新值]"
- 新增/编辑/删除联系人时自动生成跟进记录

**联系人表单字段：**

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 姓名 | 文本输入 | 是 | 联系人姓名 |
| 职位 | 文本输入 | 否 | 联系人职位 |
| 邮箱 | 文本输入 | 否 | 联系人邮箱 |
| 电话 | 文本输入 | 否 | 联系人电话 |
| 是否主要联系人 | 开关/复选框 | 否 | 标识该联系人是否为主要联系人 |

> 客户本身不可删除，仅支持联系人维度的删除操作（BR-043）。


## 九、异常状态与兜底逻辑

| 场景 | 处理方式 |
| :--- | :--- |
| 客户列表无数据 | 显示"暂无客户"占位图 |
| 搜索无结果 | 显示"未找到匹配的客户"，建议"请调整筛选条件后重试" |
| 数据加载失败 | 显示"数据加载失败，请刷新重试"，提供【重新加载】按钮 |
| 新建客户名称重复（同站点） | 校验当前站点下是否已存在同名客户，若存在提示"该客户已存在，请勿重复创建" |
| 新建客户名称重复（跨站点） | 不同站点允许同名客户，不做拦截，正常创建 |
| 客户转移时新负责人与当前相同 | 提交时校验拦截，提示"新负责人与当前负责人相同，无需转移" |
| 客户转移时新负责人已禁用 | 校验并阻止转移，提示"该负责人已不可用，请重新选择" |
| 批量转移部分成功 | 逐条执行，弹窗汇总展示成功和失败的数量及原因 |
| AI 画像无数据 | 展示"暂无 AI 画像数据，该客户通过手动创建，未关联 AI 分析" |

## 十二、客户标签管理

### 12.1 标签数据一致性

客户标签是**同一份数据**的两种管理入口：

- **客户列表**：客户行操作列【打标】→ 标签管理弹窗
- **客户详情页**：客户详情页 → 客户信息 → 标签管理

两处的标签**实时双向同步**，无任何时延。任一处修改、添加、删除标签，另一处立即反映。

### 12.2 标签分类

| 标签类型 | 来源 | 可编辑性 |
| :--- | :--- | :--- |
| AI 标签 | 系统根据客户信息、行为、邮件往来自动生成 | 仅系统可改（人工只读） |
| 手动标签 | 销售业务员或主管手动添加 | 手动可增/删/改 |

## 十三、最近签约合同算法

### 13.1 数据口径

"最近签约合同"在客户列表与客户详情页中**口径一致**，均按以下规则取数：

- 筛选范围：仅取合同状态为"**已签约**"或"**执行中**"的合同
- 排序规则：按合同创建时间倒序
- 取数：取最近 1 份

### 13.2 展示字段

| 字段 | 说明 |
| :--- | :--- |
| 合同编号 | 客户关联的最近 1 份合同编号 |
| 合同名称 | 客户关联的最近 1 份合同名称 |
| 合同金额 | 客户关联的最近 1 份合同金额（含币种） |
| 签约日期 | 客户关联的最近 1 份合同的签约日期 |
| 合同状态 | 当前状态（已签约/执行中） |

### 13.3 边界场景

| 场景 | 展示 |
| :--- | :--- |
| 客户无任何合同 | "—" |
| 客户有合同但全部为"待签约"或"已终止" | "—" |
| 客户有多份"已签约"或"执行中"合同 | 仅展示创建时间最近的一份 |
