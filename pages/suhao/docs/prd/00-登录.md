## 一、登录概述

AI 智能 CRM 采用统一身份认证机制，支持两种登录方式：**系统账号密码登录** 与 **钉钉扫码登录**。

用户登录成功后，系统根据用户角色自动加载：

- 菜单权限
- 页面权限
- 数据权限
- 操作权限

并进入对应角色的工作台首页。


## 二、登录方式

### 方式一：账号密码登录（默认）

用户使用系统管理员分配的系统账号及密码登录。

**适用角色：** 销售业务员、销售主管、区域负责人、系统管理员

**登录页面字段：**

| 字段 | 必填 | 说明 | UI 交互 |
| :--- | :--- | :--- | :--- |
| 登录账号 | 是 | 系统分配的唯一账号 | 文本输入框，占位符提示“请输入登录账号” |
| 登录密码 | 是 | 用户密码 | 密码输入框，支持显示/隐藏密码（小眼睛图标） |

**辅助功能：**

| 功能 | 说明 |
| :--- | :--- |
| 【登录】按钮 | 点击后触发认证流程 |
| 【记住账号】复选框 | 选中后，下次登录时自动填充账号（本地存储） |
| 【忘记密码】链接 | 点击后提示“请联系系统管理员重置密码”（本期不启用自助找回） |

> 设计说明：“忘记密码”不提供自助重置，仅做提示引导，确保安全可控。


### 方式二：钉钉扫码登录

支持企业钉钉扫码登录，无需额外输入账号密码。

**处理逻辑：**

| 场景 | 系统处理 |
| :--- | :--- |
| 已绑定系统账号 | 自动登录，进入工作台 |
| 未绑定系统账号 | 弹窗提示“当前钉钉账号未绑定系统用户，请联系管理员” |
| 账号已停用 | 禁止登录，提示“当前账号已停用，请联系管理员” |

**UI 布局：**
- 登录页面底部显示钉钉扫码入口
- 点击【钉钉扫码登录】按钮，弹出钉钉二维码
- 首次扫码绑定后，后续无需二次绑定


## 三、登录流程

### 账号密码登录

```
输入账号+密码
      │
      ▼
校验账号是否存在
      │
  ┌───┴───┐
  │       │
存在    不存在
  │       │
  ▼       ▼
校验密码  返回“用户名或密码错误”
  │
┌──┴──┐
│     │
匹配  不匹配
│     │
▼     ▼
加载   返回“用户名或密码错误”
角色权限
  │
  ▼
进入工作台
```

### 钉钉扫码登录

```
点击钉钉扫码登录
      │
      ▼
钉钉扫码
      │
      ▼
获取 OpenID
      │
      ▼
校验是否绑定系统账号
      │
┌─────┼─────┐
│     │     │
已绑定 未绑定 账号停用
│     │     │
▼     ▼     ▼
登录  提示   禁止登录
      联系管理员
```


## 四、登录页面 UI 设计

### 整体布局

采用居中卡片式布局：

| 区域 | 内容 |
| :--- | :--- |
| 品牌区 | 产品 Logo + “AI 智能 CRM” 产品名称 |
| 登录方式 Tab | 【账号密码登录】 \| 【钉钉扫码登录】（Tab 切换） |
| 表单区 | 账号输入框 + 密码输入框 + 记住账号复选框 |
| 操作区 | 【登录】按钮 + “忘记密码”链接 |
| 底部 | 版权信息（如 © 2026 AI CRM） |

### 字段校验规则

| 字段 | 校验规则 | 错误提示 |
| :--- | :--- | :--- |
| 登录账号 | 必填，不可为空 | “请输入登录账号” |
| 登录密码 | 必填，不可为空 | “请输入登录密码” |

> 安全设计：登录失败时，系统统一返回“用户名或密码错误”，不区分账号不存在或密码错误，防止账号枚举攻击。


## 五、登录成功

登录成功后，系统初始化以下信息：

### 用户信息

| 信息项 | 说明 |
| :--- | :--- |
| 用户 ID | 用户唯一标识 |
| 用户姓名 | 用户姓名 |
| 所属组织 | 所属组织 |
| 所属部门 | 所属部门 |
| 所属角色 | 所属角色 |
| 所属站点 | 所属站点（可能为多个） |

### 权限信息

系统自动加载权限配置：

| 权限类型 | 说明 |
| :--- | :--- |
| 菜单权限 | 控制左侧导航菜单可见性 |
| 页面权限 | 控制页面级访问 |
| 按钮权限 | 控制新增/编辑/删除/导出等操作 |
| 数据权限 | 控制数据行级可见范围（本人/本团队/本区域/全部） |

### 首页（工作台）

登录成功后，系统根据角色跳转至不同视角的工作台：

| 角色 | 首页内容 |
| :--- | :--- |
| 销售业务员 | 今日待跟进、我的线索、我的客户、我的合同 |
| 销售主管 | 团队待跟进、团队销售分析、公海数据 |
| 区域负责人 | 区域经营分析、区域销售数据、区域客户增长 |
| 系统管理员 | 系统运行情况、用户统计、站点统计 |


## 六、登录失败

系统统一返回登录失败提示，确保安全性与用户体验。

| 场景 | 提示文案 | 备注 |
| :--- | :--- | :--- |
| 用户不存在 | 用户名或密码错误 | 统一提示，不区分错误类型 |
| 密码错误 | 用户名或密码错误 | 同上 |
| 用户停用 | 当前账号已停用，请联系管理员 | 明确提示，便于用户知晓 |
| 密码过期（预留） | 当前密码已过期，请修改密码 | 本期预留，暂不启用 |
| 钉钉未绑定 | 当前钉钉账号未绑定系统用户，请联系管理员 | 引导用户联系管理员 |
| 网络异常 | 网络连接异常，请稍后重试 | 系统级兜底提示 |


## 七、退出登录

用户点击右上角头像区域，在下拉菜单中选择【退出登录】。

系统执行以下操作：

1. 清除当前登录 Token（本地存储 / Cookie）
2. 清除 Session 会话
3. 销毁前端状态
4. 重定向至登录页面

> 交互细节：退出时无需二次确认，但若存在未保存的草稿（如邮件草稿），建议弹出轻提示“退出后将丢失未保存内容”。


## 八、登录安全策略

### 8.1 会话管理

登录成功后创建用户会话（Session），同时生成 Token（JWT）返回前端。

- 前端存储 Token，每次请求携带至服务端鉴权
- 服务端维护 Session 状态

### 8.2 登录超时

用户连续无操作达到设定时间后，系统自动退出登录。

| 配置项 | 默认值 | 说明 |
| :--- | :--- | :--- |
| 超时时长 | 30 分钟 | 管理员可在【系统参数】中配置 |
| 超时提醒 | 弹窗提示“您的会话已超时，请重新登录” | 超时后自动跳转至登录页 |

### 8.3 同时登录

MVP 默认允许同一账号在多个设备/浏览器同时登录。

| 版本 | 策略 |
| :--- | :--- |
| MVP（本期） | 允许多端同时登录 |
| 后续版本 | 支持单点登录（SSO）、登录设备管理、强制下线 |

### 8.4 密码策略

| 策略项 | MVP 规则 | 后台可配置 |
| :--- | :--- | :--- |
| 初始密码 | 管理员初始化分配 | 否 |
| 用户修改密码 | 支持（个人中心） | 否 |
| 管理员重置密码 | 支持 | 否 |
| 密码长度 | ≥ 8 位 | 是 |
| 字符类型 | 支持数字、大小写字母、特殊字符（建议） | 是 |

> 设计说明：密码复杂度规则支持后台配置，但 MVP 建议使用默认规则，不强制特殊字符，降低用户使用门槛。

### 8.5 登录日志

系统自动记录所有登录行为，供管理员审计追溯。

**日志字段：**

| 字段 | 说明 |
| :--- | :--- |
| 登录时间 | 精确到秒 |
| 登录账号 | 系统账号 |
| 登录 IP | 客户端 IP 地址 |
| 登录方式 | 账号密码 / 钉钉扫码 |
| 登录结果 | 成功 / 失败（含失败原因） |
| 浏览器 | User-Agent 解析 |
| 操作系统 | User-Agent 解析 |

> 入口：管理员可在【系统管理 → 系统日志 → 登录日志】中查询。


## 九、权限初始化

登录成功后，系统完成权限的初始化加载。

### 菜单权限

决定左侧导航菜单的可见性。

| 角色 | 可见菜单 |
| :--- | :--- |
| 销售业务员 | 工作台、沟通中心、线索中心、客户中心、分析中心、个人中心 |
| 系统管理员 | 以上全部 + 站点中心、系统管理 |

### 数据权限

控制用户可查看的数据行级范围。

| 权限范围 | 说明 |
| :--- | :--- |
| 仅本人 | 仅查看自己负责的数据 |
| 本团队 | 查看所属团队的全部数据 |
| 本区域 | 查看所属区域的全部数据 |
| 全部 | 查看系统全部数据（仅管理员） |

### 操作权限

控制页面内按钮/操作的可见性及可用性。

| 操作 | 说明 |
| :--- | :--- |
| 新建 | 新增业务对象 |
| 编辑 | 修改已有数据 |
| 删除 | 逻辑删除（非物理删除） |
| 分配 | 分配线索/客户负责人 |
| 转客户 | 线索转化为客户 |
| 导出 | 数据导出为 Excel/CSV |
| 配置 | 系统配置项修改 |

### 站点权限

用户仅可查看已授权站点的数据。

**示例：**
- 用户属于：A 站、B 站
- 可查看：A 站线索、B 站客户、A 站合同
- 不可查看：C 站任何数据
