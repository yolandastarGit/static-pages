## 一、模块概述

**模块归属：** 消息管理（一级菜单）

**模块定位：** 管理消息模板、推送规则、消息提醒规则，是系统级消息能力的统一管理入口。负责业务侧消息内容的模板化、推送策略配置和业务事件触发的提醒规则。

**面向角色：** 系统管理员

**菜单结构：**

```
├── 消息管理
│   ├── 消息模板
│   ├── 推送管理
│   └── 消息提醒规则
```

> 平台级消息推送通道配置（SMTP/IMAP、钉钉 AppKey、WhatsApp Business API 等）统一归「系统管理 → 消息推送配置 / 邮件服务配置 / WhatsApp配置」。本模块聚焦业务侧的消息模板、推送规则和业务事件触发的消息提醒规则。

> **职责合并说明：** 业务事件触发的"消息提醒规则"（线索分配提醒、待跟进提醒、合同签约提醒）由原"系统管理 → 系统参数 → 消息提醒规则"小节合并到本模块下，作为「消息提醒规则」二级菜单维护（避免双层配置带来的理解成本）。


## 二、消息模板

### 2.1 列表查询

| 筛选项 | 类型 | 说明 |
| :--- | :--- | :--- |
| **关键词** | 文本输入 | 按模板名称搜索 |
| **渠道** | 下拉选择 | 按渠道筛选 |
| **状态** | 下拉选择 | 全部 / 启用 / 停用 |

### 2.2 模板列表

| 列名 | 说明 |
| :--- | :--- |
| **模板编码** | 模板唯一编码 |
| **模板名称** | 消息模板名称 |
| **触发事件类型** | 如线索分配 / 系统通知等 |
| **渠道** | 支持多标签展示 |
| **状态** | 启用 / 停用，状态标签颜色区分 |
| **更新时间** | 最近更新时间 |

### 2.3 MVP 阶段功能边界

**重要说明：** MVP 阶段，消息模板**仅提供列表展示**，**不支持新增、编辑、删除**。

- 列表顶部【新增模板】/【编辑】/【删除】按钮**全部置灰**或**不渲染**
- 模板的初始化内容由开发团队在系统版本升级时维护
- 模板变量占位符规则：`{{变量名}}`（如 `{{user_name}}` / `{{lead_name}}`），由模板渲染时替换
- 模板分类由系统初始化定义（线索类 / 客户类 / 合同类 / 系统类），业务侧不提供分类管理入口

## 四、新增推送

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :---: | :--- |
| **规则名称** | 文本输入 | 是 | 推送规则名称，规则唯一标识 |
| **关联模板** | 下拉选择 | 是 | 选择关联的消息模板（必填，模板决定消息主体内容） |
| **推送渠道** | 多选下拉 | 是 | 钉钉 / 站内信，至少选 1 个 |
| **推送对象** | 用户/角色混合选择器 | 是 | **多选混合**：同时支持选择具体用户和角色；最终接收人 = 选中用户 ∪ 选中角色下所有用户 |
| **推送文本** | 文本输入 | 否 | **选填**。不填写时使用关联模板的默认文本；填写时作为该规则的文本覆盖（支持模板变量） |


## 五、消息提醒规则

> **职责说明：** 本节为原"系统管理 → 系统参数 → 消息提醒规则"的合并版本。负责业务事件触发推送的开关、接收人、关联模板配置。

### 5.1 规则列表

| 规则名称 | 触发事件 | 推送渠道 | 关联模板 | 状态 | 操作 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 线索分配通知 | 线索分配至业务员 | 钉钉 + 站内信 | 线索分配模板 | 启用 | 编辑/停用 |
| 待跟进提醒 | 跟进周期超时 | 站内信 | 待跟进模板 | 启用 | 编辑/停用 |
| 合同签约通知 | 合同状态变更为已签约 | 钉钉 + 站内信 | 合同签约模板 | 启用 | 编辑/停用 |

### 5.2 新增/编辑消息提醒规则

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :---: | :--- |
| 规则名称 | 文本输入 | 是 | 规则名称 |
| 触发事件 | 下拉选择 | 是 | 业务事件类型（下拉项见 5.3） |
| 推送渠道 | 多选下拉 | 是 | 钉钉 / 站内信 |
| 接收人 | 用户/角色选择器 | 是 | **多选混合**（同推送管理） |
| 关联模板 | 下拉选择 | 是 | 选择消息模板（必填） |
| 状态 | 开关 | 是 | 启用 / 停用 |

### 5.3 触发事件清单

| 触发事件 | 说明 |
| :--- | :--- |
| 线索分配 | 线索被分配到业务员时触发 |
| 线索状态变更 | 线索状态变更时触发 |
| 线索自动回收 | 线索因超期未跟进被自动回收时触发 |
| 待跟进超时 | 跟进周期超过阈值未更新时触发 |
| 客户转移 | 客户转移给其他业务员时触发 |
| 客户状态变更 | 客户状态变更时触发 |
| 合同创建 | 合同被创建时触发 |
| 合同签约 | 合同状态变更为已签约时触发 |
| 合同状态变更 | 合同状态变更时触发 |

### 5.4 业务规则

- 同一触发事件可配置多条规则（不同接收人/不同模板）
- 启用状态变更后立即生效
- 停用规则后，该规则不再触发任何推送
- 接收人为空时规则不可启用


## 六、异常状态

| 场景 | 处理方式 |
| :--- | :--- |
| 列表为空 | 展示 Empty 状态，提供新增引导 |
| 保存失败 | Toast 提示具体错误原因，保留用户输入 |
| 停用推送规则 | 二次确认后执行，停用后该规则不再触发推送 |
| 关联模板已停用 | Toast 提示"关联模板已停用，请重新选择" |
| 推送对象全部失效（用户已禁用/角色已删除） | 规则保存时提示"推送对象中包含已失效的用户/角色，请重新选择" |
