# DESIGN_SYSTEM.md

# 设计目标

本项目采用现代企业 SaaS 管理后台设计规范。

整体设计原则：

- 专业
- 克制
- 高信息密度
- 操作效率优先
- 长时间办公阅读舒适
- 保持统一

禁止：

- 营销官网风格
- 花哨动画
- 大面积渐变
- 炫技设计
- 为设计而设计

设计必须服务业务。

# 冻结后补充规范

以下规范来源于 Design Review，用于后续设计与开发保持一致。

## Empty 与 No Permission

Empty State 与 No Permission 必须视觉区分。

Empty State：

* 用于当前条件下没有业务数据
* 可展示业务引导
* 可展示用户有权限执行的操作入口
* 语气为引导型

No Permission：

* 用于用户没有页面、数据或操作权限
* 使用独立权限状态页或权限状态块
* 明确说明“暂无访问权限”
* 提供返回上一页或联系管理员
* 不展示新增、编辑、导出等不可用业务操作

## Table 列规范

所有表格统一支持列配置时，遵循以下规则：

* 选择列固定在最左侧
* 主标识列不可隐藏
* 状态列默认展示
* 操作列固定在最右侧
* 操作列不可隐藏
* 表格横向滚动时，选择列、主标识列、操作列保持可见
* 表头排序状态必须可见

主标识列示例：

* 线索编号 / 线索名称
* 客户名称
* 合同编号
* 站点名称
* 用户姓名 / 登录账号

## Tag 规范

Tag 用于状态、来源、标签、AI 标识等轻量信息。

全局分类：

* 状态 Tag：用于启用、停用、已分配、跟进中、高意向、已签约、执行中、已完成等状态
* AI Tag：用于 AI 自动生成的标签，必须有 AI 标识或统一视觉区分
* 手动 Tag：用于用户手动添加的业务标签
* 只读 Tag：用于不可编辑的系统标签
* 风险 Tag：用于风险、失败、异常等状态

规则：

* 同一类型 Tag 在全系统保持一致样式
* AI Tag 与手动 Tag 必须可区分
* 可删除 Tag 悬停时显示删除入口
* 不可删除 Tag 不显示删除入口
* Tag 过多时折叠为 +N，悬停展示完整列表

## 多栏页面响应式

本项目以桌面端企业后台为主。

推荐最低工作宽度：

* 标准列表页：1280px
* 邮件三栏页：1440px
* WhatsApp 两栏页：1280px
* 角色权限与配置页：1280px
* 分析图表页：1366px

低于推荐宽度时：

* Sidebar 可折叠
* 表格允许横向滚动
* 详情侧栏允许折叠
* 邮件三栏可折叠右侧详情或 AI 面板
* WhatsApp 可折叠联系人 / AI 信息区
* 图表从双栏降为单栏堆叠

不要求移动端完整业务操作体验，但必须避免内容重叠、按钮溢出、表格不可操作。

## 权限保护视觉

对高风险权限配置页面：

* 系统管理员角色、内置角色、关键权限变更应有明显提示
* 禁止删除内置角色时，删除按钮不展示
* 关键权限保存前使用确认 Modal
* 权限变更影响范围使用说明文本展示
