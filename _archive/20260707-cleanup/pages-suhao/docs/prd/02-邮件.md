## 一、模块概述

**模块归属：** 沟通中心 → 邮件

**模块定位：** 统一管理用户绑定邮箱的所有邮件，提供邮件查看、AI 智能分析、线索生成等核心能力。
邮件收发能力依赖用户绑定邮箱，系统同步展示邮箱内已发送、草稿及垃圾箱数据。

**面向角色：** 销售业务员


## 二、页面整体布局

邮件管理页面采用**三栏式布局**：

| 区域 | 宽度 | 内容 |
| :--- | :--- | :--- |
| **左栏（列表区）** | 固定 240px | 邮件列表 |
| **中栏（正文区）** | 自适应，最小 400px | 邮件正文 |
| **右栏（AI 智能分析面板区）** | 占比 40% | AI 智能分析面板，支持折叠 |

## 五、左栏：邮件正文

**功能定位：** 展示邮件列表，邮件核心摘要，快速触达关键信息并进行线索流转。

### 列表字段

| 字段 | 说明 | 展示规则 |
| :--- | :--- | :--- |
| **复选框** | 支持多选，用于触发批量操作 | 位于列表最左侧 |
| **发件人/主题** | 显示发件人姓名和邮件主题 | 未读邮件主题加粗显示 |
| **邮件摘要** | 正文预览，截取前 100 字符 | 超过长度截断加“...” |
| **时间** | 显示收件时间 | 今天显示时分（如 10:42），历史日期显示月/日 跨年邮件展示 YYYY/MM/DD|

### 列表核心交互

1. **选中态：** 点击列表项，右侧详情面板刷新对应邮件详情。点击邮件详情后自动标记为已读；批量标记已读同样更新未读数量。

2. **复选框：** 支持多选，选中后顶部批量操作栏亮起，用于触发批量 AI 提炼等功能。

## 七、右栏： AI 智能分析面板
### 7.2 AI 智能分析区

此区域展示 AI 对当前邮件的分析结果。AI 仅作辅助，不阻塞业务。

#### 1）AI 意向智能总结

- **展示内容：** 100-200 字的自然语言总结，包含客户需求、预算信号、痛点及销售建议。
- **交互动作：** 底部提供【复制总结】按钮。

#### 2）AI 企业情报库

- **展示内容：** AI 提取的企业公开信息。
- **字段示例：**
  - 企业名称
  - 行业
  - 年营收（AI 补充信息，仅供销售参考）
  - 员工规模（AI 补充信息，仅供销售参考）
  - 总部所在地（AI 补充信息，仅供销售参考）
- **交互动作：** 若该企业已存在于 CRM，显示【查看已有客户档案】；若不存在，则不显示。
按当前站点+企业名称/邮箱域名匹配已有客户。。

> 说明：年营收、员工规模、总部为 AI 补充展示信息，仅供销售参考，不同步写入客户标准字段。

#### 3）AI 风险提醒

- **展示内容：** 安全与合规风险提示。
- **示例：** “发件邮箱与官方域名有细微差异（`@industrial-pro.co` vs `@industrial-pro.com`），建议进行二次身份校验。”

#### 4）AI 标签

- **展示内容：** AI 自动生成消息标签，仅作用于当前消息，与线索 AI 标签独立存储。
生成线索时可作为 AI 推荐标签参考，不自动同步
- **示例：** 高增长潜力、财富 500 强、疑似竞品。


## 七、异常状态与兜底逻辑

### 7.1 AI 服务不可用

- **页面表现：** AI 智能分析区显示灰色占位符，并提示“AI 服务暂时不可用，系统依赖规则引擎正常运行。”
- **业务影响：** 【生成线索】可用

### 7.2 来源站点匹配失败

- **页面表现：** 基础信息区“来源站点”显示为红色高亮下拉框，提示“未能识别来源站点，请手动选择”。
- **业务约束：** 若站点未选择，系统将阻止【生成线索】操作。
仅消息所属负责人可修改

### 7.3 邮件加载失败

- **页面表现：** 列表区显示“邮件加载失败，请刷新重试”。
- **交互：** 提供【重新加载】按钮。

## 九、草稿管理

### 9.1 功能定位

支持用户在邮件撰写过程中临时保存草稿，以及从草稿列表中重新打开、编辑恢复、删除草稿。

### 9.2 自动保存

| 维度 | 说明 |
| :--- | :--- |
| 触发时机 | 邮件撰写页打开后，每 30 秒自动保存一次 |
| 触发条件 | 内容相对上次保存有变化 |
| 保存范围 | 收件人、抄送、密送、主题、正文、附件 |
| 草稿标识 | 在收件人/主题/正文右上角显示"草稿已自动保存于 HH:MM" |
| 异常处理 | 自动保存失败时 Toast 提示"草稿自动保存失败，请手动保存"，草稿标识切换为"草稿未保存" |

### 9.3 手动保存

| 维度 | 说明 |
| :--- | :--- |
| 入口 | 邮件撰写页顶部【保存草稿】按钮 |
| 行为 | 立即保存当前邮件为草稿 |
| 反馈 | Toast 提示"草稿已保存"，回到邮件列表页 |

### 9.4 草稿列表

**入口：** 沟通中心 → 邮件 → 左侧文件夹 → 【草稿箱】

**列表字段：**

| 字段 | 说明 |
| :--- | :--- |
| 收件人 | 草稿收件人（空时显示"无"） |
| 主题 | 草稿主题（空时显示"无主题草稿"） |
| 收件邮箱 | 当前操作的邮箱账号 |
| 最后保存时间 | 自动/手动保存时间，精确到分钟 |
| 操作 | 【继续编辑】/【删除】 |

**列表交互：**

| 行为 | 说明 |
| :--- | :--- |
| 点击列表项 | 打开草稿继续编辑（覆盖现有草稿时需二次确认） |
| 选中后批量删除 | 多选后顶部【批量删除】按钮 |
| 排序 | 默认按最后保存时间倒序 |
| 分页 | 默认 20 条/页 |

### 9.5 草稿删除

| 维度 | 说明 |
| :--- | :--- |
| 单条删除 | 点击操作列【删除】→ 二次确认弹窗 → 确认后软删除（保留 30 天后物理清理） |
| 批量删除 | 多选后【批量删除】→ 显示已选数量 → 二次确认 → 逐条软删除 |
| 二次确认 | 弹窗提示"删除后 30 天内可在回收站恢复，确定删除吗？" |
| 回收站 | 左侧文件夹新增【已删除草稿】，保留 30 天后自动物理清理 |

## 十一、批量 AI 提炼弹窗

### 11.1 触发入口

邮件列表多选邮件 → 顶部【批量 AI 提炼】按钮

### 11.2 弹窗字段

| 字段 | 组件类型 | 是否必填 | 默认值 | 说明 |
| :--- | :--- | :---: | :--- | :--- |
| 已选邮件数 | 只读展示 | — | — | 展示已选中的邮件数量（上限 50 封） |
| 提炼规则 | 多选下拉 | 否 | AI 智能识别 | 选项：客户名称/联系方式/采购意向/产品需求/竞争情报 |
| 是否自动生成线索 | 开关 | 否 | 关闭 | 开启后：提炼结果直接生成"待确认"线索 |
| 来源站点 | 下拉选择 | 是 | 邮件来源站点 | 仅在"自动生成线索"开启时必填，可手动修正 |
| 影响范围提示 | 只读展示 | — | — | "AI 提炼将分析 N 封邮件，预计耗时 X 秒" |

### 11.3 提炼结果展示

| 维度 | 说明 |
| :--- | :--- |
| 展示形式 | 弹窗内表格 + 邮件分组 |
| 表格列 | 邮件主题、AI 提炼客户名称、联系方式、意向等级、是否生成线索 |
| 操作 | 单条【应用到线索】、批量【全部应用】 |
| 失败处理 | AI 提炼失败的邮件展示在"提炼失败"分组，可重试或人工处理 |

### 11.4 业务规则

- 同一邮箱账号下邮件方可批量提炼（不允许跨邮箱）
- 已读/未读邮件均可批量提炼
- 同一封邮件不允许重复提炼（已提炼过的邮件显示"已提炼"标签）


## 十二、邮件来源站点匹配失败

### 12.1 触发场景

邮件触发"生成线索"操作时，系统无法自动匹配邮件所属站点。

### 12.2 弹窗字段

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :---: | :--- |
| 来源邮件主题 | 只读展示 | — | 当前生成线索的邮件主题 |
| 来源邮箱 | 只读展示 | — | 邮件所属邮箱 |
| 来源站点 | 下拉选择 | 是 | 手动选择目标站点（选项来自系统管理 → 站点管理） |
| 默认建议 | 只读展示 | — | "根据发件人域名，建议选择：[站点名]" |

### 12.3 业务规则

- 未选择来源站点时"生成线索"按钮置灰，提示"请先选择来源站点"
- 确认修改后立即保存"邮件-站点"映射关系，作为后续相同发件人邮件的默认站点
- 映射关系存于 `email_site_mapping` 表，按 `(邮箱账号, 发件人域名)` 唯一索引
- 已建立的映射关系可由管理员在【系统管理 → 字典管理】中查看与管理
