## 一、模块概述

**模块名称：** 工作台

**模块定位：** 工作台是系统的首页，帮助各角色快速了解当前工作及业务情况

**面向角色：** 全部角色

**核心价值：**
- 一站式查看今日待办与工作提醒
- 实时掌握销售经营指标
- 快速进入核心业务页面


## 二、角色视图概览

登录成功后，系统根据角色展示不同的工作台内容：

| 角色 | 首页聚焦 |
| :--- | :--- |
| 业务员 | 今日待跟进、线索、我的客户、快捷入口 |
| 运营专员 | 站点数据、公海、销售漏斗、业务员排行 |
| 协同人 | 运营经营分析 |
| 系统管理员 | 今日待跟进、线索、我的客户、快捷入口  |

> 不同角色看到的数据范围和指标维度不同，但页面布局结构一致。


## 三、页面布局

采用 **卡片式仪表盘布局**，从上至下分为三个区域：

| 区域 | 内容 | 布局建议 |
| :--- | :--- | :--- |
| **顶部筛选栏** | 时间范围选择器 + 站点筛选（如适用） | - |
| **核心指标卡** | 6个关键数据卡片（新增线索、待跟进、成交额、有效线索率、线索成交率、客户活跃度） | 一行3-4列，等宽分布 |
| **中部图表区** | 销售漏斗 + 成交额趋势 | 左右两栏布局（漏斗居左，趋势居右） |
| **底部快捷入口** | 公海池、线索列表、客户列表、合同中心 | 一行4列图标+文字入口 |


## 四、顶部筛选栏

| 筛选项 | 说明 | 默认值 |
| :--- | :--- | :--- |
| **时间范围** | 支持按今日/本周/本月/本季度/自定义区间筛选 | 本月 |
| **站点筛选** | 仅管理员/区域负责人可见，按站点过滤数据 | 全部站点 |

> **生效范围：** 时间筛选作用于**页面所有指标和图表**（核心指标卡、销售漏斗、成交额趋势均受影响）。


## 五、核心指标卡

位于筛选栏下方，一行展示6个关键数据卡片（按角色展示不同内容）。


### 卡片一：新增线索

**定义：** 在选定时间范围内，系统新生成的线索总数。

**展示内容：**

| 元素 | 说明 |
| :--- | :--- |
| 主数值 | 新增线索总数；数据为空或加载失败时显示"--" |
| 来源分布 | 按渠道展示数量及占比 |

**来源渠道分布示例：**

| 渠道 | 说明 |
| :--- | :--- |
| 邮件 | 通过邮件识别生成的线索 |
| WhatsApp | 通过 WhatsApp 识别生成的线索 |
| 社媒 | 通过社媒渠道识别生成的线索（预留扩展） |

> 社媒渠道为本期预留扩展，数据暂为0或隐藏，待后续接入 Facebook Messenger、LinkedIn 等渠道后启用。

**展示方式：**
- 主数值以大号字体突出显示
- 来源分布以小字或标签形式展示在主数值下方
- 示例视觉：`新增线索 23` `社媒14 / 邮件6 / WhatsApp 3`


### 卡片二：待跟进

**定义：** 当前用户有跟进计划且 **下次跟进时间 = 今日** 的线索数量。

**展示内容：**

| 元素 | 说明 |
| :--- | :--- |
| 主数值 | 待跟进线索总数；数据为空或加载失败时显示"--" |
| 提示信息 | 点击可跳转至线索列表（自动筛选今日待跟进） |

> 交互：点击卡片或数值，跳转至【线索中心 → 线索列表】，并自动按“下次跟进时间 = 今日”过滤。


### 卡片三：成交额

**定义：** 在选定时间范围内，已签订合同金额的累计总和。

**展示内容：**

| 元素 | 说明 |
| :--- | :--- |
| 主数值 | 累计成交金额（含币种，如 ¥1,280,000）；数据为空或加载失败时显示"--" |
| 对比信息 | 较上一周期增长/下降百分比（如 ↑ 12.5%） |


### 卡片四：有效线索率

**定义：** 有效线索数占全部线索数的百分比。

**计算公式：** `有效线索率 = 有效线索数 / 全部线索数 × 100%`

**展示内容：**

| 元素 | 说明 |
| :--- | :--- |
| 主数值 | 百分比（如 68.5%）；数据为空或加载失败时显示"--" |
| 辅助信息 | 有效线索数 / 全部线索数（如 198 / 289） |


### 卡片五：线索成交率

**定义：** 已转化为合同的线索数占有效线索数的百分比。

**计算公式：** `线索成交率 = 已成交线索数 / 有效线索数 × 100%`

**展示内容：**

| 元素 | 说明 |
| :--- | :--- |
| 主数值 | 百分比（如 17.2%）；数据为空或加载失败时显示"--" |
| 辅助信息 | 已成交线索数 / 有效线索数（如 34 / 198） |


### 卡片六：客户活跃度

**定义：** 在统计周期（默认近30天）内有跟进行为或消息互动的客户数量及占比。

**活跃定义：** 客户在统计周期内发生过以下任一行为：
- 有新增消息（邮件 / WhatsApp）
- 有新增跟进记录
- 有新增合同

**展示内容：**

| 元素 | 说明 |
| :--- | :--- |
| 主数值 | 活跃客户数（如 86）；数据为空或加载失败时显示"--" |
| 辅助信息 | 占总客户数的百分比（如 86 / 420，20.5%） |
| 趋势 | 近7天活跃客户数变化趋势（迷你折线图或箭头指示） |


## 六、销售漏斗

**定义：** 展示线索从进入到最终成交的各阶段转化情况，帮助管理者识别转化瓶颈。

**漏斗阶段（从左到右）：**

| 阶段 | 说明 |
| :--- | :--- |
| 线索进入 | 选定时间内进入系统的线索总数 |
| 有效线索 | 经识别确认为有效的线索数 |
| 高意向 | 达到高意向阶段的线索数（系统自动判定） |
| 合同成交 | 最终签订合同的线索数 |

**展示方式：**

| 元素 | 说明 |
| :--- | :--- |
| 图表类型 | 横向漏斗图（左宽右窄） |
| 数值展示 | 每个阶段显示数量 |
| 转化率 | 相邻阶段之间显示转化百分比 |
| 颜色 | 各阶段使用渐变色，从深到浅或品牌色系 |

**示例：**
```
线索进入     ━━━━━━━━━━━━━━━━━━━━━━ 256
有效线索     ━━━━━━━━━━━━━━━━━━ 198   ↓ 77.3%
高意向       ━━━━━━━━━━━ 68            ↓ 58.6%
合同成交     ━━━━━━ 34                  ↓ 50.0%
```

> 时间筛选影响：漏斗数据受顶部时间范围筛选控制。


## 七、成交额趋势

**定义：** 展示选定时间范围内成交额的变化趋势。

**默认展示：** 近6个月的月度成交额趋势（折线图）

**展示方式：**

| 元素 | 说明 |
| :--- | :--- |
| 图表类型 | 折线图（趋势线） |
| X轴 | 时间（日/周/月，根据筛选切换） |
| Y轴 | 成交金额 |
| 数据标签 | 鼠标悬停显示具体数值 |
| 空态处理 | 无数据时显示“暂无成交数据” |


## 八、快捷入口

**定义：** 提供核心业务页面的快速跳转入口。

| 入口名称 | 跳转目标 |
| :--- | :--- |
| 公海池 | 线索中心 → 公海池 |
| 线索列表 | 线索中心 → 线索列表 |
| 客户列表 | 客户中心 → 客户列表 |
| 合同中心 | 客户中心 → 合同中心 |

**交互说明：**
- 点击卡片直接跳转至对应页面
- 鼠标悬停有轻微上浮效果（反馈交互）


## 十、交互说明

| 交互行为 | 说明 |
| :--- | :--- |
| 点击指标卡片 | 跳转至对应的详情列表页（如点击“新增线索”跳转至线索列表） |
| 时间范围切换 | 切换后所有卡片和图表数据同步刷新 |
| 站点筛选切换 | 切换后所有数据按站点过滤（仅管理员/区域负责人可见） |
| 图表悬停 | 销售漏斗和成交额趋势支持鼠标悬停显示详细数值 |
| 快捷入口点击 | 直接跳转至目标页面 |


## 十一、空态与异常态

| 场景 | 处理方式 |
| :--- | :--- |
| 无任何数据 | 各指标卡主数值显示"--"占位符，辅助信息区域留空；销售漏斗显示空状态占位图 |
| 无待跟进线索 | 待跟进卡片主数值显示"--"，提示“今日暂无待跟进的线索” |
| 无成交额数据 | 成交额卡片主数值显示"--"，趋势图显示“暂无成交数据” |
| 单个指标卡数据加载失败 | 该卡片主数值显示"--"，不影响其他卡片展示 |
| 数据加载中 | 各卡片显示骨架屏占位 |
| 数据加载失败 | 显示“数据加载失败，请刷新重试” |
