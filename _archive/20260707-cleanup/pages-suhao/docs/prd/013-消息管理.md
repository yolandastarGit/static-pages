## 一、模块概述

**模块归属：** 通知管理（一级菜单）

##通知中心

模块定位

统一配置系统各业务场景的通知提醒方式。
管理员协同人只需开启需要的通知，并配置通知渠道和通知对象即可，无需维护复杂的通知规则。

**面向角色：** 系统管理员

### 5.3 通知场景事件清单

| 通知场景事件 | 说明 |
| :--- | :--- |
| 线索状态变更 | 线索状态变更时触发 |
| 待跟进超时 | 跟进周期超过阈值未更新时触发 |
| 客户转移 | 客户转移给其他业务员时触发 |
| 合同创建 | 合同被创建时触发 |
| 合同状态变更 | 合同状态变更时触发 |
| 邮件未读数量提醒 | 邮件未读触发 |
| 邮件未读超时提醒 | 邮件未读超时 3 天触发提醒 |


### 5.4 业务规则

- 同一触发事件可配置多条规则（不同接收人/不同模板）
- 启用状态变更后立即生效
- 停用规则后，该规则不再触发任何推送
- 接收人为空时规则不可启用


## 六、异常状态

| 场景 | 处理方式 |
| :--- | :--- |
| 列表为空 | 展示 Empty 状态，提供新增引导 |
| 保存失败 | Toast 提示具体错误原因，保留用户输入 |
| 停用推送规则 | 二次确认后执行，停用后该规则不再触发推送 |
| 推送对象全部失效（用户已禁用/角色已删除） | 规则保存时提示"推送对象中包含已失效的用户/角色，请重新选择" |
