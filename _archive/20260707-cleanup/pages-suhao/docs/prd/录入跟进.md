## 一、操作概述
**操作名称：** 录入跟进
**触发入口：**

| 入口 | 位置 | 前置条件 |
| :--- | :--- | :--- |
| 入口一 | 线索列表 → 顶部【录入跟进】按钮 | 选中单条线索时按钮亮起，未选中时置灰 |
| 入口二 | 线索列表 → 操作列【跟进】按钮 | 无条件 |
| 入口三 | 线索详情 → 顶部【录入跟进】按钮 | 无条件 |

**功能定位：** 为线索新增一条跟进记录，完整记录销售与客户的每一次互动。


## 二、弹窗字段定义

| 字段 | 组件类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 线索名称 | 只读展示 | — | 展示当前线索名称，不可编辑 |
| 跟进方式 | 下拉选择 | 是 | 选项：电话 / 邮件 / WhatsApp / 会议 / 拜访 / 备注 |
| 跟进内容 | 多行文本 | 是 | 跟进记录详细内容，建议最少 10 个字符 |
| 客户关注 | 多选下拉 | 否 | 选项：报价/价格、交期、资质证书、产品定制能力、产品目录册、项目案例、求代理商合作、样品费及归属、是否为工厂、支付方式/条件、咨询起订量(MOQ)等（后台配置） |
| 当前阶段 | 下拉选择 | 条件必填 | 选项由管理员在后台配置（站点中心 → 业务流程配置）。若后台未配置任何阶段，阶段降级为非必填，仍允许保存跟进记录，同时弹窗提示"暂无跟进阶段配置，请联系管理员设置" |
| 下次跟进时间 | 日期选择器 | 否 | 默认为今日 + 3 天 |
| 跟进附件 | 文件上传 | 否 | 支持格式：图片 / PDF / Word / Excel，单个文件上限 10MB |


## 三、字段交互规则

### 3.1 线索名称

- 展示当前线索的编号及名称
- 格式：`LEAD-2026-0911 · Bella Home Deco`
- 通过批量入口触发时不展示线索名称字段（批量录入跟进不支持）

### 3.2 跟进方式

下拉选项：

| 选项 | 说明 |
| :--- | :--- |
| 电话 | 电话沟通 |
| 邮件 | 邮件往来 |
| WhatsApp | WhatsApp 沟通 |
| 会议 | 线下或线上会议 |
| 拜访 | 客户拜访 |
| 备注 | 其他类型的跟进记录 |

### 3.3 跟进内容

- 多行文本输入框
- 占位符提示：`请输入跟进内容，建议包含客户需求、反馈和下一步计划`
- 建议最少 10 个字符（前端友好提示，不强制校验）

### 3.4 客户关注

- 多选下拉组件，选项由管理员在后台配置
- 配置入口：站点中心 → 业务流程配置 → 客户关注选项管理
- 默认不选中任何选项
- 用于记录客户在跟进中表达的具体关注点，便于后续针对性跟进

### 3.5 当前阶段

- 下拉选择，选项由管理员在后台配置
- 配置入口：站点中心 → 业务流程配置 → 阶段管理
- 不同站点可配置不同阶段选项列表
- 示例选项：首次联系、需求确认、样品阶段、报价阶段、合同已成交等

**阶段与线索状态联动规则：**

| 条件 | 系统动作 |
| :--- | :--- |
| 选择阶段为“已打样” | 系统自动将线索状态变更为“高意向” |
| 选择阶段非“已打样” | 线索状态保持不变，由系统按原有规则判定 |

> 说明：“已打样”为示例阶段名称，实际匹配值由后台配置中勾选“触发高意向”标记的阶段决定。

### 3.6 下次跟进时间

- 日期选择器，精确到日（格式：YYYY-MM-DD）
- 默认值：**今日 + 3 天**
- 用户可手动修改
- 不填写时，该线索不会出现在“待跟进”快捷筛选中

### 3.7 附件

- 点击上传区域或拖拽文件进行上传
- 支持格式：图片（jpg/png/gif）、PDF、Word（doc/docx）、Excel（xls/xlsx）
- 单个文件大小上限：**10 MB**
- 多个文件可依次上传（建议上限 5 个）
- 上传后展示文件列表，支持删除


## 四、弹窗交互说明

### 4.1 打开弹窗

- 点击【跟进】或【录入跟进】按钮，弹出“录入跟进”弹窗
- 弹窗标题：`录入跟进 - [线索名称]`
- 所有可编辑字段均为空或默认值状态，等待用户填写

### 4.2 提交校验

| 校验项 | 校验规则 | 错误提示 |
| :--- | :--- | :--- |
| 跟进方式 | 不可为空 | “请选择跟进方式” |
| 跟进内容 | 不可为空 | “请输入跟进内容” |
| 当前阶段 | 条件必填，后台未配置阶段时降级为非必填 | “请选择当前阶段” / “暂无跟进阶段配置，请联系管理员设置” |
| 客户关注 | 选项取自后台配置，如有非法值则忽略 | — |
| 附件格式 | 仅支持指定格式 | “不支持该文件格式，请上传图片、PDF、Word 或 Excel 文件” |
| 附件大小 | 单个文件 ≤ 10 MB | “文件大小超过 10 MB 限制” |

### 4.3 提交成功

1. 关闭弹窗
2. 线索列表/详情页的跟进记录列表自动刷新
3. 线索的“最近跟进”字段更新为当前时间
4. 若阶段触发高意向（如“已打样”），线索状态自动变更为“高意向”
5. 页面顶部显示成功提示：`跟进记录录入成功`

### 4.4 提交失败

- 网络异常或服务端错误时，弹窗提示：`跟进记录录入失败，请稍后重试`
- 保留用户已填写的内容，不关闭弹窗，允许用户重新提交

### 4.5 取消/关闭

- 点击弹窗右上角关闭按钮或【取消】按钮，关闭弹窗
- 若用户已填写内容，弹出二次确认：`您填写的内容尚未保存，确认退出吗？`
- 确认后关闭弹窗，不保存内容


## 五、业务规则

| 规则编号 | 规则说明 |
| :--- | :--- |
| BR-030 | 每次关键沟通或状态变化必须新增跟进记录 |
| BR-031 | 跟进记录仅允许新增，不修改历史，不删除 |
| BR-033 | 高意向由系统基于跟进阶段自动判定，阶段后台可配置 |

**数据写入：** 提交后，系统写入以下数据：

| 目标对象 | 字段 | 来源 |
| :--- | :--- | :--- |
| 跟进记录 | 跟进方式 | 用户选择 |
| 跟进记录 | 跟进内容 | 用户输入 |
| 跟进记录 | 客户关注 | 用户选择 |
| 跟进记录 | 阶段 | 用户选择 |
| 跟进记录 | 下次跟进时间 | 用户选择 |
| 跟进记录 | 跟进时间 | 系统自动（提交时间） |
| 跟进记录 | 跟进人 | 系统自动（当前用户） |
| 线索 | 最近跟进时间 | 系统自动（提交时间） |
| 线索 | 状态 | 系统自动（阶段为“已打样”时变更为“高意向”） |


## 六、异常状态处理

| 场景 | 处理方式 |
| :--- | :--- |
| 附件上传失败 | 提示“附件上传失败，请重新上传”，保留其他已填内容 |
| 附件格式不支持 | 上传时即时校验并提示“不支持该文件格式” |
| 附件超过大小限制 | 上传时即时校验并提示“文件大小超过 10 MB 限制” |
| 网络异常提交失败 | 提示“跟进记录录入失败，请稍后重试”，保留已填内容 |
