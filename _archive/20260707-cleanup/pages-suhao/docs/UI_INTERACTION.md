# UI_INTERACTION.md

# 目标

本规范用于统一整个后台系统的交互方式。

所有页面必须遵循相同的交互规则。

禁止同一类业务出现不同的交互模式。

交互设计以：

* 高效率
* 一致性
* 易学习
* 易维护

为第一原则。

---

列表 → 编辑

不要随意跳转新页面。

# 一致性原则

同一种业务：

必须采用相同交互。

例如：

所有新增：

Drawer。

所有删除：

确认框。

所有详情：

详情页。

不要出现多个版本。

---

# 最终原则

每一个交互决策，都应满足以下要求：

1. 用户无需思考即可理解。
2. 同类操作保持一致。
3. 尽量减少点击次数。
4. 保持业务连续性，不频繁跳转页面。
5. 优先使用成熟企业后台交互模式，而非创新交互。

# 冻结后补充规范

以下规范来源于 Design Review，用于保证 UI Design Phase 后续执行一致。

## 搜索请求

所有服务端搜索：

必须通过：

* 点击查询
* 按 Enter

触发请求。

允许实时变化的场景仅限于：

* 当前页面已加载数据的本地过滤
* 输入框本地建议词过滤

例如：

WhatsApp 会话搜索如果请求服务端，必须点击查询或 Enter。

如果仅过滤当前已加载会话，可以本地实时过滤，但不得触发接口请求。

## 沟通中心业务操作顺序

邮件与 WhatsApp 的业务操作区保持一致。

推荐顺序：

1. 生成线索
2. 查看线索
3. 查看客户
4. 回复
5. 更多

已生成线索时：

隐藏「生成线索」主按钮，展示「查看线索」。

已归集客户时：

展示「查看客户」。

AI 不可用时：

仅降级 AI 区域，不影响生成线索、查看线索、查看客户等 CRM 主流程。

## 配置页保存

配置页采用分组保存。

适用于：

* 系统参数
* 角色权限

规则：

* 每个 Tab 或配置分组独立保存
* 保存按钮固定在当前配置区右上或底部操作条
* 保存成功后 Toast 提示
* 保存失败保留用户输入
* 切换 Tab、离开页面或关闭 Drawer 前，如存在未保存内容，必须提示确认

## 分析页钻取

分析中心中的指标卡、图表、排行数据支持钻取时：

* 必须先校验目标页面菜单权限
* 无目标页面权限时，不展示可点击态，或点击后 Toast 提示权限不足
* 有权限时，跳转目标页面并回填筛选条件
* 钻取后的数据仍受当前用户数据权限控制

## Empty State 操作入口

Empty State 中的操作入口必须受按钮权限控制。

有权限时：

展示对应操作入口，例如新增、分配、刷新。

无权限时：

不展示不可用操作入口，仅展示：

* 调整筛选
* 刷新
* 联系管理员

## 公海线索详情操作

从公海池进入线索详情时：

页面默认以查看和分配为主。

未分配线索不展示：

* 回收
* 录入跟进
* 客户转交

编辑、修改状态、标记无效、标记丢失等操作：

仅在用户拥有对应权限时展示。

分配成功后：

线索进入线索列表，并按已分配线索规则展示后续操作。

## 合同终态操作

合同状态为以下终态时：

* 已完成
* 已终止
* 已作废

所有入口统一隐藏编辑类和状态变更类操作。

包括：

* 合同中心列表
* 合同详情
* 客户详情合作合同
* 线索详情关联合同

终态合同仅允许查看。

## Toolbar 操作收纳

列表页 Toolbar 保持高频优先。

默认展示：

* 主要新增或核心动作
* 刷新
* 导出（如有权限且业务需要）

批量操作仅在选中数据后出现。

低频操作进入「更多」。

单行操作超过三个时，必须收纳到「更多」。
